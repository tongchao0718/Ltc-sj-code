# 项目记忆（project-memory）

> **模板**：`02-子应用通用模板/00-项目记忆模板.md`  
> **落盘**：`AIEP-WEB/src/docs/子应用文档/{app-code}/04-AI治理与审计/00-项目记忆.md`  
> **借鉴**：Axhub Make `project-memory` 工作流（轻量版）

## 何时启用

- `ai-full-process-design` **每次会话启动**（读取，必要时初始化）
- 步骤 3/4 中 PO/客户做出新决策或否决某方案
- 会话结束前仍有未完成的跨步动作
- 用户说「记住这个决策 / 更新项目记忆」

## 必要输入

- `app-code`
- 当前会话中的决策、否决、待办（来自对话或会议纪要）

## Workflow A：初始化

1. 若 `00-项目记忆.md` 不存在，从模板复制到 `04-AI治理与审计/`
2. 填写元信息：`app-code`、`platform_type`、`flow_type`
3. 若需求说明书已有明确决策，迁移至「已确认决策」表

## Workflow B：增量更新

1. 读取现有 `00-项目记忆.md`
2. 按类型追加：
   - **已确认决策** → 表格新行（日期、摘要、确认人、是否已写入 PRD）
   - **明确否决** → 表格新行（Agent 后续不得重复提议）
   - **风格偏好** → 列表追加
   - **跨会话待办** → 保持 ≤5 条；完成项删除
3. 更新「变更日志」与「最后更新」日期
4. 输出 **记忆更新摘要**

## Workflow C：启动读取（ai-full-process-design 调用）

1. Read `00-项目记忆.md`
2. 在 **阶段门禁状态块** 中增加一行：
   - `**项目记忆**：已加载 | 缺失（建议初始化）`
3. 若「明确否决」非空，本步生成方案时须规避
4. 若「跨会话待办」有过期项（>2 周未处理），在「下一动作」中提示

## 写入边界

| 可写入 | 不可写入 |
|--------|----------|
| 口头已确认、PRD 未同步的决策 | API 契约、GWT（→ SDD） |
| 客户否决的方案 | 已冻结 PRD 正文（应直接改 03-PRD） |
| 风格偏好、走查顺序 | 敏感凭据、个人信息 |

**同步规则**：决策写入记忆后，须在步骤 4 PRD 冻结前合并进 `03-PRD` 或 `04`，并标记「是否已写入 PRD/04」列为是。

## 输出：记忆更新摘要

```markdown
## 项目记忆更新
- **路径**：…/04-AI治理与审计/00-项目记忆.md
- **本次新增决策**：{N} 条
- **本次新增否决**：{N} 条
- **待办**：{N} 条开放 / {N} 条已完成
- **须同步 PRD**：{是/否 — 列表}
```

## 禁止

- 用项目记忆替代 `19-G2-A` 或 PRD 冻结
- 在记忆中写 API 路径作为契约真值
- 待办超过 5 条不清理
