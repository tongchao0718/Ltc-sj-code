# 界面 → 文档回写矩阵

> UI 代码为体验真值；SDD 为契约真值。回写时按下列顺序，避免下游文档引用过期上游。

## 扫描清单（从代码提取）

| 扫描项 | 代码位置 | 写入目标 |
|--------|----------|----------|
| 路由 path、name、component | `router/index.js`、`router/modules/*` | 04 §页面清单、映射表 |
| 菜单/侧栏 | 布局组件、`menu` 配置 | 04 §信息架构、01 §页面清单 |
| 页面分区 | `.vue` template 顶层区块 | 04 §页面说明 |
| 表单/表格字段标签 | `.vue` + `columns`/`form` 定义 | 04；03 功能需求摘要（非 SDD 字段） |
| 按钮与操作 | 事件处理、`@click` | 04 §交互；03 §流程摘要 |
| 四态 | `loading`/`empty`/`error` 分支 | 04 §状态设计 |
| 接口调用 | `api/`、`axios`、`mock` | 映射表接口名列 + SDD 引用 |
| 标注锚点 | `PrdAnnotationAnchor :id` | `annotations/*.js` |

## 各文档回写要点

### `04-界面设计文档.md`

- §页面清单：与 router 100% 对齐（页面 ID、路由、组件路径）
- 每 P0 页：布局分区、关键组件、四态说明、与上页/下页跳转
- §变更记录：追加版本行（日期 + 摘要）

### `页面-路由-接口-数据表映射表.md`

- 每行：页面 ID | 路由 | Vue 文件 | 接口名（详见 SDD §x）| 主表
- 新增/删除路由必须增删行；**禁止**留空路由

### `01-需求说明书.md`

- 仅当 UI 出现**文档未登记**的新页面/新 SA 级能力
- 新 SA 编号须与 PO 确认或沿用 03 模块映射；**禁止**自造与 03 冲突的 SA

### `03-PRD设计评审文档.md`

- 改：模块「（1）功能需求」「（3）界面设计」与 UI 一致的部分
- 不改：API 字段表（仍「详见 SDD §x」）
- 冻结版：只写报告 §待变更评审，**不**直接改正文（除非 PO 已书面同意）

### `annotations/*.js`

- 有变更的功能点：更新四段式 `sections`（纯文本）
- 新增功能点：新 `id` + `PrdAnnotationAnchor`（委托 `prd-page-annotation`）

### `00-项目记忆.md`

- 记录：走查后定稿的 UI 决策、否决项、待 PO 确认项

### `应用操作说明书.md`

- 按实际菜单路径更新操作步骤；截图说明可标「待补图」

## 差异严重级

| 级 | 示例 | 回写要求 |
|----|------|----------|
| **P0 blocking** | 代码有路由、映射表无；P0 页四态文档缺失 | 必须本次修复 |
| **P1 warning** | 04 文案与 UI 差一词 | 应修复 |
| **P2 info** | 变更记录未追加 | 报告记录即可 |

## SDD 冲突处理

UI/Mock 与 SDD 字段不一致时：

1. **不**直接改 SDD
2. 报告 §SDD 待办 列出：页面、字段、UI 表现、SDD 条目
3. 建议：`sdd-generation` 修订 + `20-G2-B`（若 Gate-2 已关闭）
