# AI 智能 CRM 管理（ai-smart-crm）需求说明书

## 1. 范围

- 子应用编码：`ai-smart-crm`
- **platform_type**：`web-admin`（Web 管理台本期交付；原生 APP 见需求 §5，独立工程）
- **flow_type**：`standard`
- 交付形态：**双端同步建设**
  - **APP 端**：iOS + Android（跨端统一工程，同期交付）
  - **Web 管理台**：PC 浏览器（AIEP 子应用，独立路由与打包）
- 路由与入口：
  - APP 深链 Scheme：`aiep://ai-smart-crm/`（见 §5.4）
  - Web 管理台路由前缀：`/ai-smart-crm-admin`
- 账号体系：**独立账号**（本子应用自洽，本期不打通 AIEP 主应用统一 SSO）
- 版本：`V1.2`

### 1.1 产品定位

**AI 智能 CRM** 采用「**移动作业 + Web 管理**」双客户端架构：

| 客户端 | 定位 | 核心用户 |
|--------|------|----------|
| **APP（iOS/Android）** | 外勤一线作业端 | 销售代表、客服、主管（移动督办） |
| **Web 管理台** | 后台管理与运营端 | 运营管理员、销售主管、管理层 |

共享同一后端 API 与数据模型。APP 以「单手可达、3 步内完成核心动作」为体验原则；Web 以「批量配置、全量分析、权限治理」为体验原则。双端均叠加 AI 能力（画像、话术、跟进建议、知识问答、销售预测）。

### 1.2 本期目标

同步建设并同期交付：

1. **APP 端**：移动工作台 → 客户/线索/商机作业 → AI 随身助手 → 推送触达  
2. **Web 管理台**：组织与账号 → 业务规则配置 → 批量导入导出 → 全量报表与审计  
3. **独立账号体系**：注册/登录/鉴权/角色权限，APP 与 Web 共用，数据与 AIEP 主账号隔离  

形成可独立部署、双端协同的 CRM 子应用闭环。

### 1.3 目标用户

| 角色 | 典型诉求 |
|------|----------|
| 销售代表 | 快速录入客户、跟进商机、获取 AI 话术与下一步建议 |
| 销售主管 | 查看团队漏斗、预测业绩、分配线索、督办逾期跟进 |
| 客服专员 | 受理工单、查看客户历史、协同销售 |
| 运营管理员 | 在 Web 配置阶段/标签/公海规则、维护知识库、批量导入、查看审计 |
| 管理层 | APP 看移动简报；Web 看全量经营报表与 AI 预测 |

### 1.4 不做项（本期明确排除）

1. 不包含与 **AIEP 主应用账号体系的 SSO 打通**（独立账号；后续版本可评估联邦登录）。
2. 不包含财务 ERP 全量记账、发票开具、税务申报能力。
3. 不包含营销自动化（EDM 群发、广告投放平台）的完整建设，仅预留活动记录与外部来源登记。
4. 不包含自研 IM 客户端（企微/钉钉/飞书仅外链唤起，不做会话同步）。
5. 不包含自研大模型训练与微调平台，AI 能力通过统一 AI 网关 / LLM API 调用实现。
6. 不包含 iPad/平板专属布局优化（APP 按手机竖屏适配；Web 管理台适配 1366×768 及以上）。
7. 不包含应用商店上架运营（资质、审核、ASO），但须具备可构建的双端安装包产物。
8. 不包含微信小程序 / 鸿蒙原生独立客户端（仅 iOS + Android + Web 管理台）。

### 1.5 双端协同原则（强制）

| 能力域 | APP 端 | Web 管理台 | 数据一致性 |
|--------|--------|------------|------------|
| 客户/线索/商机 CRUD | 一线录入与跟进 | 批量维护、分配、合并 | 实时一致，以服务端为准 |
| 业务规则配置 | 只读消费 | 配置发布 | 发布后 APP 下次拉取或推送生效 |
| 报表分析 | 个人/团队简报 | 全量多维报表与导出 | 指标口径统一，SDD 定义 |
| 知识库 | 检索问答 | 内容录入与版本管理 | 同一知识库源 |
| 账号权限 | 登录与个人设置 | 用户/角色/组织管理 | 同一账号体系 |

### 1.6 成功指标（立项基线）

| 指标 | 目标值 | 统计口径 |
|------|--------|----------|
| APP 冷启动至首页可交互 | ≤ 2.5s（P95） | 中端 Android / iPhone，4G 网络 |
| APP 核心列表首屏可见 | ≤ 2s（P95） | 375px 宽，4G 网络 |
| Web 管理台首屏可交互 | ≤ 3s（P95） | 1366×768，内网环境 |
| 主流程单手可达（APP） | 100% | 新建跟进、查看待办、外呼入口在拇指可达区 |
| 弱网可完成跟进提交 | ≥ 95% | 离线草稿恢复成功率 |
| 推送触达至打开转化 | ≥ 40% | 跟进提醒类推送（试点团队） |
| 独立账号登录成功率 | ≥ 99.5% | APP + Web 合计，排除用户输错 |
| Web 批量导入成功率 | ≥ 98% | 合法模板行的一次导入成功率 |
| AI 跟进建议采纳率 | ≥ 30% | 被采纳建议数 / 推送建议数 |
| 双端核心链路可追溯 | 100% | 客户-商机-跟进-配置变更可串联查询 |

## 5. 需求详细说明

### 5.1 销售运营中心

#### 5.1.1 销售工作台（SA-1）

1. **首页 Tab 布局**  
   顶部下拉刷新；首屏依次为 KPI 横滑卡片、今日待办列表、漏斗摘要条、AI 洞察卡片。

2. **概览指标区**  
   个人维度（默认）：今日待跟进数、本周新增线索、在途商机金额、赢单率；支持切换本周/本月（主管可看团队 Tab）。

3. **待办与提醒区**  
   卡片列表展示：待跟进商机、新分配线索、AI 推荐任务、待处理工单；左滑可「延后/忽略」（AI 建议类）。

4. **漏斗摘要区**  
   横向迷你漏斗条，展示各阶段数量；点击进入商机 Tab 对应阶段筛选。

5. **悬浮快捷入口（FAB）**  
   底部居中或右下角：快速「记跟进 / 新建线索 / 扫名片」三选一动作面板。

#### 5.1.2 客户管理（SA-2）

1. **客户列表（查）**  
   支持按客户名称、行业、级别、标签、负责人、创建时间、最近跟进时间筛选；分页、排序、列配置、导出（受权限控制）。

2. **客户新增/编辑（增改）**  
   必填：客户名称、客户类型（企业/个人）、负责人；选填：行业、规模、地址、官网、来源渠道、备注；支持标签多选与客户分级（A/B/C/D）。

3. **客户详情 360° 视图**  
   顶部：客户名、意向评分徽章、AI 摘要一行；Sticky 操作栏：外呼、发消息（唤起系统分享/剪贴板）、记跟进、AI 助手。  
   内容区纵向 Tab/分段：概览、联系人、商机、动态时间线、合同、工单；支持下拉刷新。

4. **客户合并与停用（删）**  
   支持重复客户合并（保留主记录、迁移关联数据）；停用后不在默认列表展示，历史数据保留可查。

5. **权限与数据隔离**  
   销售仅看本人及下属负责客户；管理员可看全量；跨团队查看需授权。

#### 5.1.3 线索管理（SA-3）

1. **线索列表与筛选**  
   字段含线索名称、来源、状态（新线索/跟进中/已转化/已失效）、负责人、评分、创建时间；支持批量操作。

2. **线索录入**  
   单条快速表单（必填项 ≤ 4 个）、名片 OCR（SA-14）、通讯录单选导入；来源登记支持展会/转介绍/广告等枚举。

3. **线索分配与公海池**  
   支持手动分配、自动轮询分配、回收至公海（超期未跟进触发）；公海领取规则可配置（每日上限、保护期）。

4. **线索转化**  
   一键转化为客户 + 联系人 + 初始商机；转化时映射字段并保留线索来源链路。

5. **线索评分**  
   规则评分（行业、规模、来源权重）与 AI 评分（SA-8）双轨展示，可配置以何者为主。

#### 5.1.4 联系人管理（SA-4）

1. 联系人 CRUD，关联客户；字段含姓名、职位、手机、邮箱、微信、决策角色（决策者/影响者/使用者）、是否主联系人。  
2. 同一客户仅允许一个主联系人；变更主联系人需记录日志。  
3. 联系人详情展示参与商机、历史活动、相关工单。  
4. 决策链在 APP 端以列表+标签展示（决策者/影响者），复杂关系图放 R2。

#### 5.1.5 商机与销售漏斗（SA-5）

1. **移动端双视图**  
   - **列表视图（默认）**：卡片展示商机名、客户、金额、阶段色条、预计成交日；支持下拉刷新与无限滚动。  
   - **阶段看板**：横向滑动分列（每列一个阶段），卡片点击进入详情；长按或右滑唤起「变更阶段」ActionSheet。

2. **商机 CRUD**  
   新建采用分步表单（选客户 → 填金额/阶段 → 确认）；编辑在详情页就地修改核心字段。

3. **阶段流转**  
   通过底部 ActionSheet 选择目标阶段；若该阶段有必填项（如报价附件），在当前页引导补全后提交。回退阶段需填写原因（弹窗）。

4. **赢单/输单关闭**  
   底部固定操作栏提供「赢单 / 输单」；输单原因用选择器；赢单可跳转合同登记（SA-7）。

5. **漏斗统计**  
   在「我的」或团队简报（SA-15）中以折叠卡片展示阶段转化率，不做复杂多维分析。

#### 5.1.6 活动与跟进记录（SA-6）

1. 活动类型：电话、拜访、微信、会议、其他；支持**语音转文字**速记（调用系统听写或 AI 网关）。  
2. **外勤能力**：一键外呼（`tel:` 协议）；拜访签到（GPS + 时间，R1）；挂断后自动弹出快速跟进页。  
3. 跟进计划：下次跟进时间用日期滚轮选择器；到期触发本地提醒 + 推送（SA-17）。  
4. 时间线：客户/商机详情以时间轴卡片展示，支持按类型筛选。  
5. 离线草稿：弱网或无网时跟进内容存本地，恢复网络后提示同步；冲突时以服务端为准并提示合并。

#### 5.1.7 合同与回款登记（SA-7）

1. 合同基础信息：合同编号、关联商机与客户、签约日期、合同金额、起止日期、状态（草稿/生效/完成/终止）。  
2. 回款计划：支持多期计划（计划日期、计划金额、状态）；实际到账登记触发状态更新。  
3. 合同列表与详情：支持筛选、导出；详情展示回款进度条与异常（逾期回款）。  
4. 与商机联动：赢单商机可一键生成合同草稿并预填金额。  
5. 边界：本期不做复杂审批流，仅做状态登记与备注；审批流预留扩展字段。

### 5.2 AI 智能能力

#### 5.2.1 AI 客户画像与意向评分（SA-8）

1. **画像标签**  
   根据客户属性、行为（跟进频次、商机阶段、工单情况）、外部导入标签，生成结构化画像（如「价格敏感」「决策周期长」「高成长行业」）。

2. **意向评分**  
   0–100 分制，展示分数、等级（高/中/低）、较上周变化；评分模型可配置权重。

3. **可解释性**  
   展示 Top 3 影响因子（如「7 天内 3 次主动咨询」「商机进入方案阶段」）；支持销售反馈「不准」并备注。

4. **刷新策略**  
   客户/商机/活动变更后触发异步重算；列表页展示最近评分时间。

5. **异常处理**  
   AI 服务不可用时展示最近缓存分数并标记「待更新」；不阻断核心 CRM 流程。

#### 5.2.2 AI 销售助手（SA-9）

1. **场景入口**  
   客户/商机详情页底部「AI 助手」按钮，唤起**半屏 Bottom Sheet** 对话面板；支持追问一轮。

2. **能力清单**  
   - 话术推荐：按场景（首次触达、异议处理、催单）生成 2–3 条，一键复制到剪贴板。  
   - 微信消息草稿：短文本为主，适配 IM 粘贴。  
   - 异议应对：语音或文字输入异议，返回要点列表。

3. **人机协同**  
   生成内容须用户确认后复制；采纳/弃用按钮固定展示；流式输出时显示打字机效果与停止按钮。

4. **合规与安全**  
   敏感词过滤；禁止自动生成虚假承诺；调用记录留痕（提示词摘要、模型版本、耗时）。

5. **限流与配额**  
   按角色配置每日调用上限；超限给出明确提示。

#### 5.2.3 AI 跟进建议与任务推荐（SA-10）

1. 每日批处理 + 关键事件触发（阶段变更、长期未跟进、客户行为异常）生成建议。  
2. 建议内容：建议动作、推荐理由、建议跟进时间、关联客户/商机链接。  
3. 用户动作：采纳（转跟进计划/待办）、忽略、延后；动作反馈用于优化。  
4. 首页待办卡片 + 「消息」Tab 双入口；未读角标；支持左滑忽略。  
5. 与推送联动（SA-17）：高优先级建议可推送，点击深链直达。

#### 5.2.4 AI 销售预测（SA-11）

1. 预测维度：个人、团队、产品线（按商机标签）、时间窗口（本月/本季/本年）。  
2. 输出：预测成交额、置信区间、基线对比（同比/环比）；列出关键支撑商机与风险商机。  
3. 假设说明：展示模型使用的特征摘要（在途商机金额、历史赢单率、阶段分布）。  
4. 人工校正：主管可对预测备注「市场因素」；备注进入报告但不直接改模型输出。  
5. 发布切片：R1；MVP 可提供规则版预测（加权在途金额），AI 版在 R1 增强。

#### 5.2.5 AI 知识库（SA-12）

1. 知识分类：产品手册、FAQ、竞品分析、销售 playbook、案例库（内容由后端/Web 管理台维护）。  
2. **APP 检索问答**：搜索框 + 对话式问答页；答案附带引用卡片，点击可查看原文摘要。  
3. **快捷入口**：AI 助手内「查知识」、我的页面「销售宝典」；支持收藏常用条目。  
4. 权限：按角色控制可见知识包；对外资料与内部资料分级。  
5. 反馈闭环：「有帮助/无帮助」反馈；弱网时展示最近缓存的热门 FAQ。

### 5.3 服务与数据

#### 5.3.1 客户服务工单（SA-13）

1. 工单列表：卡片流展示；Tab 筛选「待我处理 / 我创建的 / 全部」；支持下拉刷新。  
2. 工单创建：从客户详情一键创建；类型、描述、优先级、拍照附件；最少 3 步完成。  
3. 处理与协作：回复区区分「客户可见 / 内部备注」；支持转派（选择同事）。  
4. SLA 与推送：高优先级逾期标红；新工单推送至消息 Tab（SA-17）。  
5. 关闭：选择关闭原因即完成；客户详情时间线同步展示工单动态。

#### 5.3.2 移动端数据采集（SA-14）

1. **名片 OCR**：调用相机拍摄名片，识别姓名、手机、公司、职位；用户确认后生成线索草稿。  
2. **通讯录导入**：在系统授权前提下，单选联系人导入为线索（不做全量同步）。  
3. **拍照附件**：跟进/工单支持拍照或相册选图上传；压缩后上传，展示上传进度。  
4. **校验与去重**：手机号格式校验；与现有客户/线索重复时提示合并或放弃。  
5. **边界**：不做 Excel 大批量导入（归 Web 管理台）；OCR 失败时支持手工录入兜底。

#### 5.3.3 移动报表与简报（SA-15）

1. **个人简报（默认）**  
   本周跟进次数、新增线索、在途商机金额、赢单数；迷你漏斗图与趋势折线（7 日）。

2. **团队简报（主管）**  
   团队业绩排行 Top 5、逾期跟进名单、漏斗转化概览；点击成员可下钻其商机列表。

3. **AI 效果卡片**  
   建议采纳率、助手使用次数（本人/团队）。

4. **交互**  
   下拉刷新；时间范围切换（本周/本月）；不做 Excel 导出，支持系统分享截图（可选 R2）。

5. **性能**  
   指标数据优先读缓存摘要，后台静默刷新。

#### 5.3.4 APP 设置与个人中心（SA-16）

1. **登录入口（依赖 SA-20）**  
   手机号/账号 + 密码登录；登出；Token 自动刷新；可选生物识别解锁（本地快捷验证，不替代服务端鉴权）。

2. **消息设置**  
   跟进提醒、分配通知、AI 建议推送的独立开关；免打扰时段。

3. **隐私与显示**  
   敏感字段脱敏开关；缓存清理；当前版本号与检查更新。

4. **离线策略**  
   离线草稿保留时长、仅 Wi-Fi 同步附件等选项。

5. **边界**  
   销售阶段、公海规则、角色权限等**业务配置不在 APP 修改**，由后端/Web 管理台下发，APP 只读消费。

#### 5.3.5 消息推送与触达（SA-17）

1. **推送类型**  
   跟进到期提醒、线索分配、工单到达、AI 高优先级建议、系统公告。

2. **推送内容**  
   标题 + 一行摘要 + 深链参数（目标页类型、业务 ID）。

3. **触达策略**  
   合并同类通知（如 5 条跟进提醒合并为 1 条）；免打扰时段排队至时段结束。

4. **点击行为**  
   冷启动/热启动均可路由至目标页；失效数据展示友好空态页。

5. **回执**  
   送达、点击、忽略回执上报，供 SA-15 统计。

#### 5.3.6 APP 独立构建与发布（SA-18）

1. **工程要求**  
   独立移动端工程（建议 uni-app / React Native / Flutter，SDD 阶段冻结技术选型）；与 `AIEP-SERVER` API 对接。

2. **构建产物**  
   - Android：`.apk` / `.aab` 测试包  
   - iOS：TestFlight / 企业签测试包  

3. **环境与配置**  
   API 基址、AI 网关、推送 AppKey、深链 Scheme 可配置（dev/staging/prod）。

4. **基础能力**  
   启动页、登录态校验、全局错误边界、网络状态栏、基础埋点（页面 PV、核心按钮点击）。

5. **验收**  
   双端核心流程可跑通；深链从推送直达目标页；离线草稿可恢复。

#### 5.3.7 Web 管理台（SA-19）

1. **仪表盘**  
   全公司 KPI：线索量、商机金额、赢单率、逾期跟进、AI 采纳率；支持时间、团队、负责人筛选。

2. **主数据管理（与 APP 同源）**  
   客户、线索、商机、联系人、合同、工单的列表查询、详情查看、批量分配/变更负责人；字段口径与 APP 一致。

3. **业务规则配置**  
   - 销售阶段：名称、顺序、默认概率、阶段颜色、必填字段规则  
   - 字典与标签：行业、来源、丢单原因、工单类型、客户标签组  
   - 公海规则：回收天数、领取上限、保护期、自动分配策略  
   - AI 配置：模型网关、配额、提示词模板版本  

4. **批量导入导出**  
   Excel 模板下载；客户/线索/联系人/商机导入；行级校验与错误报告；按筛选条件导出（敏感字段脱敏）。

5. **知识库维护**  
   知识分类、富文本/Markdown 录入、附件、版本与生效日期；发布后对 APP 检索可见。

6. **全量报表**  
   线索来源、漏斗转化、业绩排行、跟进活跃度、合同回款、AI 效果；图表 + 明细下钻；导出 Excel/PNG。

7. **审计日志**  
   配置变更、导入导出、权限变更、关键业务操作可查；按操作人、时间、类型筛选。

8. **交互规范**  
   侧栏导航 + 列表/详情/表单标准页；符合 AIEP 系统设计规范；1366×768 及以上可用。

#### 5.3.8 独立账号与认证体系（SA-20）

1. **账号模型**  
   本子应用独立用户域（`crm_user`），与 AIEP 主应用用户表不关联；支持手机号或邮箱 + 密码。

2. **注册与开通**  
   - 默认：**管理员在 Web 创建账号**并初始密码/重置密码（本期不做开放自助注册）  
   - 首次登录强制修改密码（可配置）  

3. **登录与鉴权**  
   - APP / Web 共用登录 API，签发 JWT（Access + Refresh）  
   - 支持登出、单端登出、管理员强制下线  
   - 登录失败 N 次锁定（可配置）  

4. **组织与角色**  
   - 组织：公司 → 部门/团队 → 用户（树形）  
   - 预置角色：销售、主管、客服、管理员、管理层（只读报表）  
   - 数据权限：本人 / 本团队 / 全公司（按角色绑定）  

5. **权限下发**  
   菜单与 API 权限由角色驱动；Web 管理台可配置角色-权限矩阵；变更后 APP/Web 下次请求或推送生效。

6. **安全要求**  
   密码加密存储（bcrypt/argon2）；Token 安全传输（HTTPS）；敏感操作二次确认；操作审计留痕。

#### 5.3.9 Web 子应用独立打包（SA-21）

1. 遵循 AIEP 子应用创建指南：独立 Vite 配置、入口、路由，产物输出至 `dist/ai-smart-crm-admin/`。  
2. 路由前缀 `/ai-smart-crm-admin`；支持 Hash 路由嵌入主应用与独立访问。  
3. 构建命令：`npm run build:ai-smart-crm-admin`。  
4. 环境变量：API 基址、AI 网关、与 APP 共用同一后端。  
5. 验收：构建成功；登录后核心管理页可访问；权限拦截正确。

### 5.7 需求边界与异常场景（强制）

1. **数据边界**：子应用独立库 + 独立账号库；APP 与 Web 共用业务 API；不与 AIEP 主应用用户表混用。  
2. **账号边界**：独立账号体系；本期不支持 AIEP SSO 单点登录；企业微信等联邦登录归 R2。  
3. **AI 边界**：AI 输出为建议性质，不自动修改客户/商机核心字段；自动写库需用户确认（除批量离线评分）。  
4. **双端异常最小覆盖**：  
   - 断网 / 弱网 / 请求超时  
   - AI 网关超时 / 限流  
   - 推送权限未授权 / 深链目标失效  
   - OCR 识别失败 / 附件上传失败  
   - 公海领取并发冲突  
   - 商机阶段规则校验失败  
   - 离线草稿与服务端冲突  
   - 权限不足（403）与资源不存在（404）  
   - 登录 Token 过期 / 账号被锁定 / 被强制下线  
   - Web 导入部分行失败 / 配置发布冲突  
5. **恢复动作**：每类异常须定义重试、人工介入、降级或明确错误提示，且可在操作日志定位。  
6. **联动要求**：`03-PRD设计评审文档` 须分别覆盖 APP 与 Web 管理台页面；对应模块至少 3 条能力点。
