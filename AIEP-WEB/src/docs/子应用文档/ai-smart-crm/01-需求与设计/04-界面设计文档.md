## 二、全局界面规范（APP）

### 2.1 壳层结构

```
┌─────────────────────────────┐
│ Status Bar（系统）           │
├─────────────────────────────┤
│ NavBar：标题 + 返回/操作      │  ← 子页面
├─────────────────────────────┤
│                             │
│        Content Area         │
│                             │
├─────────────────────────────┤
│ TabBar（5项，子页面隐藏）     │  ← 首页/客户/商机/消息/我的
└─────────────────────────────┘
         [FAB] 仅首页/客户页
```

- Tab 图标+文案：首页、客户、商机、消息、我的
- 消息 Tab 未读角标（红点/数字，>99 显示 99+）
- 子页面隐藏 TabBar，左上返回

### 2.2 通用 UI 状态机（纯 UI，归 PRD）

| 状态 | 视觉 | 交互 |
|------|------|------|
| loading | 骨架屏（卡片×3 + 列表条×5） | 禁止重复触发 |
| success | 正常内容 | 下拉刷新可用 |
| empty | 插画 + 主文案 + CTA 按钮 | CTA 跳转新建/筛选 |
| error | 错误插画 + 原因 + 「重试」 | 点击重试重新请求 |
| offline | 顶部黄条「网络不可用，已保存草稿」 | 恢复后黄条消失 |

## 三、APP 核心页面设计

### 3.1 P-APP-00 登录页（M20）

**布局**

```
        [Logo]
   AI 智能 CRM
┌─────────────────────┐
│ 账号/手机号          │
├─────────────────────┤
│ 密码            👁   │
├─────────────────────┤
│      [ 登 录 ]       │  ← 主按钮，高度 48px
└─────────────────────┘
```

| 元素 | 规则 |
|------|------|
| 账号输入 | 必填；失焦校验非空 |
| 密码 | 必填；支持明文切换 |
| 登录按钮 | 提交中 loading+禁用；失败 Shake+Toast |
| 错误文案 | 401「账号或密码错误」；423「账号已锁定，联系管理员」 |

**状态机**：`idle → submitting → success(跳首页) | error`

---

### 3.2 P-APP-01 销售工作台（M01）

**布局（自上而下）**

1. **顶栏**：问候语「早上好，{姓名}」+ 日期
2. **KPI 横滑卡片**（宽 140px，间距 12px）：待跟进数、本周线索、在途金额、赢单率
3. **今日待办**（列表卡片，最多 5 条 +「查看全部」）
4. **漏斗摘要条**（横向迷你条图，点击进入商机 Tab）
5. **AI 洞察卡片**（高意向客户 Top3，点击进入客户详情）
6. **FAB**（右下 56px）：展开面板 → 记跟进 / 扫名片 / 新建线索

| 交互 | 说明 |
|------|------|
| 下拉刷新 | 重拉 KPI+待办+漏斗 |
| 待办左滑 | AI 建议类可「延后/忽略」 |
| 空态 | 「暂无待办，去跟进商机吧」+ 按钮跳商机 Tab |

**标注映射**：KPI 区→SA-1；待办→SA-1/10；FAB→SA-6/14

---

### 3.3 P-APP-02/03 客户列表与详情（M02）

**列表页**

- 顶部分段：**我的客户 | 全部**（主管可见团队筛选入口在筛选器内）
- 搜索栏：占位「搜索客户名称/手机」
- 筛选图标：行业、级别、标签（底部 Sheet 多选）
- 卡片项：头像 initials、客户名、级别徽章、意向分（若有）、负责人、最近跟进时间

**详情页**

```
┌─ Sticky 操作栏 ─────────────┐
│ [外呼] [记跟进] [AI助手]      │
├─────────────────────────────┤
│ 客户名  A级  意向分 82 ↑      │
│ AI 摘要一行…                  │
├─ Tab：概览|联系人|商机|动态 ──┤
│  （Tab 内容区）               │
└─────────────────────────────┘
```

| Tab | 内容 |
|-----|------|
| 概览 | 基本信息字段列表 |
| 联系人 | 联系人卡片列表 + 新增 |
| 商机 | 关联商机卡片 |
| 动态 | 活动时间线（M06） |

### 3.4 P-APP-04/05 线索与公海（M03/M14）

**列表顶部分段**：线索 | 公海

**公海卡片**：线索名、来源、评分、领取按钮（主色描边）

**录入页（分步）**

### 3.5 P-APP-06/07/08 商机（M05）

**列表/看板切换**：顶部 Segmented「列表 | 看板」

**看板**：横向 ScrollView 分列，每列标题=阶段名+数量；卡片显示商机名、客户、金额、预计成交日

**改阶段**：长按卡片 → ActionSheet 选目标阶段 → 若缺必填项弹补充表单

**详情页底部固定栏**：「变更阶段」「赢单」「输单」

### 4.2 P-WEB-01 管理仪表盘（M19）

- 第一行：4 个 Statistic 卡片（线索、商机金额、赢单率、逾期跟进）

### 4.3 P-WEB-02~04 主数据列表（M02/03/05）

**统一列表模板**

- 顶部：筛选 Form（inline）+ 查询/重置
- 工具栏：新建、批量分配、导出
- 表格：Arco Table，分页 20 条
- 行操作：查看、编辑、分配

### 4.4 P-WEB-05/06 业务配置（M19）

**销售阶段配置**

- 可拖拽排序列表（阶段名、颜色选择器、默认概率、必填字段多选）

**公海规则**

### 4.5 P-WEB-07 数据导入（M19）

## 八、交互闭环审查（V1.1）

> 审查范围：`AIEP-WEB/src/apps/ai-smart-crm-admin/` 可交互原型。  
> 共享状态：`store/crmStore.js`（Web 与 APP 手机壳共用内存数据，刷新后重置）。

### 8.1 已修复（原「按钮无响应」）

| 模块 | 原问题 | 现行为 |
|------|--------|--------|
| 客户管理 | 查询/重置/新建/编辑/导出无效 | 筛选生效；弹窗新建/编辑写入列表；CSV 导出；批量分配 |
| 线索管理 | 领取/转化/跟进无效 | 公海领取→我的线索；转化→客户；跟进弹窗保存 |
| 商机管理 | 详情/变更阶段无效 | 抽屉详情；阶段变更同步概率 |
| 用户管理 | 新建不落库；编辑/重置无效 | 列表增改；重置密码确认弹窗 |
| 销售阶段 | 编辑无效 | 编辑弹窗改名称/颜色/概率 |
| 数据导入 | 模板/上传无反馈 | 下载 CSV 模板；上传后写入对应列表 |
| APP 原型 | 新建/看板/消息/AI/FAB 等大量占位 | 见 §8.2 |

### 8.2 APP 手机壳已闭环路径

登录 → 工作台待办跳转 → 客户详情（概览/商机/动态 Tab）→ 跟进保存 → AI 复制/采纳 → 商机列表/看板 → 阶段切换 → 消息采纳/忽略（角标递减）→ FAB（跟进/扫名片/新建线索）→ 我的设置开关 → 退出。

### 8.3 仍为占位（符合 MVP/R1 约定）

| 项 | 说明 |
|----|------|
| P-WEB-08 经营报表 | R1 全量报表，MVP 引导至仪表盘 |
| P-WEB-10~12 | 角色/组织/审计页尚未建路由 |
| 登录鉴权 | 任意非空账号可进，无 JWT 持久化 |
| 后端 API | 原型为前端内存 Mock，非真实接口 |
