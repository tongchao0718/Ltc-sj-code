# AI 智能 CRM 管理 产品设计文档

## 1. 产品概述

### 1.1 产品名称

AI 智能 CRM 管理（ai-smart-crm）

### 1.2 产品定位

双客户端智能化 CRM：**APP（iOS/Android）** 服务外勤一线作业，**Web 管理台** 服务配置治理与批量运营。共享独立账号体系与业务 API，与 AIEP 主应用账号隔离。AI 能力贯穿客户洞察、销售助手、跟进建议与知识问答。

### 1.3 产品价值

- **移动优先**：外勤场景 3 步内完成跟进、外呼、建档，弱网可草稿恢复
- **AI 增效**：上下文话术、跟进建议、意向评分可解释，降低销售遗漏与沟通成本
- **双端协同**：Web 配置一次、双端生效；报表口径统一，管理可追溯
- **独立交付**：子应用可独立账号、独立打包、独立部署

### 1.4 目标用户

| 客户端 | 角色 | 核心场景 |
|--------|------|----------|
| APP | 销售代表 | 待办、客户跟进、商机推进、AI 助手 |
| APP | 客服专员 | 工单处理、客户历史查看 |
| APP | 销售主管 | 移动简报、逾期督办 |
| Web | 运营管理员 | 规则配置、导入导出、知识库、账号 |
| Web | 管理层 | 全量报表、审计 |

## 2. 系统架构

### 2.1 技术栈（SDD 冻结口径）

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| APP 端 | **uni-app（Vue 3）** + uni-ui | iOS/Android 跨端；深链 Scheme `aiep://ai-smart-crm/` |
| Web 管理台 | **Vue 3** + Vue Router + Pinia + **Arco Design** | 路由前缀 `/ai-smart-crm-admin`；AIEP 子应用独立打包 |
| 后端 | **Node.js + Express** + Mongoose | 目录 `AIEP-SERVER/src/apps/ai-smart-crm/` |
| 数据库 | **MongoDB** | 业务库 + 独立账号域 |
| 中间件 | Redis（缓存/限流，MVP 可选）、推送网关 | AI 调用经统一 AI 网关 |
| 鉴权 | JWT（Access + Refresh） | APP/Web 共用，独立 `crm_user` 域 |

## 3. 交互原则

### 3.1 APP 端（遵循 `系统设计规范-ant.md` 移动端章节 + iOS HIG / Material 3）

- **移动优先**：基准宽度 375px；核心操作拇指可达；热区 ≥ 44×44px
- **步骤控制**：新建跟进、扫名片、改阶段等主路径 ≤ 5 步
- **导航**：底部 Tab 固定 5 项；详情页统一左上返回；推送深链可直达详情
- **反馈**：点击 100ms 内有响应；>300ms 显示骨架屏；成功 Toast、失败 Dialog 带原因
- **弱网**：跟进支持离线草稿；断网/超时/服务异常文案区分
- **表单**：键盘弹起时滚动聚焦字段；枚举用选择器；提交按钮底部常驻

### 3.2 Web 管理台（遵循 AIEP 系统设计规范 + Arco 模式）

- **列表→详情→操作** 三段式；复杂配置用分步表单或抽屉
- **批量操作** 需二次确认；导入显示行级错误报告
- **权限**：无权限菜单不展示；接口 403 统一提示

### 3.3 双端一致口径

- 客户/商机/线索状态枚举、阶段颜色、指标口径以 SDD 标准编码为准
- 配置发布后 APP 下次启动或下拉刷新拉取；关键变更可推送通知

## 4. 设计风格（ui-ux-pro-max · CRM 产品类型 #101）

### 4.1 风格定位

- **风格**：Flat Design + Minimalism（专业 B2B CRM）
- **气质**：Professional + Clean hierarchy
- **动效**：150–300ms 状态过渡；禁止纯装饰动效

### 4.2 色彩系统

| Token | 色值 | 用途 |
|-------|------|------|
| primary | `#2563EB` | 主按钮、链接、选中 Tab |
| primary-light | `#3B82F6` | 悬停、次级强调 |
| success / won | `#059669` | 赢单、成功、高意向 |
| background | `#F8FAFC` | 页面背景 |
| surface | `#FFFFFF` | 卡片、面板 |
| text-primary | `#0F172A` | 标题、正文 |
| text-secondary | `#64748B` | 辅助说明 |
| border | `#E4ECFC` | 分割线、边框 |
| danger | `#DC2626` | 错误、逾期、输单 |

商机阶段看板：每阶段配置独立阶段色（Web 可配置），默认按蓝→靛→紫→绿渐变表达推进感。

### 4.3 字体与排版

| 端 | 字体 | 字号基线 |
|----|------|----------|
| APP | 系统默认（PingFang SC / Roboto） | 正文 16px，辅助 14px，标题 18–20px |
| Web | Arco 默认（14px 基准） | 表格 14px，标题按 Arco Typography |

行高 1.5；正文不小于 12px。

### 4.4 组件模式（CRM 典型）

- **联系人/客户**：左头像 + 名称 + 标签 + 右箭头卡片
- **商机看板**：横向分列卡片；金额 + 概率 + 预计成交日
- **活动时间线**：左侧时间节点 + 右侧类型图标与摘要
- **AI 助手**：半屏 Bottom Sheet（APP）/ 右侧抽屉（Web）
- **快捷操作**：FAB 展开面板（记跟进 / 扫名片 / 新建线索）

### 4.5 无障碍与触控

- 对比度 ≥ 4.5:1；图标按钮带 aria-label / accessibilityLabel
- 不单靠颜色传达状态（配合图标/文案）
- 支持系统字号缩放；Reduced motion 下降级动画

## 5. 发布与迭代策略

| 项 | 策略 |
|----|------|
| 发布方式 | APP 安装包（TestFlight/内测 APK）+ Web 子应用嵌入 AIEP 或独立部署 |
| MVP 范围 | APP SA-1~10、14、16~18；Web SA-19~21；共用 SA-20 |
| 迭代节奏 | 双周迭代；MVP 后 R1 补知识库/工单/全量报表 |
| 灰度 | 按组织/团队灰度推送与 AI 配额 |
| 回滚 | 后端可回滚 API；APP 强制最低版本；Web 静态资源版本回退 |
| 风险控制 | 配置发布需审计；导入批次可标记失效；AI 输出不自动写库 |
