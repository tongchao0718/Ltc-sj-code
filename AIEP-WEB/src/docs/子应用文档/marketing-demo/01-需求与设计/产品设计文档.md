# 售电公司营销分析 Demo 产品设计文档

## 1. 产品概述

### 1.1 产品名称

售电公司多维信息查询与分析系统（Demo）— `marketing-demo`

### 1.2 产品定位

面向电网营销、交易结算及监管业务人员的 **Web 管理端 Demo**，汇聚售电公司工商、交易、结算、风险等多维数据，提供九维查询、四维分析、业务画像、一键报告、结算协议全生命周期与履约管控能力。数据层以 Mock + 可替换接口契约为主，界面与交互按生产级 YXUI 规范建设。

### 1.3 产品价值

- **规范落地**：全站遵循《营销设计规范》，可作为营销域子应用 UI 标杆
- **全链路闭环**：MD-1～MD-23 端到端可走通，Mock 主数据跨模块关联
- **真实交互**：所有操作走 Mock API，loading/empty/error/success 四态完整
- **独立交付**：可 `build:marketing-demo` 独立打包，亦可嵌入 AIEP 主应用应用中心

### 1.4 目标用户

| 角色 | 核心场景 |
|------|----------|
| 营销业务人员 | 九维查询、导出、跳转画像/报告 |
| 数据分析人员 | 四维分析、图表钻取、对比研判 |
| 监管/管理人员 | 全省格局看板、一键分析报告 |
| 协议管理员 | 模板 CRUD、签订/变更/续签/作废 |
| 履约管理员 | 临期预警、流程跟踪、六维评估 |
| 系统管理员 | 数据接入配置、同步日志、脱敏权限 |

## 2. 系统架构

### 2.1 技术栈

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| 前端框架 | Vue 3 + Vue Router | 组合式 API；Hash 路由 |
| 状态管理 | Pinia（`marketingStore.js`） | Mock 数据持久化、消息、配置 |
| UI 实现 | 自研 YXUI Token + CSS 变量 | **不污染**主应用 Arco 全局样式；作用域 `.marketing-demo-scope` |
| 图表 | ECharts（`MdLineChart.vue` 等） | 看板折线/柱状/雷达/热力 |
| Mock API | `mock/api.js` + `mock/initialData.js` | 可替换为真实 REST |
| 打包 | Vite 独立配置 | `build/vite.marketing-demo.config.js` |
| 主应用嵌入 | `subApps.js` + `router/index.js` + `App.vue` | 路由前缀 `/marketing-demo` |

### 2.2 信息架构

| 一级模块 | 二级页面 | 产品目标 | 关键交互边界 |
|------|------|------|------|
| 首页 | 综合工作台 | KPI 总览、预警摘要、快捷入口 | 看板类；卡片可跳转各模块 |
| 数据接入 | 接入总览 / 工商 / 交易 / 结算 / 治理 | 三源同步监控与配置 | 立即同步→日志→下游查询可读 |
| 综合查询 | 九维查询页（9） | 统一筛选→表格→详情→导出 | 列表查询类；临期/未结清高亮 |
| 多维分析 | 效益 / 市场 / 风险 / 对比 | 图表研判与钻取 | 看板类；≥4 条钻取路径 |
| 业务画像 | 列表 + 详情 | 五类标签 + 自定义 + 历史 | 详情类；PDF 导出 |
| 一键报告 | 模板库 + 生成 | 三标准模板 + 定制 + 三格式 | 表单类；溯源跳转查询页 |
| 结算协议 | 模板 / 生命周期 / 上传 | 四环节状态机 + 校核 | 列表+表单+弹窗 |
| 履约管控 | 临期 / 流程 / 评估 | 预警配置 + 五节点 + 六维 | 列表+流程+看板 |

详细流程、接口、DoD 见 `03-PRD设计评审文档.md` 与 `04-界面设计文档.md`。

### 2.3 端到端主链路

```
数据接入(MD-1) → 九维查询(MD-2~10) → 多维分析(MD-11~15)
    → 业务画像(MD-16) → 一键报告(MD-17)
    → 结算协议(MD-18~20) → 履约管控(MD-21~23)
```

统一主键：`companyId` / 统一社会信用代码。

## 3. 交互原则

### 3.1 壳层与导航（YXUI §布局）

- **结构**：顶栏 64px + 侧栏 235px（可收至 54px）+ 页签 36px + 内容区
- **路由前缀**：嵌入主应用 `/marketing-demo/*`；独立打包无前缀
- **页签**：打开菜单项新增 Tab；首页 Tab 不可关闭；关闭 Tab 时路由回退至相邻 Tab
- **侧栏**：分组折叠；当前路由高亮 `#3A96FE` 左边框

### 3.2 列表查询统一范式（MD-2～MD-10）

1. 筛选区：组合条件 + 重置 + 查询（变更条件重置 pageNo=1）
2. 工具栏：导出 Excel/PDF
3. 表格：斑马纹、hover `#EDF5FF`、分页 10/20/50
4. 行操作：详情弹窗；可跳转画像/报告/分析
5. 四态：loading 骨架 / empty 插画+文案 / error 重试 / success 正常

### 3.3 弹窗与确认

- 详情 Dialog 宽度 60%～80%
- 删除/作废/驳回须 MessageBox 二次确认
- 提交中按钮 loading + 禁用，防重复提交

### 3.4 跨模块跳转

- 报告溯源：携带 `companyId`、查询条件 query 参数，目标页自动查询并高亮行
- 分析钻取：汇总图表点击 → 明细弹窗或跳转查询页
- 画像入口：查询详情弹窗「查看画像」→ `/profile/:companyId`

### 3.5 权限 Demo（NFR-7）

- 角色 A「业务只读」：查询/分析/画像/报告只读；协议审核按钮 disabled
- 角色 B「协议审核」：协议全生命周期可操作
- 无权限：按钮 disabled + Tooltip 说明原因

## 4. 设计风格（营销设计规范 · YXUI）

### 4.1 风格定位

- **风格**：营销 2.0 Web UE · 专业 B 端后台
- **气质**：信息密度适中、层级清晰、蓝白主调
- **动效**：150–300ms 过渡；禁止纯装饰动效

### 4.2 色彩 Token

| Token | 色值 | 用途 |
|-------|------|------|
| primary | `#3A96FE` | 主按钮、链接、选中态、侧栏高亮 |
| primary-hover | `#5AA8FF` | 按钮悬停 |
| success | `#67C23A` | 成功、正常状态 |
| warning | `#E6A23D` | 临期预警、未结清欠费 |
| danger | `#F56C6C` | 高风险、失败、作废 |
| background | `#F0F2F5` | 页面背景 |
| surface | `#FFFFFF` | 卡片、面板 |
| table-hover | `#EDF5FF` | 表格行 hover |
| text-primary | `#333333` | 标题、正文 14px |
| text-secondary | `#666666` | 辅助说明 |
| border | `#DCDFE6` | 分割线 |

### 4.3 布局尺寸

| 元素 | 尺寸 |
|------|------|
| 顶栏高度 | 64px |
| 侧栏宽度 | 235px（收起 54px） |
| 页签高度 | 36px |
| 正文字号 | 14px |
| 基准分辨率 | 1920×1080；最小 1366×768 |

### 4.4 组件复用清单

| 组件 | 路径 | 用途 |
|------|------|------|
| MdQueryLayout | `components/MdQueryLayout.vue` | 九维查询统一壳（筛选+表格+分页） |
| MdDetailModal | `components/MdDetailModal.vue` | 详情 Descriptions 弹窗 |
| MdLineChart | `components/MdLineChart.vue` | 折线/柱状图封装 |
| useMarketingBase | `composables/useMarketingBase.js` | 路由前缀、相对路径 |
| useMarketingTabs | `composables/useMarketingTabs.js` | 页签状态机 |

## 5. 发布与迭代策略

- **发布方式**：AIEP 主应用嵌入（`/app-center` 入口）+ `dist/marketing-demo` 独立静态包
- **迭代节奏**：Demo 一次性全量交付 MD-1～23；缺陷在 G3 前闭环
- **风险控制**：样式作用域隔离；Mock API 契约与 SDD 同步冻结后禁止口头改字段
