# marketing-demo 详细设计文档

## 1. 模块实现设计

| 模块 | 核心实现 | 关键文件 |
|------|----------|----------|
| 壳层 | 顶栏+侧栏+页签；侧栏 235/54 收缩；多页签路由 | `MarketingDemoApp.vue`, `useMarketingTabs.js`, `menu.js` |
| M01 接入 | 三源 syncIntegration；14 Tab getBusinessInfoTabs；治理 saveGovernance | `mock/api.js`, `data-integration/*` |
| M02 查询 | MdQueryLayout + queryConfigs 驱动；queryList 分页筛选 | `components/MdQueryLayout.vue`, `config/queryConfigs.js` |
| M03 分析 | ECharts 看板；钻取 router.push 或 MdDetailModal | `components/MdLineChart.vue`, `views/analysis/*` |
| M04 画像 | crud profiles；详情标签增删 persistStore | `views/profile/*`, `mock/api.js` |
| M05 报告 | generateReport Mock；溯源带 query 跳转 | `views/report/*` |
| M06 协议 | crud + updateAgreementStatus 四环节 SM-01 | `views/agreement/*` |
| M07 履约 | saveExpiryDays；runAssessment 六维雷达 | `views/performance/*` |
| 嵌入 | useMarketingBase base 前缀；独立 Hash 双模式 | `composables/useMarketingBase.js` |

## 2. 状态管理

- 全局数据：`marketingStore`（reactive + localStorage `marketing-demo-store-v1`）
- 页签：`useMarketingTabs` 内存态
- 请求四态：页面内 `loading / success / empty / error`（SM-03）

## 3. 组件复用

| 组件 | 用途 |
|------|------|
| MdQueryLayout | 九维查询统一筛选+表格+导出 |
| MdCrudLayout | 模板/列表 CRUD |
| MdDetailModal | 详情弹窗 |
| MdFilterForm | 筛选表单 |
| MdPagination | 分页 |

## 4. 异常处理

- Mock delay 模拟网络；失败时页面 error 态 + 重试按钮
- persistStore 失败静默（localStorage 不可用）
- 导出 exportMock 返回 fileName 供下载提示

## 5. G3 测试要点

- 各查询页筛选 → 分页重置 → 四态
- 同步后下游查询数据一致（同一 companyId）
- 协议状态机非法流转拦截
- 主应用沉浸/平台模式 + AT-22
