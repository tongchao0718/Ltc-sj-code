# 电费协议核查（power-fee-protocol-check）需求说明书

## 1. 范围

- 子应用编码：`power-fee-protocol-check`
- 主路由前缀：`/power-fee-protocol-check`
- 版本：`V2.1`

本期目标：建设“协议核查管理 -> 异常分析治理 -> 识别能力优化”的全链路体系，实现协议识别、核查、治理与持续优化闭环。

## 5. 需求详细说明（V2.0基线）

### 5.1 协议核查管理

#### 5.1.1 协议模板库管理

1. 列表查询与展示（查）  
提供模板管理列表页，支持按协议类型、模板编码、模板名称、生效状态、版本号、更新时间进行筛选，支持分页、排序、导出；列表字段至少包含模板编码、模板名称、协议类型、当前版本、状态、更新人、更新时间，并支持查看详情抽屉展示字段配置与映射关系。

2. 新增模板（增）  
支持新建模板并选择协议类型（预存交费协议、分次结算协议、高压预购电协议、低压后付费协议、高压后付费协议），填写模板基础信息后进入字段配置；模板定义需抽取关键字段，包括协议编号、用电户号、用户户名、交费方式、余额告警阈值、协议有效期、签订日期等。新增保存后生成初始版本并记录创建人、创建时间。

3. 编辑与版本管理（改）  
支持在原模板基础上编辑字段规则、校验规则、基准映射关系，保存时自动生成新版本并保留历史版本；支持版本对比、启停、回滚。对已发布模板的编辑采用“草稿-发布”模式，避免直接覆盖线上生效版本。

4. 删除与停用（删）  
支持删除未被引用的草稿模板；已被规则或任务引用的模板不允许物理删除，仅允许停用。删除/停用前需二次确认并提示影响范围，操作需全量记录审计日志。

5. 协议对应档案信息库配置  
以营销系统用电客户档案为数据源，按模板字段映射关系定期抽取或同步核心档案信息，构建协议档案信息基准库，作为 OCR 识别结果自动校验基准；映射配置支持列表查询、单条维护、批量校验与差异提示。

#### 5.1.2 协议样本标注管理

1. 样本列表与详情展示（查）  
提供样本管理列表页，支持按协议类型、样本状态、导入批次、标注人、质检结果、时间范围查询；支持分页、排序、批量选择与导出。详情页展示原文件预览、标注结果、质检意见、操作日志。

2. 样本导入与登记（增）  
从营销系统收集五类电费协议非结构化文件，支持批量导入、单条补录与来源登记，形成 OCR 样本初始库；入库时自动生成样本唯一标识并记录批次信息。

3. 样本更新与重标（改）  
支持对样本元数据、标注结果、质检结论进行修订；驳回样本支持退回重标并保留历史版本，形成“待标注-标注中-待质检-驳回重标-已通过”状态流转闭环。

4. 样本删除与失效（删）  
支持删除未进入训练/校核流程的无效样本；已被引用样本不允许物理删除，改为“失效”处理并记录原因，确保可追溯。

5. 协议样本预处理与标注  
执行梳理、分类去重、无效剔除、质量核查，形成有效样本；按标注规范进行样本清洗、字段标注、签章识别标注，形成高质量标注数据。

#### 5.1.3 核查规则管理

1. 规则列表与命中明细展示（查）  
建立可配置规则库，支持按模板分类、规则编号、规则类型（完整性/一致性/合规性）、生效状态、版本号查询；列表支持分页、排序、导出，详情展示规则表达式、适用场景、历史版本、命中统计与执行日志。

2. 规则新增（增）  
支持新增规则并配置核对方式、触发条件、优先级、生效范围；新增时执行语法校验与冲突校验，通过后进入草稿状态。

3. 规则编辑发布（改）  
支持编辑草稿规则、复制规则、发布新版本；规则启停或内容变更必须生成新版本，自动记录版本号、修改时间、修改人。支持版本对比与回滚。

4. 规则删除停用（删）  
支持删除未发布规则；已发布或被场景引用规则不允许物理删除，仅允许停用，并要求填写停用原因。

5. 规则应用场景配置与执行留痕  
支持不同核查场景绑定规则集，实现一场景一套针对性规则；记录执行时间、协议文件 ID、规则结果（通过/不通过）、异常信息等，支持按任务与规则维度追溯。

#### 5.1.4 核查任务策略管理

1. 策略与任务列表展示（查）  
提供策略列表页与任务列表页。策略列表支持按供电单位、协议场景、执行模式、状态查询；任务列表支持按任务批次、执行状态、时间范围查询。页面需展示任务进度、完成数量、识别成功率、失败数量与平均耗时等指标。

2. 策略新增与任务发起（增）  
支持按供电单位、协议场景配置自动/手工任务策略，维护任务明细；支持单次任务立即发起与按策略批量生成任务。

3. 策略编辑与任务处置（改）  
支持编辑策略参数（调度周期、执行窗口、重试次数、告警阈值）；支持任务暂停、恢复、重试、改派等运维动作并保留操作日志。

4. 策略删除停用（删）  
支持删除未生效策略；已生效策略仅允许停用，停用后不再自动生成新任务，但历史任务仍可查询追溯。

5. 核查任务生成与实时监控  
任务引擎按策略筛选增量协议，批量发起 OCR 与核查流程；页面实时展示任务进度、完成数量、识别成功率等指标，并对超时/失败任务触发告警。

### 5.2 协议核查异常分析及治理

#### 5.2.1 协议识别核查

针对存量协议按资源分配分批执行核查计划，监控执行状态并处理异常，确保任务按计划完成。

#### 5.2.2 协议信息提取引擎

配置营销系统新增业扩用户、协议维护变动用户的信息提取规则，形成增量核查对象。

#### 5.2.3 协议信息解析

1. 协议信息解析：解析户号、户名、用电地址、交费方式及签署信息并结构化存储。  
2. 协议文字识别：非结构化文档文字识别为可编辑文本。  
3. 协议表格识别：识别表头、表尾、单元格及行列位置。  
4. 协议签字签章识别：提取签章签字图像并输出识别结果。

#### 5.2.4 识别结果校核

1. 协议校核服务：  
- 完整性校核服务：协议类型匹配、必要条款完整性、必填内容完整性。  
- 准确性校核服务：将协议关键字段与档案信息比对判定准确性。  

2. 识别结果校核：基于 OCR 结构化数据执行完整性与准确性核查并落库。  
3. 识别效果检查：对比原始文件与识别结果，针对高错误率问题执行人工检查与重新识别。

#### 5.2.5 识别问题结果

系统记录模板类型选择准确性、条款完整性、必填项完整性、档案一致性等异常问题。

#### 5.2.6 问题分析及治理

1. 识别问题分析：定时汇总 OCR 结果，区分技术准确性问题与业务签署规范问题，形成分析报告。  
2. 核查问题治理：自动分类归集并生成标准治理工单，推送责任单位处理。  
3. 核查问题跟踪：按协议类型、问题类型、单位、状态、时间等维度查询与导出问题清单。

#### 5.2.7 核查异常监控

1. 核查任务监控：执行效率评估、日志记录、异常告警、分级查询、异常详情、处理标记、监控报表。  
2. 核查结果监控：问题汇总看板、识别质效看板、治理看板、固定报表。

### 5.3 协议识别能力优化

依据异常监控结果持续优化模板格式特征与关键信息分布，基于迭代样本提升识别准确率，形成“样本分析-识别优化-测试验证-反馈迭代”常态化闭环。

#### 5.3.1 协同复核

1. 协议识别问题反馈：业务人员反馈漏字、错字、格式错误等问题，技术人员复核。  
2. 人工复核纠偏：重点纠偏签名、数值、日期、相似字符误判、复杂版面误读、上下文错误。  
3. 错误样本挖掘：沉淀典型错误案例，形成易错字库。

### 5.5 需求边界与异常场景（强制）

1. 边界范围：仅覆盖电费协议核查链路，不包含跨系统结算核心改造与财务记账改造。  
2. 数据边界：营销系统为唯一档案基准数据源；外部补录数据必须标记来源与可信级别。  
3. 异常场景最小覆盖：OCR 失败、表格错位、签章识别失败、规则引擎超时、任务执行中断、工单派发失败。  
4. 恢复动作要求：每类异常必须定义重试、人工介入、回滚或降级动作，且可在日志中定位。  
5. 防遗漏要求：新增需求必须同步补齐本章矩阵中的“输入/处理/输出/异常/验收”五类信息。  

### 5.6 菜单与信息架构约束（强制）

1. 前端菜单展示名称禁止使用模块编号或需求编号（如 M01、SA-1 等），统一使用业务语义名称。  
2. 菜单必须采用层级结构，且最大深度不超过五级（L1~L5）；当前基线按“一级分组 + 二级功能”落地。  
3. 当信息架构复杂导致可能超过五级时，必须通过菜单合并、页内 Tab、筛选器或面包屑分流降级，不得新增第六级菜单。  
4. 菜单名称需与页面标题、面包屑、权限点名称一致，避免一项功能多命名。  
5. 上述规则需与产品设计文档、PRD 设计评审文档和页面实现保持一致；不一致时以本条为触发项执行文档联动更新。
