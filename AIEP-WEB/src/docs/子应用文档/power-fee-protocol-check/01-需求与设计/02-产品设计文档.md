# 电费协议核查（power-fee-protocol-check）产品设计文档

## 1. 产品定位

面向电费协议核查场景，建设“配置-核查-治理-优化”一体化平台，提升协议数据治理效率与合规水平。

### 2.1 菜单设计规范

1. 开发实现中的菜单名称禁止带编号（如 M01、M02、SA-1 等）。  
2. 菜单采用“一级菜单 + 二级菜单”层级结构，一级仅做分组，二级为可点击功能页面。  
3. 页面标题、面包屑、侧栏菜单名称需保持一致，统一使用业务名称，不使用编号别名。  
4. 模块编号仅用于需求追踪、接口映射、评审与测试文档，不出现在前端菜单展示中。  
5. 菜单层级最大深度为五级（L1~L5）；超过五级必须通过合并菜单、页内 Tab 或面包屑分流优化。  
6. 本项目当前落地范式为二级菜单；后续扩展需保持兼容且不得突破“五级上限”。

## 3. 核心业务价值

- 统一核查标准，降低规则口径分散风险；
- 强化异常闭环，提高问题治理效率；
- 建立持续优化机制，提升 OCR 识别准确率与稳定性。
