## 一、项目概述

### 1.1 项目简述

建设协议核查全流程平台，覆盖模板配置、样本标注、规则管理、任务策略、识别核查、信息提取、信息解析、结果校核、问题治理、任务监控、结果监控、能力优化、协同复核等 14 个模块。

### 1.2 项目目标

1. 建立统一核查标准和规则版本体系。
2. 实现任务自动生成与异常闭环治理。
3. 建立识别能力持续优化机制。

### 1.3 交付范围

- 前端范围：14 个业务模块页面、路由、交互、监控看板与问题闭环操作。
- 文档范围：需求、PRD、SDD、详细设计、数据库、门禁与审计。
- 非本期范围：跨系统结算核心改造。

## 二、系统流程

### 2.1 业务流程

```mermaid
flowchart LR
  A[M01 模板与基准] --> B[M02 样本标注]
  B --> C[M03 规则发布]
  C --> D[M04 任务策略]
  D --> E[M06 增量提取]
  E --> F[M05 核查执行]
  F --> G[M07 协议解析]
  G --> H[M08 结果校核]
  H --> I[M09 问题记录]
  I --> J[M10 问题治理]
  J --> K[M11/M12 监控看板]
  K --> L[M13 能力优化]
  L --> M[M14 协同复核]
  M --> C
```



主链路：协议模板配置 → 样本标注 → 规则发布 → 任务生成 → 信息提取 → 识别核查 → 解析校核 → 异常落库 → 治理闭环 → 监控统计 → 优化迭代 → 协同复核回写。

### 2.2 操作流程（评审关注）

- **导航流程**：一级菜单分组（协议核查管理 / 异常分析及治理 / 识别能力优化）+ 二级功能页；菜单名称不带 M/SA 编号；当前基线为二级菜单，不超过五级深度。  
- **数据流程**：营销系统档案与协议源数据 → M06 增量提取 → M05/M07 OCR 解析 → M08 规则校核 → M09 问题建档 → M10 工单治理。  
- **表单流程**：模板/规则/策略采用「草稿-发布」；任务生成、核查执行、治理派单需二次确认与幂等防重。  
- **异常流程**：提取/解析/校核/任务失败可重试；低置信度转 M14 复核；工单逾期升级；告警抑制避免风暴。

### 2.3 评审准入标准

1. 信息架构稳定（与《02-产品设计文档》一级/二级菜单一致）。

3. 接口清单或 Mock 契约可对齐（见第七章与 `protocolFullFlowRoutes.js`）。
4. 物理模型和标准编码已成稿（见第五、六章及数据库设计文档）。

### 2.5 边界、依赖与约束（强制）

1. 外部系统依赖：营销系统（档案与协议源数据）、OCR 识别服务、规则引擎服务、消息通知服务。
2. 数据一致性约束：同一协议文件在“解析 -> 校核 -> 问题治理”链路中必须使用统一文件唯一标识。
3. 性能约束：批量任务需支持并发执行并保证可恢复；长任务必须有进度可见与超时告警。
4. 合规约束：涉及客户敏感字段展示需脱敏；操作日志需满足审计追踪。
5. 交付约束：任何功能模块上线前，必须具备最小可验收脚本和异常兜底说明。

### 2.6 菜单架构约束（强制）

1. 菜单展示名称禁止带模块编号或需求编号（如 M01、SA-1），仅允许业务语义名称。
2. 菜单层级最大深度不超过五级（L1~L5）；超过上限必须在 IA 评审阶段完成降级设计。
3. 当前项目基线采用“一级菜单分组 + 二级菜单功能”结构，二级菜单为页面路由承载层。
4. 模块编号仅用于需求追踪矩阵、接口契约与测试用例，不作为 UI 展示信息。

### 3.1 协议模板库管理（M01）

#### （1）功能需求

- 支持预存交费、分次结算、高压预购电、低压后付费、高压后付费五类协议模板的创建、复制、启停与版本管理。  
- 模板字段需支持必填/非必填、字段类型、字段长度、校验规则、字段映射关系配置。  
- 模板需与营销档案基准库建立字段级映射，并支持映射变更留痕与版本追溯。  
- 【V2.2调整】支持模板管理列表页按协议类型、模板编码、状态、版本、更新时间筛选，支持分页、排序、导出与详情抽屉展示。  
- 【V2.2调整】模板管理必须具备增删改查闭环：新增模板、编辑草稿并发布、删除未引用草稿、停用已引用模板（替代物理删除）。

#### （2）功能流程

模板列表查询 -> 模板创建 -> 字段配置 -> 草稿保存 -> 发布生效 -> 版本追溯/停用。

#### （3）功能设计

支持模板列表、模板详情抽屉、模板复制、字段校验、版本展示、发布确认与停用确认。

#### （4）数据存储设计

`protocol_template`、`protocol_baseline`。

#### （5）注意事项

标准字段不可随意删除，变更需留痕。
已被规则或任务引用的模板禁止物理删除，仅允许停用。

#### （6）输入/输出与异常

- **输入**：营销档案基准数据；模板筛选条件（协议类型、编码、状态、版本）；字段/校验/映射配置；操作人身份与权限。  
- **输出**：已发布模板及版本号；`protocol_baseline` 映射关系；审计日志。  
- **异常**：映射冲突阻断保存；发布失败回滚上一版本；被引用模板禁止物理删除，仅可停用。  
- **验收**：Given 已选择五类协议之一并完成字段配置，When 执行发布，Then 新版本可追溯且可被 M03/M04 引用。

### 3.2 协议样本标注管理（M02）

#### （1）功能需求

- 支持从营销系统批量导入五类协议非结构化样本，并记录来源、导入批次、文件唯一标识。  
- 支持样本预处理（分类、去重、无效剔除、质量核查）并输出可标注样本池。  
- 支持文本框选标注、字段标注、签章标注、标注质检与返工重标闭环。  
- 【V2.2调整】支持样本列表按协议类型、标注状态、导入批次、质检结果、时间范围查询，支持分页、排序、批量操作和导出。  
- 【V2.2调整】样本管理具备增删改查闭环：样本导入/补录、标注及重标修订、删除未引用样本、失效已引用样本并留痕。

#### （2）功能流程

样本列表查询 -> 样本导入 -> 去重清洗 -> 文本/签章标注 -> 质检 -> 入库/重标 -> 失效管理。

#### （3）功能设计

标注状态可追踪，可回退重标；支持样本详情页展示原文件、标注明细、质检意见与操作日志。

#### （4）数据存储设计

`protocol_sample`、`sample_annotation`。

#### （5）注意事项

无效样本需剔除并记录原因。
已被训练或校核流程引用的样本不得物理删除，需执行失效并保留审计记录。

#### （6）输入/输出与异常

- **输入**：营销系统批量/单条协议文件；导入批次与来源登记；标注与质检操作指令。  
- **输出**：可标注样本池；`sample_annotation` 标注明细；质检结论与状态流转记录。  
- **异常**：无效样本剔除并记录原因；标注失败支持重标；被引用样本仅可失效不可物理删除。  
- **验收**：Given 样本已通过预处理，When 完成标注并通过质检，Then 标注结果可查询且状态可追溯。

### 3.3 核查规则管理（M03）

#### （1）功能需求

- 支持按模板类型维护完整性、一致性、合规性规则，并为每条规则分配唯一编号。  
- 支持规则表达式、核对方式、触发条件、生效范围、优先级与启停状态配置。  
- 支持规则版本控制、发布审批、历史版本对比与一键回滚。  
- 【V2.2调整】支持规则列表按模板类型、规则编号、规则类型、状态、版本查询，支持分页、排序与导出。  
- 【V2.2调整】规则管理具备增删改查闭环：新增草稿、编辑发布新版本、删除未发布规则、停用已发布规则。

#### （2）功能流程

规则列表查询 -> 新增/复制规则 -> 语法及冲突校验 -> 发布新版本 -> 场景绑定 -> 执行日志记录。

#### （3）功能设计

支持规则详情、场景绑定规则集、版本对比与回滚、停用确认、命中明细与执行日志查看。

#### （4）数据存储设计

`protocol_rule`、`rule_version`、`rule_execute_log`。

#### （5）注意事项

规则发布必须生成版本号。
已发布或被场景引用规则禁止物理删除，仅允许停用并记录原因。

#### （6）输入/输出与异常

- **输入**：规则草稿/发布请求；场景绑定参数；语法与冲突校验上下文。  
- **输出**：已发布 `rule_version`；场景规则集；`rule_execute_log` 执行留痕。  
- **异常**：冲突校验失败禁止发布；支持回滚至稳定版本；已发布规则仅可停用。  
- **验收**：Given 规则已绑定核查场景，When 触发校核任务，Then 命中规则与版本号可在日志中定位。

### 3.4 核查任务策略管理（M04）

#### （1）功能需求

- 支持按供电单位、协议类型、核查场景、时间窗口配置任务策略。  
- 支持自动执行与手工执行两种模式，支持单次/周期性任务生成。  
- 支持任务主单与任务明细联动维护，并实时展示生成数量、执行进度、成功率。  
- 【V2.2调整】支持“策略列表+任务列表”双列表展示：策略侧管理配置，任务侧跟踪执行，均支持筛选、分页、排序与导出。  
- 【V2.2调整】任务策略管理具备增删改查闭环：新增策略与任务发起、编辑策略参数、删除未生效策略、停用已生效策略。

#### （2）功能流程

策略列表查询 -> 策略新增/编辑 -> 批量任务生成或手工发起 -> 执行监控 -> 异常处置（暂停/恢复/重试） -> 指标统计。

#### （3）功能设计

实时展示任务数量、进度、识别成功率、失败数量与平均耗时；支持任务详情查看执行日志与异常原因。

#### （4）数据存储设计

`task_strategy`、`protocol_task`、`task_metric`。

#### （5）注意事项

任务失败需支持重试与告警。
已生效策略停用后不再自动生成新任务，但历史任务必须可持续追溯。

#### （6）输入/输出与异常

- **输入**：`task_strategy` 配置；供电单位/场景/时间窗口；手工或定时触发参数。  
- **输出**：`protocol_task` 主单与明细；`task_metric` 进度与成功率指标。  
- **异常**：任务生成失败可重试；重复提交防重（409）；超时告警联动 M11。  
- **验收**：Given 策略已启用，When 批量生成任务，Then 任务列表展示进度且失败可重试。

### 3.5 协议识别核查执行（M05）

#### （1）功能需求

- 针对存量协议按资源分配分批执行核查计划，监控执行状态并处理异常，确保任务按计划完成。  
- 支持与任务策略（M04）衔接：分批调度、执行进度、完成率与失败明细可查询；支持暂停、恢复、重试等运维动作并留痕。  
- 【V2.3对齐需求】计划执行可监控；异常须可定位、可恢复（参见需求说明书 5.2.1、5.5）。

#### （2）功能流程

核查计划/批次就绪 -> 分批调度执行 OCR 与核查子流程 -> 执行监控与指标采集 -> 异常处置（重试/补偿/人工介入）-> 批次归档。  

#### （3）功能设计

执行与监控视图：批次队列、运行中/已完成/失败列表、异常原因、操作日志与计划完成率。  

#### （4）数据存储设计

`protocol_check_task`、`check_execution_log`（批次与执行明细可按详细设计拆表）。  

#### （5）注意事项

任务中断须支持断点续跑或可重放；超时须触发告警并与 M11 监控联动。  

#### （6）输入/输出与异常

- **输入**：核查计划/批次配置；资源分配与调度指令；M04 任务上下文。  
- **输出**：批次执行状态；完成率与失败明细；`check_execution_log`（或计划表扩展字段）。  
- **异常**：批次中断断点续跑；失败文件入异常队列；暂停/恢复/重试须留痕。  
- **验收**：Given 存量协议已纳入计划，When 执行分批核查，Then 计划完成率可查询且异常可定位恢复。

### 3.6 协议信息提取引擎（M06）

#### （1）功能需求

- 配置营销系统新增业扩用户、协议维护变动用户的信息提取规则，形成增量核查对象。  
- 支持提取规则维护、增量对象生成、提取日志与统计；与任务生成及核查执行链路衔接。  
- 【V2.3对齐需求】源数据缺失、字段不一致、接口失败时须兜底（标记、补录清单、重试），外部补录须标记来源与可信级别（需求 5.5）。

#### （2）功能流程

维护提取规则 -> 获取营销增量/变更数据 -> 规则筛选生成增量对象 -> 记录提取日志与统计 -> 输出至核查任务输入。  

#### （3）功能设计

提取规则配置页、增量对象列表、提取日志检索、失败补录与重试入口。  

#### （4）数据存储设计

`extract_object`、`extract_rule`、`extract_log`。  

#### （5）注意事项

提取失败不得静默丢弃；须与档案基准口径一致，避免重复生成核查对象。  

#### （6）输入/输出与异常

- **输入**：营销增量/变更用户数据；`extract_rule` 配置；接口拉取触发条件。  
- **输出**：`extract_object` 增量核查对象；`extract_log` 提取日志与统计。  
- **异常**：源数据缺字段标红并进入补齐清单；接口失败可重试；外部补录须标记来源与可信级别。  
- **验收**：Given 提取规则已配置，When 执行增量提取，Then 对象列表与日志可追溯到源数据。

### 3.7 协议信息解析（M07）

#### （1）功能需求

- 支持非结构化协议文本识别为可编辑文本，并按字段规则抽取结构化数据。  
- 支持复杂表格识别，包含表头/表尾/单元格文本及行列位置信息提取。  
- 支持签字签章区域识别、签章类型标注与置信度输出。

#### （2）功能流程

文件入库 -> OCR 识别 -> 版面解析 -> 字段抽取 -> 结果结构化 -> 质量标记。

#### （3）功能设计

支持解析置信度输出、疑难件标记、失败重试与人工复核入口。

#### （4）数据存储设计

`protocol_parse_result`、`parse_confidence_log`。

#### （5）注意事项

低置信度结果必须进入人工复核队列，不得直接进入自动校核。

#### （6）输入/输出与异常

- **输入**：协议文件 `file_id`；OCR/版面解析任务；字段抽取规则。  
- **输出**：`protocol_parse_result` 结构化字段；`parse_confidence_log` 置信度。  
- **异常**：低置信度转 M14 复核；解析失败保留错误码与原文快照；支持失败重试。  
- **验收**：Given 协议文件已入库，When 执行解析，Then 文字/表格/签章三类信息结构化落库。

### 3.8 识别结果校核（M08）

#### （1）功能需求

- 支持完整性校核（必填项、必要条款、模板匹配）与准确性校核（档案一致性）双引擎执行。  
- 支持按协议类型动态选择校核规则集，并输出字段级命中结果和异常原因。  
- 支持校核结论结构化落库，为问题治理和统计分析提供标准输入。

#### （2）功能流程

加载解析结果 -> 应用规则集 -> 与档案基准比对 -> 生成校核结论 -> 异常项入库。

#### （3）功能设计

支持按协议类型切换校核策略；支持规则命中明细与异常原因展示。

#### （4）数据存储设计

`protocol_verify_result`、`verify_item_detail`。

#### （5）注意事项

校核引擎异常时需输出可重试状态，避免任务静默失败。

#### （6）输入/输出与异常

- **输入**：`protocol_parse_result`；M03 规则集；`protocol_baseline` 档案基准。  
- **输出**：`protocol_verify_result` 校核结论；`verify_item_detail` 字段级差异。  
- **异常**：档案缺值输出「无法判定」；规则引擎异常进入重试队列；同一 `file_id` 全链路一致。  
- **验收**：Given 解析结果已生成，When 执行双引擎校核，Then 字段级结论可解释且可关联规则编号。

### 3.9 识别问题结果记录（M09）

#### （1）功能需求

- 支持将校核异常统一建档，关联协议文件、任务批次、规则编号、责任单位。  
- 支持问题类型、问题等级、问题来源、影响范围等标签化管理。  
- 支持问题生命周期状态流转（新建、确认、处理中、待复核、关闭）。

#### （2）功能流程

异常项接收 -> 去重聚合 -> 问题建档 -> 状态初始化 -> 流转治理。

#### （3）功能设计

支持问题等级、问题来源、关联规则、关联任务、关联文件的多维度标注。

#### （4）数据存储设计

`protocol_problem`、`problem_relation`。

#### （5）注意事项

重复问题归并规则必须可配置，避免问题台账膨胀。

#### （6）输入/输出与异常

- **输入**：M08 校核异常项；任务批次、规则编号、责任单位标签。  
- **输出**：`protocol_problem` 问题台账；`problem_relation` 多实体关联。  
- **异常**：重复问题归并冲突需人工确认；误报支持撤销留痕。  
- **验收**：Given 校核产生异常项，When 完成建档与归并，Then 问题状态可流转且关联任务/文件可查。

### 3.10 问题分析及治理（M10）

#### （1）功能需求

- 支持按问题类型与责任归属自动分派治理工单，支持人工改派与批量派发。  
- 支持工单处理过程留痕（处理意见、附件、处理时长、处理结果）并可追溯。  
- 支持治理闭环验收与复盘，形成可复用治理策略。

#### （2）功能流程

问题分类 -> 工单生成 -> 责任分派 -> 处理反馈 -> 验收关闭 -> 复盘沉淀。

#### （3）功能设计

支持逾期预警、升级流转、批量处理与治理报表输出。

#### （4）数据存储设计

`governance_ticket`、`ticket_process_log`、`governance_summary`。

#### （5）注意事项

工单状态变更必须记录操作者与时间戳，满足审计要求。

#### （6）输入/输出与异常

- **输入**：M09 问题记录；分派规则与责任单位；处理意见与附件。  
- **输出**：`governance_ticket` 工单；`ticket_process_log` 流转日志；`governance_summary` 汇总。  
- **异常**：逾期自动升级；派单失败重发并告警；状态变更须记录操作者与时间戳。  
- **验收**：Given 问题已确认，When 自动生成并处理工单，Then 工单可闭环且治理结果可追溯。

### 3.11 核查任务监控（M11）

#### （1）功能需求

- 支持任务执行效率评估，包括总耗时、平均耗时、并发处理能力、失败率。  
- 支持异常事件监控与告警（OCR失败、规则引擎异常、网络超时等）并记录严重级别。  
- 支持按任务 ID、时间、错误类型多维检索日志并输出任务监控报表。

#### （2）功能流程

采集任务指标 -> 异常检测 -> 告警触发 -> 人员处理 -> 结果复核 -> 报表沉淀。

#### （3）功能设计

支持按任务 ID、错误类型、时间范围快速检索并查看详情。

#### （4）数据存储设计

`task_monitor_log`、`task_alert_record`、`task_monitor_report`。

#### （5）注意事项

告警策略需支持阈值配置与抑制规则，避免告警风暴。

#### （6）输入/输出与异常

- **输入**：任务执行指标流；异常事件（OCR 失败、规则引擎异常、超时等）。  
- **输出**：`task_monitor_log`；`task_alert_record`；`task_monitor_report`。  
- **异常**：告警抑制避免风暴；任务异常支持标记与备注；业务侧日志脱敏展示。  
- **验收**：Given 任务执行中，When 触发阈值告警，Then 可按任务 ID/错误类型检索并处理。

### 3.12 核查结果监控（M12）

#### （1）功能需求

- 支持问题汇总看板：按问题类型、协议类型、单位、状态进行统计分析。  
- 支持识别质效看板：展示识别总量、成功率、准确率、关键字段错误率、处理进度。  
- 支持治理进度看板与固定周期报表，量化治理成效与整改质量。

#### （2）功能流程

汇总统计 -> 指标计算 -> 看板刷新 -> 固定报表生成 -> 周期推送。

#### （3）功能设计

支持按单位、协议类型、问题类型、时间周期等维度钻取分析。

#### （4）数据存储设计

`result_monitor_stat`、`result_dashboard_snapshot`。

#### （5）注意事项

指标定义必须统一口径，口径调整必须同步版本记录。

#### （6）输入/输出与异常

- **输入**：M09/M10 问题与治理结果；M11 任务指标；统计维度（单位、协议类型、周期）。  
- **输出**：`result_monitor_stat`；`result_dashboard_snapshot`；固定周期报表。  
- **异常**：统计失败可回补；口径变更须版本记录；单位角色仅看本单位数据。  
- **验收**：Given 统计周期到达，When 刷新看板，Then 问题汇总/质效/治理三类指标口径一致可钻取。

### 3.13 协议识别能力优化（M13）

#### （1）功能需求

- 支持按协议类型沉淀错误样本并形成优化任务，驱动识别能力持续迭代。  
- 支持规则、样本、模型三类优化方案协同发布，并保持版本可追溯。  
- 支持优化结果量化评估（准确率提升、误报率下降、处理效率提升）。

#### （2）功能流程

样本分析 -> 规则/模型优化 -> 批量验证 -> 效果对比 -> 发布/回滚。

#### （3）功能设计

支持优化版本管理、前后对比、回滚策略与效果追踪。

#### （4）数据存储设计

`optimize_iteration`、`optimize_effect_report`。

#### （5）注意事项

优化发布需通过回归验证，关键指标劣化时自动触发回滚。

#### （6）输入/输出与异常

- **输入**：错误样本与优化方案（规则/样本/模型）；回归验证批次。  
- **输出**：`optimize_iteration`；`optimize_effect_report` 效果对比。  
- **异常**：关键指标劣化自动回滚；失败迭代保留证据链。  
- **验收**：Given 完成优化迭代，When 通过回归验证并发布，Then 准确率/误报率改善可量化证明。

### 3.14 协同复核（M14）

#### （1）功能需求

- 支持业务侧提交识别问题反馈，自动进入复核任务池并分派处理人。  
- 支持人工复核纠偏关键字段（签名、数值、日期、地址等），并回写校核结果。  
- 支持沉淀典型错误样本与易错字库，反哺规则与识别能力优化。

#### （2）功能流程

问题反馈 -> 任务分派 -> 人工复核 -> 纠偏确认 -> 样本入库 -> 规则回写。

#### （3）功能设计

支持高频错误类型标注、易错字库维护、复核 SLA 统计。

#### （4）数据存储设计

`review_feedback`、`manual_review_task`、`error_corpus`。

#### （5）注意事项

复核结论必须回写至问题台账，确保端到端闭环。

#### （6）输入/输出与异常

- **输入**：业务侧问题反馈；待复核字段与原文对照；M09 问题关联。  
- **输出**：`review_feedback`；`manual_review_task` 结论；`error_corpus` 易错字库；回写 M08/M09。  
- **异常**：反馈积压超阈值告警；复核冲突升级裁决。  
- **验收**：Given 业务提交识别问题，When 复核员完成纠偏，Then 结论回写问题台账且样本可反哺 M13。

## 四、其他非功能需求

与《01-需求说明书》第 3 节（NFR-1~NFR-5）一致，评审口径如下：

1. **性能要求**：批量任务生成与核查执行支持并发；单任务列表查询 P95 < 2s（演示环境基线）；长任务须展示进度条与预计完成时间。
2. **可用性要求**：关键写接口成功率 ≥ 99%（生产目标）；失败可重试且保留 `requestId`/`trace_id` 可追溯。
3. **兼容性要求**：Chrome/Edge 最新两个大版本；1366×768 与 1920×1080 布局可用；子应用支持 Hash 独立运行与主站 `/power-fee-protocol-check` 嵌入。
4. **安全要求**：客户敏感字段脱敏展示；模板/规则/工单状态变更全量审计；接口鉴权与角色边界见 2.4 第 7 项（待联调）。
5. **可维护性要求**：需求-模块-接口-数据表四向可追溯；文档变更须同步 04 映射表与 DB 设计文档。
6. **可测试性要求**：每模块至少 1 条 Given/When/Then（见第三章（6）节）；7.3 DoD 清单作为测试准入基线。
