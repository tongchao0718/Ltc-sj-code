## 一、项目概述

### 1.1 项目简述

建设协议核查全流程平台，覆盖模板配置、样本标注、规则管理、任务策略、识别核查、信息提取、信息解析、结果校核、问题治理、任务监控、结果监控、能力优化、协同复核等 14 个模块。

### 1.2 项目目标

1. 建立统一核查标准和规则版本体系。  
2. 实现任务自动生成与异常闭环治理。  
3. 建立识别能力持续优化机制。  

### 1.3 交付范围

- 前端范围：14 个业务模块页面、路由、交互、监控看板与问题闭环操作。
- 文档范围：需求、PRD、SDD、详细设计、数据库、门禁与审计。
- 非本期范围：跨系统结算核心改造。

## 二、系统流程

### 2.1 业务流程

协议模板配置 -> 样本标注 -> 规则发布 -> 任务生成 -> 识别核查 -> 异常治理 -> 优化迭代。

### 2.2 操作流程（评审关注）

- 数据流程：营销系统提取 -> OCR解析 -> 规则校核 -> 异常落库。  
- 治理流程：异常分类 -> 工单推送 -> 处置跟踪 -> 闭环归档。  

### 2.3 评审准入标准

1. 模块编号与需求编号映射完整。  
2. 映射表与接口契约可追溯。  
3. 物理模型与标准编码已成稿。  

### 2.4 评审会前检查清单

| 序号 | 检查项 | 责任角色 | 是否完成（Y/N） | 备注 |
|------|------|------|------|------|
| 1 | 文档版本、评审轮次已更新 | 产品 | N | 待评审 |
| 2 | 模块编号与需求编号一致 | 产品/研发 | N | 待评审 |
| 3 | 页面-路由-接口-数据表映射完整 | 研发 | N | 待评审 |
| 4 | 核心异常路径已定义 | 产品/测试 | N | 待评审 |
| 5 | 菜单命名与层级规范一致（不带编号、最多五级） | 产品/研发/测试 | N | 待评审 |

### 2.5 边界、依赖与约束（强制）

1. 外部系统依赖：营销系统（档案与协议源数据）、OCR 识别服务、规则引擎服务、消息通知服务。  
2. 数据一致性约束：同一协议文件在“解析 -> 校核 -> 问题治理”链路中必须使用统一文件唯一标识。  
3. 性能约束：批量任务需支持并发执行并保证可恢复；长任务必须有进度可见与超时告警。  
4. 合规约束：涉及客户敏感字段展示需脱敏；操作日志需满足审计追踪。  
5. 交付约束：任何功能模块上线前，必须具备最小可验收脚本和异常兜底说明。  

### 2.6 菜单架构约束（强制）

1. 菜单展示名称禁止带模块编号或需求编号（如 M01、SA-1），仅允许业务语义名称。  
2. 菜单层级最大深度不超过五级（L1~L5）；超过上限必须在 IA 评审阶段完成降级设计。  
3. 当前项目基线采用“一级菜单分组 + 二级菜单功能”结构，二级菜单为页面路由承载层。  
4. 模块编号仅用于需求追踪矩阵、接口契约与测试用例，不作为 UI 展示信息。  

## 三、功能说明

### 3.1 协议模板库管理（M01）

#### （1）功能需求
- 支持预存交费、分次结算、高压预购电、低压后付费、高压后付费五类协议模板的创建、复制、启停与版本管理。  
- 模板字段需支持必填/非必填、字段类型、字段长度、校验规则、字段映射关系配置。  
- 模板需与营销档案基准库建立字段级映射，并支持映射变更留痕与版本追溯。  
- 【V2.2调整】支持模板管理列表页按协议类型、模板编码、状态、版本、更新时间筛选，支持分页、排序、导出与详情抽屉展示。  
- 【V2.2调整】模板管理必须具备增删改查闭环：新增模板、编辑草稿并发布、删除未引用草稿、停用已引用模板（替代物理删除）。

#### （2）功能流程
模板列表查询 -> 模板创建 -> 字段配置 -> 草稿保存 -> 发布生效 -> 版本追溯/停用。

#### （3）功能设计
支持模板列表、模板详情抽屉、模板复制、字段校验、版本展示、发布确认与停用确认。

#### （4）数据存储设计
`protocol_template`、`protocol_baseline`。

#### （5）注意事项
标准字段不可随意删除，变更需留痕。
已被规则或任务引用的模板禁止物理删除，仅允许停用。

### 3.2 协议样本标注管理（M02）

#### （1）功能需求
- 支持从营销系统批量导入五类协议非结构化样本，并记录来源、导入批次、文件唯一标识。  
- 支持样本预处理（分类、去重、无效剔除、质量核查）并输出可标注样本池。  
- 支持文本框选标注、字段标注、签章标注、标注质检与返工重标闭环。  
- 【V2.2调整】支持样本列表按协议类型、标注状态、导入批次、质检结果、时间范围查询，支持分页、排序、批量操作和导出。  
- 【V2.2调整】样本管理具备增删改查闭环：样本导入/补录、标注及重标修订、删除未引用样本、失效已引用样本并留痕。

#### （2）功能流程
样本列表查询 -> 样本导入 -> 去重清洗 -> 文本/签章标注 -> 质检 -> 入库/重标 -> 失效管理。

#### （3）功能设计
标注状态可追踪，可回退重标；支持样本详情页展示原文件、标注明细、质检意见与操作日志。

#### （4）数据存储设计
`protocol_sample`、`sample_annotation`。

#### （5）注意事项
无效样本需剔除并记录原因。
已被训练或校核流程引用的样本不得物理删除，需执行失效并保留审计记录。

### 3.3 核查规则管理（M03）

#### （1）功能需求
- 支持按模板类型维护完整性、一致性、合规性规则，并为每条规则分配唯一编号。  
- 支持规则表达式、核对方式、触发条件、生效范围、优先级与启停状态配置。  
- 支持规则版本控制、发布审批、历史版本对比与一键回滚。  
- 【V2.2调整】支持规则列表按模板类型、规则编号、规则类型、状态、版本查询，支持分页、排序与导出。  
- 【V2.2调整】规则管理具备增删改查闭环：新增草稿、编辑发布新版本、删除未发布规则、停用已发布规则。

#### （2）功能流程
规则列表查询 -> 新增/复制规则 -> 语法及冲突校验 -> 发布新版本 -> 场景绑定 -> 执行日志记录。

#### （3）功能设计
支持规则详情、场景绑定规则集、版本对比与回滚、停用确认、命中明细与执行日志查看。

#### （4）数据存储设计
`protocol_rule`、`rule_version`、`rule_execute_log`。

#### （5）注意事项
规则发布必须生成版本号。
已发布或被场景引用规则禁止物理删除，仅允许停用并记录原因。

### 3.4 核查任务策略管理（M04）

#### （1）功能需求
- 支持按供电单位、协议类型、核查场景、时间窗口配置任务策略。  
- 支持自动执行与手工执行两种模式，支持单次/周期性任务生成。  
- 支持任务主单与任务明细联动维护，并实时展示生成数量、执行进度、成功率。  
- 【V2.2调整】支持“策略列表+任务列表”双列表展示：策略侧管理配置，任务侧跟踪执行，均支持筛选、分页、排序与导出。  
- 【V2.2调整】任务策略管理具备增删改查闭环：新增策略与任务发起、编辑策略参数、删除未生效策略、停用已生效策略。

#### （2）功能流程
策略列表查询 -> 策略新增/编辑 -> 批量任务生成或手工发起 -> 执行监控 -> 异常处置（暂停/恢复/重试） -> 指标统计。

#### （3）功能设计
实时展示任务数量、进度、识别成功率、失败数量与平均耗时；支持任务详情查看执行日志与异常原因。

#### （4）数据存储设计
`task_strategy`、`protocol_task`、`task_metric`。

#### （5）注意事项
任务失败需支持重试与告警。
已生效策略停用后不再自动生成新任务，但历史任务必须可持续追溯。

### 3.5 核查异常分析治理（M05）

#### （1）功能需求
- 支持对识别与校核异常进行自动分类、等级判定、归属单位识别。  
- 支持自动生成治理工单并推送责任单位，记录受理、处理、复核、关闭全过程。  
- 支持问题治理进度跟踪、逾期预警、治理结果分析与统计报表输出。  

#### （2）功能流程
异常识别 -> 分类归集 -> 工单派发 -> 闭环确认。

#### （3）功能设计
提供多维看板：问题汇总、识别质效、治理进度。

#### （4）数据存储设计
`protocol_issue`、`issue_ticket`、`governance_report`。

#### （5）注意事项
高等级异常需升级处理。

### 3.6 识别能力优化（M06）

#### （1）功能需求
- 支持基于识别监控结果对高频错误字段与低准确率协议类型进行专项优化。  
- 支持优化方案版本化管理（规则优化、样本扩充、模型参数调整）并留痕。  
- 支持优化前后效果对比验证，不达标自动触发回滚。  

#### （2）功能流程
错误反馈 -> 人机复核 -> 样本扩充 -> 模型调优 -> 效果验证。

#### （3）功能设计
支持错误案例沉淀与易错字库维护。

#### （4）数据存储设计
`optimize_feedback`、`optimize_iteration`。

#### （5）注意事项
优化迭代需关联规则和样本版本。

### 3.7 协议信息解析（M07）

#### （1）功能需求
- 支持非结构化协议文本识别为可编辑文本，并按字段规则抽取结构化数据。  
- 支持复杂表格识别，包含表头/表尾/单元格文本及行列位置信息提取。  
- 支持签字签章区域识别、签章类型标注与置信度输出。  

#### （2）功能流程
文件入库 -> OCR 识别 -> 版面解析 -> 字段抽取 -> 结果结构化 -> 质量标记。

#### （3）功能设计
支持解析置信度输出、疑难件标记、失败重试与人工复核入口。

#### （4）数据存储设计
`protocol_parse_result`、`parse_confidence_log`。

#### （5）注意事项
低置信度结果必须进入人工复核队列，不得直接进入自动校核。

### 3.8 识别结果校核（M08）

#### （1）功能需求
- 支持完整性校核（必填项、必要条款、模板匹配）与准确性校核（档案一致性）双引擎执行。  
- 支持按协议类型动态选择校核规则集，并输出字段级命中结果和异常原因。  
- 支持校核结论结构化落库，为问题治理和统计分析提供标准输入。  

#### （2）功能流程
加载解析结果 -> 应用规则集 -> 与档案基准比对 -> 生成校核结论 -> 异常项入库。

#### （3）功能设计
支持按协议类型切换校核策略；支持规则命中明细与异常原因展示。

#### （4）数据存储设计
`protocol_verify_result`、`verify_item_detail`。

#### （5）注意事项
校核引擎异常时需输出可重试状态，避免任务静默失败。

### 3.9 识别问题结果记录（M09）

#### （1）功能需求
- 支持将校核异常统一建档，关联协议文件、任务批次、规则编号、责任单位。  
- 支持问题类型、问题等级、问题来源、影响范围等标签化管理。  
- 支持问题生命周期状态流转（新建、确认、处理中、待复核、关闭）。  

#### （2）功能流程
异常项接收 -> 去重聚合 -> 问题建档 -> 状态初始化 -> 流转治理。

#### （3）功能设计
支持问题等级、问题来源、关联规则、关联任务、关联文件的多维度标注。

#### （4）数据存储设计
`protocol_problem`、`problem_relation`。

#### （5）注意事项
重复问题归并规则必须可配置，避免问题台账膨胀。

### 3.10 问题分析及治理（M10）

#### （1）功能需求
- 支持按问题类型与责任归属自动分派治理工单，支持人工改派与批量派发。  
- 支持工单处理过程留痕（处理意见、附件、处理时长、处理结果）并可追溯。  
- 支持治理闭环验收与复盘，形成可复用治理策略。  

#### （2）功能流程
问题分类 -> 工单生成 -> 责任分派 -> 处理反馈 -> 验收关闭 -> 复盘沉淀。

#### （3）功能设计
支持逾期预警、升级流转、批量处理与治理报表输出。

#### （4）数据存储设计
`governance_ticket`、`ticket_process_log`、`governance_summary`。

#### （5）注意事项
工单状态变更必须记录操作者与时间戳，满足审计要求。

### 3.11 核查任务监控（M11）

#### （1）功能需求
- 支持任务执行效率评估，包括总耗时、平均耗时、并发处理能力、失败率。  
- 支持异常事件监控与告警（OCR失败、规则引擎异常、网络超时等）并记录严重级别。  
- 支持按任务 ID、时间、错误类型多维检索日志并输出任务监控报表。  

#### （2）功能流程
采集任务指标 -> 异常检测 -> 告警触发 -> 人员处理 -> 结果复核 -> 报表沉淀。

#### （3）功能设计
支持按任务 ID、错误类型、时间范围快速检索并查看详情。

#### （4）数据存储设计
`task_monitor_log`、`task_alert_record`、`task_monitor_report`。

#### （5）注意事项
告警策略需支持阈值配置与抑制规则，避免告警风暴。

### 3.12 核查结果监控（M12）

#### （1）功能需求
- 支持问题汇总看板：按问题类型、协议类型、单位、状态进行统计分析。  
- 支持识别质效看板：展示识别总量、成功率、准确率、关键字段错误率、处理进度。  
- 支持治理进度看板与固定周期报表，量化治理成效与整改质量。  

#### （2）功能流程
汇总统计 -> 指标计算 -> 看板刷新 -> 固定报表生成 -> 周期推送。

#### （3）功能设计
支持按单位、协议类型、问题类型、时间周期等维度钻取分析。

#### （4）数据存储设计
`result_monitor_stat`、`result_dashboard_snapshot`。

#### （5）注意事项
指标定义必须统一口径，口径调整必须同步版本记录。

### 3.13 协议识别能力优化（M13）

#### （1）功能需求
- 支持按协议类型沉淀错误样本并形成优化任务，驱动识别能力持续迭代。  
- 支持规则、样本、模型三类优化方案协同发布，并保持版本可追溯。  
- 支持优化结果量化评估（准确率提升、误报率下降、处理效率提升）。  

#### （2）功能流程
样本分析 -> 规则/模型优化 -> 批量验证 -> 效果对比 -> 发布/回滚。

#### （3）功能设计
支持优化版本管理、前后对比、回滚策略与效果追踪。

#### （4）数据存储设计
`optimize_iteration`、`optimize_effect_report`。

#### （5）注意事项
优化发布需通过回归验证，关键指标劣化时自动触发回滚。

### 3.14 协同复核（M14）

#### （1）功能需求
- 支持业务侧提交识别问题反馈，自动进入复核任务池并分派处理人。  
- 支持人工复核纠偏关键字段（签名、数值、日期、地址等），并回写校核结果。  
- 支持沉淀典型错误样本与易错字库，反哺规则与识别能力优化。  

#### （2）功能流程
问题反馈 -> 任务分派 -> 人工复核 -> 纠偏确认 -> 样本入库 -> 规则回写。

#### （3）功能设计
支持高频错误类型标注、易错字库维护、复核 SLA 统计。

#### （4）数据存储设计
`review_feedback`、`manual_review_task`、`error_corpus`。

#### （5）注意事项
复核结论必须回写至问题台账，确保端到端闭环。

### 3.15 评审防遗漏清单（强制）

1. 每个模块必须具备“输入、处理、输出、异常、恢复”五类定义。  
2. 每个模块必须至少定义 1 条验收标准与 1 条失败处理策略。  
3. 模块说明、接口契约、数据模型三者必须可追溯（同一模块编号）。  
4. 未达到上述要求的模块，不得通过设计评审。
