## 2. 关键状态

- 任务状态：`pending/running/success/failed`
- 样本状态：`collected/preprocessed/annotated/invalid`
- 工单状态：`new/assigned/processing/resolved/closed`
- 解析状态：`queued/parsing/parsed/low_confidence/parse_failed`
- 校核状态：`waiting_verify/verifying/passed/rejected/verify_failed`
- 复核状态：`to_review/reviewing/corrected/rejected`

## 2.1 标准编码口径（与数据库文档一致）

| 属性名称 | 编码类型 | 枚举值 |
|------|------|------|
| 任务状态 | `TASK_STATUS` | `pending/running/success/failed` |
| 工单状态 | `ISSUE_STATUS` | `new/assigned/processing/resolved/closed` |
| 协议类型 | `PROTOCOL_TYPE` | `TYPE_A~TYPE_E` |
| 规则类型 | `RULE_TYPE` | `integrity/consistency/compliance` |

> 说明：字段级“表中文名/字段中文名/标准编码名（系统功能）”以 `02-数据库设计文档.md` 的“物理模型清单（全量）”为准。
> 说明：样本状态、解析状态、校核状态、复核状态当前未纳入“标准编码清单”统一管理，按模块状态机定义执行；如后续纳入标准编码，需同步更新本表与数据库设计文档。

## 3. 关键异常处理

- OCR识别失败：进入技术异常池并触发复核任务
- 规则执行失败：记录日志并告警
- 工单超时：升级到责任主管
- 解析低置信度：自动进入人工复核队列并标记疑难件
- 提取接口超时：按指数退避重试，重试失败转人工补录清单

## 4. 模块详细设计（开发实现）

### 4.1 M01-M04（配置与任务生成）

1. M01 模板管理：模板字段含必填、类型、长度、映射规则；模板发布产生版本快照。  
2. M02 样本标注：导入后执行去重与质量核查，标注后进入质检流转。  
3. M03 规则管理：规则按完整性/一致性/合规性分类，支持版本对比与回滚。  
4. M04 策略任务：按单位+场景+时间窗口生成任务主单与明细，支持防重与重试。  

### 4.2 M05-M09（识别校核与问题建档）

1. M05 核查执行：按批次调度存量任务，支持暂停/恢复与断点续跑。  
2. M06 提取引擎：提取新增与变动用户对象，形成增量核查对象集。  
3. M07 协议解析：执行文字、表格、签章解析，输出结构化数据与置信度。  
4. M08 结果校核：调用完整性与准确性校核，输出字段级差异与异常原因。  
5. M09 问题记录：异常项建档并关联任务、规则、文件，支持归并与分级。  

### 4.3 M10-M14（治理监控与持续优化）

1. M10 治理派单：自动分类问题并生成工单，支持升级流转与闭环验收。  
2. M11 任务监控：采集执行效率和错误日志，按阈值触发告警与抑制规则。  
3. M12 结果监控：生成问题看板、质效看板、治理看板与固定报表。  
4. M13 能力优化：对高频错误进行样本扩充和规则优化，输出前后对比报告。  
5. M14 协同复核：承接业务反馈，人工纠偏后回写结果并沉淀易错字库。

## 5. 接口与状态机实现约束

1. 所有写接口必须返回 `code/message/data/requestId`，并记录 `operator` 与时间戳。  
2. 列表接口统一 `pageNo/pageSize/total`，筛选变更后分页强制回到第一页。  
3. 页面状态机最小集合：`loading/success/empty/error`；异常场景必须提供恢复动作。

## 8. 非功能实现要求（开发阶段）

1. 幂等：任务生成、工单派发、复核提交接口必须具备幂等保障。  
2. 可观测：关键链路必须输出 `requestId`，并在日志中关联 `task_id/file_id`。  
3. 异常恢复：超时、断网、4xx、5xx 均需提供前后端协同恢复动作。  
4. 安全：敏感字段按展示脱敏规则输出，审计角色仅可只读查看。  

## 9. 关键页面时序（最小实现）

### 9.1 模板配置与发布页面（M01）

1. 页面初始化：加载模板列表、模板类型与字段规则字典。  
2. 用户操作：新增/编辑模板，配置字段与映射规则。  
3. 请求发送：提交模板保存与发布请求。  
4. 结果处理：保存成功后刷新列表；发布成功后生成版本快照。  
5. 后续动作：若发布失败，提示原因并保留草稿，支持继续编辑重试。  

### 9.2 任务策略与执行页面（M04/M05）

1. 页面初始化：加载策略配置、单位范围、场景配置与历史任务。  
2. 用户操作：配置策略参数并发起批量任务。  
3. 请求发送：调用任务生成与执行接口，创建任务主单与明细。  
4. 结果处理：展示任务进度、成功率、失败明细与重试入口。  
5. 后续动作：异常任务进入补偿流程，支持暂停/恢复与断点续跑。  

### 9.3 异常治理与复核页面（M10/M14）

1. 页面初始化：加载问题台账、工单状态与待复核任务。  
2. 用户操作：执行派单、改派、处理回执与人工纠偏。  
3. 请求发送：提交工单流转与复核结果回写请求。  
4. 结果处理：更新问题状态、工单状态与复核记录。  
5. 后续动作：生成治理汇总与复核沉淀，必要时触发升级告警。
