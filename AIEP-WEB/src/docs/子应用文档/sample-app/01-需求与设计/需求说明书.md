# 示例应用（sample-app）需求说明书

## 1. 范围

- 子应用编码：`sample-app`
- 主路由前缀：`/sample-app`
- 版本：`V1.0`

本需求说明书用于定义 sample-app 在 AI 全流程下的首版需求范围与验收标准。

## 2. 功能需求

| 编号 | 需求 | 验收 |
|------|------|------|
| SA-1 | 仪表盘工作台 | 页面可访问；统计卡片和图表渲染成功；空态可见 |
| SA-2 | 列表查询 | 支持筛选、分页、排序；切换筛选后分页重置为第一页 |
| SA-3 | 表单提交 | 支持必填/格式校验；失败保留输入；重复提交防重 |
| SA-4 | 结果与异常页 | 成功/失败/403/404/500 页面可访问且回退路径可用 |
| SA-5 | 独立打包 | `build:sample-app` 成功，产物可直接访问 |

## 3. 非功能需求

| 编号 | 需求 | 验收 |
|------|------|------|
| NFR-1 | 响应式布局 | 1366x768 与 1920x1080 布局可用 |
| NFR-2 | 可维护性 | 需求-模块-接口-数据表可追溯 |
| NFR-3 | 交互可观测 | 请求失败有可读提示，关键链路可追踪 |

## 4. 用户故事地图（概要）

| 用户活动 | 用户任务 | 用户故事 | 对应需求（编号/名称） | 发布切片 | 验收标准（摘要） |
|------|------|------|------|------|------|
| 查看运营状态 | 浏览核心指标 | 作为运营人员，我希望快速看到核心指标，以便判断业务状态 | SA-1 / 仪表盘工作台 | MVP | Given 用户进入工作台，When 页面加载完成，Then 核心指标卡片与图表渲染成功 |
| 检索数据 | 筛选和翻页 | 作为业务人员，我希望通过筛选和分页找到目标数据，以便高效处理 | SA-2 / 列表查询 | MVP | Given 用户设置筛选条件，When 执行查询并翻页，Then 结果按条件返回且分页重置规则正确 |
| 提交信息 | 填写并提交表单 | 作为操作人员，我希望提交时有明确校验和失败回显，以便降低错误率 | SA-3 / 表单提交 | MVP | Given 用户填写表单，When 点击提交，Then 校验通过时提交成功，失败时保留输入并给出错误提示 |
