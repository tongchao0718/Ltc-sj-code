# 示例应用操作说明

## 前置条件

- Node.js >= 18，npm >= 9。  
- 首次运行前执行：`npm install`。  
- 若端口被占用，请按终端提示更换端口并使用新地址访问。  

## 在主应用中使用

1. 启动：`npm run dev`
2. 打开应用中心 →「进入应用」，或访问 `/sample-app`
3. 使用左侧菜单切换各功能模块：
   - **仪表盘**：工作台、实时监控
   - **数据可视化**：数据分析、多维数据分析
   - **列表页**：查询表格、卡片列表
   - **表单页**：分步表单、分组表单
   - **详情页**：基础详情页
   - **结果页**：成功页、失败页
   - **异常页**：403、404、500 页面
   - **个人中心**：个人资料、用户设置

## 功能操作说明

### 工作台
- 查看数据统计卡片，了解系统概览
- 查看内容数据图表，了解数据趋势
- 查看线上热门内容，了解热门内容排行
- 查看内容类型占比，了解内容分布

### 数据可视化
- 数据分析：查看多种图表展示，包括折线图、柱状图等
- 多维数据分析：查看复杂数据可视化，支持多维度数据展示

### 列表页
- 查询表格：使用筛选条件查询数据，支持分页
- 卡片列表：使用标签切换不同类型的卡片，支持搜索

### 表单页
- 分步表单：按照步骤填写表单，支持表单验证
- 分组表单：填写分组形式的表单，支持复杂表单逻辑

### 个人中心
- 个人资料：查看和修改个人信息
- 用户设置：修改用户账号设置

## 独立运行（开发）

```bash
npm run dev:sample-app
```

浏览器访问开发服务器提示的地址，打开 `/build/sample-app.html`。

## 独立运行（构建后）

```bash
npm run build:sample-app
```

在资源管理器中打开 `dist/sample-app/index.html`（推荐使用 `npm run preview` 或静态服务器以避免浏览器本地策略限制）。

## 常见问题

- 页面白屏：优先检查依赖是否安装完整、控制台是否有资源加载错误。  
- 路由打不开：确认访问路径是否带 `/sample-app` 前缀。  
- 本地文件限制：直接双击 `index.html` 受浏览器策略影响时，使用 `npm run preview`。
