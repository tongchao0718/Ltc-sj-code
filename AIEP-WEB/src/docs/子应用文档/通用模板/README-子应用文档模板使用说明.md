# 子应用文档通用模板使用说明

## 1. 适用范围

本模板用于多子应用项目的标准文档交付，覆盖需求、产品、评审、设计、数据库、操作六类核心文档。

适用对象：

- 新建子应用立项与评审
- 既有子应用迭代版本评审
- 研发、测试、产品跨角色协同

## 2. 模板清单

| 序号 | 模板文件 | 目的 |
|------|------|------|
| 01 | `01-需求说明书模板.md` | 定义需求范围、验收标准、追踪矩阵 |
| 02 | `02-产品设计文档模板.md` | 定义产品定位、信息架构、交互边界 |
| 03 | `03-PRD设计评审文档模板.md` | 评审主文档，沉淀流程、契约、DoD、问题闭环 |
| 04 | `04-详细设计文档模板.md` | 定义页面时序、状态机、异常处理、技术边界 |
| 05 | `05-数据库设计文档模板.md` | 统一演示模式与落地模式的数据口径 |
| 06 | `06-应用操作说明书模板.md` | 提供运行、使用、排障说明 |

## 3. 使用步骤

1. 复制 `通用模板` 目录到目标子应用目录（示例：`docs/子应用文档/<app-code>/`）。
2. 将文件名中的“模板”改为正式文档名。
3. 全局替换占位符：
   - `<子应用名称>`
   - `<app-code>`
   - `<主路由前缀>`
   - `<版本号>`
   - `<评审轮次>`
   - `<负责人>`
4. 按真实业务补齐接口、库表、标准编码、测试证据。
5. 评审前使用 PRD 内“评审会前检查清单”逐项勾选。

## 4. 维护规则

- 文档采用“单文档主干 + 版本变更记录 + 局部变更标记”模式。
- PRD 作为评审主文档，需求、产品、详细设计、数据库文档与其保持对齐。
- 每次评审会后至少更新：
  - PRD 版本变更记录
  - 评审问题清单状态
  - 需求追踪矩阵
  - 评审结论

## 5. 建议交付门槛

- 必填项完整率 >= 90%
- 关键接口契约冻结完成
- 关键异常链路可验证
- DoD 可执行且有证据产物路径
