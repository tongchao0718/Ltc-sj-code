# 代码开发

## 目的

基于已确认需求、PRD、UI 进行可审计编码，实现过程可追踪、可验证、可回退。

## 触发场景

以下条件满足后使用本技能：
- 需求确认、界面确认门禁已通过
- SDD 与技术补齐结论已确认
- 团队进入开发测试阶段

## 必要输入

- 需求基线
- 已确认 PRD
- UI 规格
- G2-B 技术评审结论（接口契约、数据库模型、后端实现边界）

## 模板调用要求（强制）

1. 优先从 `{TEMPLATE_REPO_PATH}` 复制模板后填写（默认：`核心文档/AI+产品落地/02-子应用通用模板/`）。  
2. 本阶段必用模板：
   - `04-详细设计文档模板.md`
   - `14-G3开发任务拆解表模板.md`
3. 若涉及数据模型变更，必须追加 `05-数据库设计文档模板.md`。  
4. 若模板路径不存在，必须自动生成同结构骨架，不得中断执行。  
5. 未按模板章节完整输出，或任务拆解与实现不一致，阶段门禁判定为 `fail` 或 `blocked`（缺关键输入/依赖时）。

## 必交付物

1. 实现任务拆解（小颗粒、可评审）
2. 与需求一致的代码改动
3. 非显性复杂逻辑说明
4. 本地验证结果（build/lint/type/test）
5. 需求到代码映射日志
6. 样机到真实接口/数据落地映射（Mock -> API/DB）

## 执行步骤

1. 将实现拆成最小安全增量
2. 逐增量实施代码变更
3. 每次增量后执行快速校验
4. 持续维护需求追溯关系
5. 修复新增 lint/type 问题
6. 输出最终验证与追溯总结

## 阶段门禁

满足以下条件才可进入测试阶段：
- 改动范围内 build/lint/type 通过
- 需求追溯映射完整
- 无未解决阻塞项
- 前端样机相关 Mock 已替换为真实 API/数据实现（或记录明确例外）

未通过则继续修复，不得提测；若受外部依赖阻塞，状态标记为 `blocked`。
