# 需求澄清

## 目的

在进入 PRD 或编码前统一需求口径。未完成本技能产物，不允许进入下一阶段。

## 触发场景

以下情况必须使用本技能：
- 用户提出新功能或优化需求
- 需求存在歧义、冲突或约束缺失
- 团队即将进入 PRD 设计阶段

## 模板调用要求（强制）

1. 必须从 `核心文档/AI+产品落地/02-子应用通用模板/` 复制模板后填写。  
2. 本阶段必用模板：`01-需求说明书模板.md`。  
3. 若未按模板章节输出或缺失必填章节，阶段门禁判定为 `fail`。  
4. 输出结果需回填到子应用目录，不得直接改写模板母版文件。

## 必交付物

必须一次性输出以下内容：
1. **业务目标**：问题定义与价值预期（1段）
2. **范围边界**：范围内、范围外、关键假设
3. **用户故事**：As a / I want / so that 格式
4. **验收标准**：编号化、可测试、无歧义
5. **风险清单**：合规、数据、排期、依赖风险
6. **待确认项**：尚未决策的问题

## 执行步骤

1. 用一段话复述本次目标
2. 提取用户显式需求
3. 推导隐性约束（时间、数据、平台、合规）
4. 产出用户故事
5. 由用户故事生成验收标准
6. 补充边界场景与失败场景
7. 按固定顺序输出交付物

## 阶段门禁

满足以下条件才可进入 PRD 阶段：
- 验收标准完整且可测试
- 范围边界无自相矛盾
- 待确认项已显式列出

若任一条件不满足，停留在本阶段继续澄清。
