# 测试验证

## 目的

以统一验证标准判断功能是否正确、稳定、可发布。

## 触发场景

以下场景必须使用本技能：
- 代码实现已完成
- 需要给出 merge/release 的 go/no-go 结论

## 必要输入

- 需求验收标准
- 实现总结与改动模块
- 环境约束（dev/test/staging）

## 模板调用要求（强制）

1. 必须从 `核心文档/AI+产品落地/02-子应用通用模板/` 复制模板后填写。  
2. 本阶段必用模板：
   - `08-SDD校验清单模板.md`
   - `09-G2-G3关卡自动门禁检查模板.md`
3. 如涉及自动化门禁执行，需补充 `12-G2-G3自动门禁脚本契约模板.md`。  
4. 未按模板输出测试与门禁结论，阶段门禁判定为 `fail`。

## 必交付物

1. 测试矩阵（标准 -> 用例）
2. 执行证据（通过/失败与说明）
3. 回归影响评估
4. 缺陷清单（严重级别与责任人）
5. 放行建议（go / conditional go / no-go）

## 执行步骤

1. 基于验收标准构建测试矩阵
2. 执行主流程、边界、失败路径
3. 校验改动范围的非功能指标
4. 对受影响模块执行定向回归
5. 记录缺陷并分级
6. 输出放行建议与依据

## 阶段门禁

满足以下条件才可进入合并/发布：
- 所有关键用例通过
- 无阻塞缺陷未关闭
- 回归风险已记录并被接受

未通过则回退 `code-development` 修复。

```markdown
- 标准1 -> 用例ID

### 2) 执行结果
- 用例ID：Pass/Fail，证据

### 3) 回归评估
- 影响模块：
- 回归结论：

### 4) 缺陷清单
- 严重级别 / 描述 / 责任人 / 状态

### 5) 放行建议
- 结论：
- 依据：
```
