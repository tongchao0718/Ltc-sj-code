# 界面生成

## 目的

将 PRD 转化为可开发的 UI 规格，并形成可评审前端交互样机（UI-first）。关键 UI 输出未齐备前，不允许进入编码阶段。

## 触发场景

以下场景使用本技能：
- PRD 已通过评审
- 需要页面级功能与交互定义
- 需要设计到研发的可执行交接

## 必要输入

- 已确认 PRD
- 现有设计系统或风格约束

## 模板调用要求（强制）

1. 必须从 `核心文档/AI+产品落地/02-子应用通用模板/` 复制模板后填写。  
2. 本阶段必用模板：`02-产品设计文档模板.md`。  
3. 产出的页面结构与交互状态需与 PRD 模板章节一一可映射。  
4. 未按模板章节完整输出，阶段门禁判定为 `fail`。

## 必交付物

1. 信息架构与导航图
2. 页面清单与优先级（MVP 优先）
3. 页面级线框文本规格
4. 交互与状态定义（loading/empty/error/success）
5. 组件清单（复用 / 新建）
6. 可访问性基线（键盘、对比度、语义）
7. 前端交互样机范围清单（可 Mock）
8. G2-A 客户界面确认项清单

## 执行步骤

1. 从 PRD 用户流程抽取页面
2. 按页面定义模块结构
3. 补齐交互行为与状态流转
4. 为每个模块映射组件方案
5. 增加响应式与可访问性约束
6. 标注未决设计项

## 阶段门禁

满足以下条件才可进入编码阶段：
- 所有 MVP 页面都有模块级规格
- 关键流程状态处理完整
- 组件归属（复用/新建）明确
- 已形成前端交互样机并完成 G2-A 界面确认记录

未通过则继续完善 UI 规格或补齐样机确认。
