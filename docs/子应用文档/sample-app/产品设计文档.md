# 示例应用（sample-app）产品设计

## 1. 产品概述

### 1.1 产品名称

示例应用（sample-app）

### 1.2 产品定位

展示多子应用架构下的典型布局和功能，参考 Arco Design Pro 的设计风格，提供完整的企业级应用示例。

### 1.3 产品价值

- **示范作用**：展示多子应用架构的最佳实践
- **功能完整**：包含企业级应用的常见功能模块
- **设计规范**：遵循 Arco Design Pro 的设计规范
- **可扩展性**：可作为新子应用的模板和参考
- **学习资源**：为开发者提供学习 Vue 3 和 Arco Design 的示例

## 2. 系统架构

### 2.1 技术栈

- **前端框架**：Vue 3
- **构建工具**：Vite
- **路由**：Vue Router 4（主应用使用 HTML5 History 模式，子应用使用 Hash 模式）
- **状态管理**：Pinia
- **UI 组件库**：Arco Design
- **数据可视化**：ECharts
- **HTTP 客户端**：Axios
- **国际化**：Vue I18n
- **样式**：原生 CSS + CSS 变量 + Less
- **测试框架**：Vitest

### 2.2 架构设计

#### 2.2.1 目录结构

│   ├── form/          # 表单页相关页面

│   ├── monitor/       # 实时监控页面
│   ├── form/          # 分组表单页面

#### 2.2.2 核心流程

1. **子应用启动**：加载子应用框架，显示左侧菜单和内容区域
2. **菜单导航**：点击左侧菜单，跳转到对应页面
3. **独立运行**：直接打开子应用的 HTML 文件运行
4. **子应用开发**：单独启动子应用的开发服务器
5. **子应用打包**：独立打包子应用为静态文件

## 3. 功能模块

> 本文仅保留产品级信息架构和交互边界，详细流程、接口契约、数据模型、DoD 统一维护在 `PRD设计评审文档.md`。

| 一级模块 | 二级页面 | 产品目标 | 关键交互边界 |
|------|------|------|------|
| 仪表盘 | 工作台、实时监控 | 提供业务全局状态感知 | 支持快捷跳转、图表刷新、监控状态查看 |
| 数据可视化 | 数据分析、多维分析 | 提供趋势和多维对比决策能力 | 筛选联动、图表切换、空态展示 |
| 列表页 | 查询表格、卡片列表 | 支撑检索、筛选和批量操作 | 查询参数可追踪，导入导出有状态反馈 |
| 表单页 | 分步表单、分组表单 | 支撑结构化信息录入与提交 | 字段校验、步骤状态、失败保留输入 |
| 详情页 | 基础详情页 | 支撑对象详情查看与操作 | 标签切换稳定、返回路径明确 |
| 结果页 | 成功页、失败页 | 提供操作结果反馈与后续指引 | 重试/返回入口明确，避免迷路 |
| 异常页 | 403、404、500 | 提供异常解释和恢复路径 | 异常文案明确，回退路径可执行 |
| 个人中心 | 个人资料、用户设置 | 支撑用户信息维护和安全配置 | 权限字段隔离、保存反馈明确 |

## 4. 交互原则

- **主应用内路径**：带 `/sample-app` 前缀
- **侧栏导航**：分组可折叠，当前路由高亮
- **菜单背景**：左侧菜单背景为白色，与 Arco Design Pro 一致
- **内容区域**：背景为白色，卡片和表格具有适当的阴影和边框
- **响应式设计**：适配不同屏幕尺寸
- **交互反馈**：点击按钮有反馈，表单验证有提示，加载状态有显示
- **导航逻辑**：菜单点击跳转到对应页面，面包屑显示当前位置
- **独立运行**：子应用支持独立运行，无需主应用

## 6. 技术实现

### 6.1 前端实现

- **Vue 3**：使用 Composition API，代码结构清晰
- **Vue Router**：主应用使用 HTML5 History 模式，子应用使用 Hash 模式，支持本地文件访问
- **Pinia**：使用 Pinia 进行状态管理，支持跨组件数据共享和持久化存储
- **Arco Design**：使用 Arco Design 组件库，提供统一的 UI 组件和设计规范
- **ECharts**：使用 ECharts 实现数据可视化图表，支持丰富的图表类型和交互
- **Axios**：使用 Axios 进行 HTTP 请求，支持请求拦截和响应拦截
- **Vue I18n**：使用 Vue I18n 实现国际化支持，支持多语言切换
- **CSS 变量**：统一的设计系统，便于主题管理
- **Less**：使用 Less 预处理器，提高样式开发效率
- **响应式设计**：适配不同屏幕尺寸
- **组件库**：可复用组件，提高开发效率
- **Vitest**：使用 Vitest 进行单元测试，提高代码质量

### 6.2 构建配置

- **Vite**：快速的构建工具，支持热更新
- **独立配置**：为子应用提供独立的构建配置
- **相对路径**：使用 `base: './'`，支持本地文件访问
- **IIFE 格式**：避免模块加载问题，提高兼容性
- **代码压缩**：使用 terser 进行代码压缩，减少文件体积

## 7. 部署与维护

### 7.1 部署方案

1. **打包**：`npm run build:sample-app`
2. **部署选项**：
   - **独立部署**：将 `dist/sample-app/` 目录部署到服务器
   - **本地运行**：直接使用 `index.html` 或 `start.html`
   - **集成部署**：作为主系统的一部分部署

### 7.2 维护指南

- **代码规范**：遵循 Vue 3 最佳实践，使用 ESLint 和 Prettier 确保代码质量
- **目录结构**：保持统一的目录结构，将复杂组件拆分为更小、更可维护的组件
- **命名规范**：
  - 组件文件名：使用 PascalCase 命名风格
  - 路由名称：使用 PascalCase 命名风格，前缀与子应用名称一致
  - 变量和函数：使用 camelCase 命名风格
- **文档更新**：及时更新相关文档，确保文档与代码保持同步
- **设计规范**：遵循《系统设计规范》文档中的设计标准，确保界面风格一致
