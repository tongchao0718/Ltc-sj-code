# 示例应用（sample-app）产品设计

## 1. 定位

展示多子应用架构下的典型布局：侧栏导航、协议类功能分组、通用功能模块占位。

## 2. 信息架构

- **首页**：功能入口卡片，跳转各模块。
- **协议管理**：总览、清单、增量、监控、智能（占位页，可替换为真实业务）。
- **功能模块**：模块 1～3 占位，用于演示扩展。

## 3. 交互原则

- 主应用内路径带 `/sample-app` 前缀；独立包使用 Hash，无前缀。
- 侧栏分组可折叠，当前路由高亮。

## 4. 后续迭代

可接入真实协议数据、权限与后端 API，并同步更新本目录下需求与详细设计文档。
