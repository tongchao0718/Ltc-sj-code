# 示例应用数据库设计

## 1. 设计口径

`sample-app` 当前可支持两种运行模式：

1. **演示模式（默认）**：前端静态演示，可使用 Mock 数据，无强制依赖真实数据库。  
2. **落地模式（评审口径）**：按 `PRD设计评审文档.md` 的物理模型与接口契约对接服务端。  

为避免口径冲突，本文档采用“**演示可降级、评审可落地**”原则：即使演示阶段无真实库表，评审与实施仍以 PRD 库表设计为准。

## 2. 最小库表清单（与 PRD 对齐）

| 功能域 | 最小必需表 |
|------|------|
| 仪表盘 | `dashboard_metric_snapshot`、`dashboard_trend_daily`、`hot_content_rank`、`content_type_ratio` |
| 数据可视化 | `viz_dataset`、`viz_dimension_value`、`viz_chart_snapshot` |
| 列表与查询 | `content_collection`、`content_collection_item`、`import_task`、`export_task` |
| 表单与个人中心 | `user_profile`、`user_security`、`user_notice_config`、`form_step_draft`、`form_submit_record` |
| 结果与异常 | `result_log`、`exception_log` |
| 权限范围 | `role_data_scope` |

## 3. 数据一致性约束（最小集）

- 所有状态字段必须使用标准编码（如 `CONTENT_STATUS`、`RESULT_TYPE`）。  
- 关键链路必须具备 `trace_id` 用于结果与异常关联追踪。  
- 导入导出任务必须持久化任务状态，禁止仅以内存状态表示。  
- 用户数据遵循最小权限原则，个人中心默认仅可写本人数据。  

## 4. 变更管理

- 新增/变更库表必须同步更新：`PRD设计评审文档.md` 的“物理模型”和“接口契约清单”。  
- 评审前需完成字段冻结；字段变更必须附评审变更单。  
- 演示模式与落地模式存在差异时，以 PRD 评审结论为最终依据。
