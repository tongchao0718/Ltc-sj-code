# <子应用名称> 数据库设计文档

## 1. 设计口径

本子应用支持两种模式：

1. 演示模式：前端演示或 Mock 数据，可无真实库表。
2. 落地模式：按 PRD 评审文档物理模型与接口契约对接服务端。

原则：演示可降级，评审可落地。评审与实施以 PRD 为准。

## 4. 数据一致性约束

- 状态字段统一使用标准编码。
- 关键链路保留 `trace_id` 便于问题追踪。
- 异步任务需持久化任务状态。
- 权限数据遵循最小权限原则。

## 5. 变更管理

- 新增/变更库表需同步更新 PRD 的“物理模型”和“接口契约”。
- 字段冻结后变更需走评审变更单。
- 模式冲突时，以评审结论为最终依据。
