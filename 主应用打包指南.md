# 主应用打包指南

## 1. 指南概述

本指南详细说明如何为应用系统的主应用进行打包，包括打包配置、执行步骤、验证方法和最佳实践。遵循本指南可以确保主应用打包的正确性和一致性。

## 2. 打包准备

### 2.1 环境准备

1. **安装 Node.js**：
   - 确保安装 **Node.js 18.0.0 或更高版本**（与 Vite 6 及本仓库 `package.json` 一致）
   - 确保安装 npm 8.0.0 或更高版本

2. **安装依赖**：
   ```bash
   npm install
   ```

3. **验证环境**：
   ```bash
   node --version
   npm --version
   ```

### 2.2 代码准备

1. **检查代码**：
   - 确保主应用代码已完成开发
   - 检查代码是否存在语法错误
   - 验证路由配置是否正确

2. **清理缓存**：
   ```bash
   npm cache clean --force
   ```

## 3. 打包配置

### 3.1 主打包配置

**`package.json`** 中的打包脚本配置：

```json
{
  "scripts": {
    "build": "vite build"
  }
}
```

### 3.2 主应用打包配置（与仓库一致）

当前主应用使用 **Vite 默认 ESM 分包**，**不**输出到 `dist/main`，**不**使用根目录 `vite.config.js` 生成主应用 `start.html`（子应用独立打包的 `start.html` 见 **`子应用打包指南.md`** 与 **`build/vite.sample-app.config.js`**）。

**根目录 `vite.config.js`**（与仓库一致）：

```javascript
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'
import { resolve } from 'path'
import { fileURLToPath } from 'url'

const __dirname = fileURLToPath(new URL('.', import.meta.url))

export default defineConfig({
  plugins: [vue()],
  resolve: {
    alias: {
      '@': resolve(__dirname, 'src')
    }
  }
})
```

**说明**：

- **`base`**：未显式设置时默认为 `/`，适合部署在域名根路径；若需子路径部署，再设置 `base: '/子路径/'`。
- **`build.outDir`**：默认为 **`dist/`**（非 `dist/main`）。
- **产物格式**：Rollup 默认 **ES 模块** + 代码分包，**不是**主应用级 IIFE 单文件（与示例子应用 **`build:sample-app`** 不同）。

### 3.3 主应用 HTML 入口

**`index.html`**（与仓库一致；图标以 `public/` 为准）：

```html
<!doctype html>
<html lang="zh-CN">
  <head>
    <meta charset="UTF-8" />
    <link rel="icon" type="image/png" href="/app-icon.png" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>应用系统</title>
  </head>
  <body>
    <div id="app"></div>
    <script type="module" src="/src/main.js"></script>
  </body>
</html>
```

## 4. 打包执行

### 4.1 执行打包命令

1. **打包主应用**：
   ```bash
   npm run build
   ```

2. **监控打包过程**：
   - 观察打包输出日志
   - 检查是否有错误或警告
   - 验证打包是否成功完成

### 4.2 打包结果

打包完成后，**`dist/`** 目录（根目录下，**非** `dist/main`）将包含例如：

```
dist/
├── index.html          # 主应用入口 HTML（引用 type="module" 的 JS）
├── assets/             # JS / CSS 等（带内容哈希的文件名）
│   ├── index-xxxxx.js
│   ├── index-xxxxx.css
│   └── ...
└── ...                 # 自 public/ 复制的静态资源（如 app-icon.png）
```

**说明**：

- 主应用 **不** 生成 `dist/start.html`；若需「单文件 + 本地打开」模式，请参考 **子应用** 的 `npm run build:sample-app` 产物。
- `assets/` 内文件名含哈希，以实际构建为准。

## 5. 打包验证

### 5.1 功能验证

1. **本地服务器验证（推荐）**：
   - 使用 `npm run preview` 或任意静态服务器指向 **`dist/`**
   - **不要**依赖直接双击 `index.html` 用 `file://` 打开：ES 模块与 `createWebHistory` 在本地文件协议下通常异常
   - 验证路由跳转、刷新后当前页能否正确恢复（需服务器对 SPA 做回退到 `index.html` 的配置）

### 5.2 资源验证

1. **静态资源**：
   - 检查 CSS、JavaScript 文件是否正确加载
   - 验证图片、字体等静态资源是否正常显示

2. **路由验证**：
   - 测试所有路由是否正常跳转
   - 验证页面刷新后是否能正确显示

### 5.3 目录验证

1. **目录结构**：
   - 检查打包输出目录结构是否完整
   - 验证所有必要文件是否生成

2. **文件完整性**：
   - 检查 HTML、CSS、JavaScript 文件是否存在
   - 验证文件大小是否合理

## 6. 部署指南

### 6.1 部署方式

1. **静态文件部署**：
   - 将 **`dist/`** 目录（根目录打包产物）复制到静态文件服务器根路径或子路径（若子路径需同步配置 `vite.base`）
   - 通过服务器 URL 访问主应用

2. **离线 / 内网分发**：
   - 将 **`dist/`** 打包为 ZIP；用户解压后**仍需**通过本地 HTTP 服务（如 `preview`、nginx、VS Code Live Server）访问，**不要**指望直接双击 `index.html` 替代完整主应用

3. **CDN 部署**：
   - 将静态资源上传到 CDN
   - 修改 HTML 文件中的资源路径为 CDN 地址

### 6.2 部署注意事项

1. **路径配置**：
   - 若部署在子目录，需设置 **`vite.config.js` 的 `base`**，与服务器挂载路径一致
   - 验证 `index.html` 引用的 `assets/*` 是否 404

2. **浏览器兼容性**：
   - 测试在主流浏览器中的兼容性
   - 确保在不同浏览器中都能正常运行

3. **性能优化**：
   - 检查打包文件大小，确保优化合理
   - 验证加载速度和响应时间

## 7. 最佳实践

### 7.1 打包配置

- 主应用以 **仓库根目录 `vite.config.js`** 为准；默认 **ESM 分包 + 输出 `dist/`**
- 需要 **子路径部署** 时配置 `base`，并在服务器上为 SPA 配置 **fallback 到 `index.html`**
- **CDN 版 / IIFE 单包**，属可选方案，本仓库主应用**未**内置；勿与 **`build/vite.sample-app.config.js`**（子应用）混淆

### 7.2 依赖管理

- 使用 `package-lock.json` 或 `yarn.lock` 锁定依赖版本
- 定期更新依赖，保持依赖版本的一致性
- 避免使用过多的依赖，减少打包体积

### 7.3 代码优化

- 移除未使用的代码和依赖
- 使用代码分割，减少初始加载时间
- 优化图片和静态资源，减少文件大小

### 7.4 验证流程

- 建立标准化的打包验证流程
- 记录打包过程和结果
- 定期验证打包结果的一致性

## 8. 常见问题及解决方案

### 8.1 打包失败

**症状**：打包命令执行失败
**原因**：依赖错误、配置错误或代码错误
**解决**：
- 检查依赖是否正确安装
- 验证打包配置是否正确
- 检查代码是否存在语法错误

### 8.2 资源加载失败

**症状**：页面空白或 404
**原因**：`base` 与部署路径不一致，或静态资源未随 `dist/` 一并上传
**解决**：
- 核对 **`vite` 的 `base`** 与 CDN/子目录路径
- 用浏览器开发者工具查看 `assets/*.js` 是否 404
- 主应用 **无** `start.html`；子应用独立包见 **`子应用打包指南`**

### 8.3 路由跳转失败或刷新 404

**症状**：主应用内点击正常，但刷新或直接访问深层路径 404
**原因**：**`createWebHistory`** 需要服务器将所有前端路由回退到 **`index.html`**
**解决**：
- 在 nginx / IIS / Netlify 等配置 **SPA fallback**
- 或仅在明确需求下改用 Hash 路由（需改代码，与现网 `src/router/index.js` 不一致）

### 8.4 打包文件过大

**症状**：打包文件体积过大，影响加载速度
**原因**：依赖过多或代码未优化
**解决**：
- 移除未使用的依赖
- 使用代码分割
- 优化静态资源
