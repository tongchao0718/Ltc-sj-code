# 应用系统创建指南

## 1. 概述

本指南详细说明如何创建一个与应用系统相同架构的多子应用系统框架。通过遵循本指南的步骤，您可以构建一个支持多个独立子应用的 Vue 3 系统。

### 1.2 与当前仓库对齐（必读）

搭建或核对实现时，**以仓库代码为唯一事实来源**，并与 **`文档对齐清单.md`**、**`framework-baseline.json`** 交叉核对。以下约定与旧版文档不同，按此执行才能与现网 **1:1**：

| 项         | 正确约定（现网）                                                              | 常见错误（旧文档示例）                                          |
| --------- | --------------------------------------------------------------------- | ---------------------------------------------------- |
| 应用中心路由    | **`/app-center`**                                                     | `/apps`                                              |
| 使用帮助 / 关于 | **独立页面** `/help`、`/about`（`HelpPage.vue` / `AboutPage.vue`）           | 仅 `HelpModal` / `AboutModal` 弹窗                      |
| 项目审查      | **`/review-center`**（`ReviewCenter.vue`）                              | `/review`、`views/review/` 多文件结构（与现网不一致）              |
| 主应用打包     | **`npm run build`** **→** **`dist/`**，根目录 `vite.config.js` 为默认 ESM 分包 | `dist/main`、主应用内 `copyHtmlPlugin` + IIFE（本仓库**未**采用） |
| 示例子应用路由   | 与 Arco Pro 一致：`dashboard/workplace`、`list/*`、`user/info` 等            | `Home.vue`、`protocol/*`、`Module1` 等旧示例               |

下文已按优先级修订；若某段仍含「历史示例」字样，请以 **`src/router/index.js`**、**`src/App.vue`** 为准。

## 2. 环境要求

- **Node.js**：**18.0 或更高版本**（Vite 6 要求；与 `package.json` 中 Vite 大版本一致）
- **npm**：8.0 或更高版本（建议随 Node LTS 自带版本）
- **Git**：用于版本控制
- **编辑器**：VS Code 或其他代码编辑器

### 2.1 环境安装

#### 2.1.1 Windows 一键安装脚本

创建一个名为 `install-env.ps1` 的 PowerShell 脚本：

```powershell
# 检测 Node.js 是否已安装
Write-Host "检测 Node.js 安装状态..."
try {
    $nodeVersion = node -v
    Write-Host "Node.js 已安装: $nodeVersion"
} catch {
    Write-Host "Node.js 未安装，开始安装..."
    # 下载并安装 Node.js LTS
    Write-Host "正在下载 Node.js LTS..."
    Invoke-WebRequest -Uri "https://nodejs.org/dist/v20.11.1/node-v20.11.1-x64.msi" -OutFile "nodejs-installer.msi"

    Write-Host "正在安装 Node.js..."
    Start-Process msiexec.exe -ArgumentList "/i nodejs-installer.msi /quiet /qn /norestart" -Wait

    # 清理安装文件
    Remove-Item nodejs-installer.msi
}

# 检测 Git 是否已安装
Write-Host "检测 Git 安装状态..."
try {
    $gitVersion = git --version
    Write-Host "Git 已安装: $gitVersion"
} catch {
    Write-Host "Git 未安装，开始安装..."
    # 下载并安装 Git
    Write-Host "正在下载 Git..."
    Invoke-WebRequest -Uri "https://github.com/git-for-windows/git/releases/download/v2.43.0.windows.1/Git-2.43.0-64-bit.exe" -OutFile "git-installer.exe"

    Write-Host "正在安装 Git..."
    Start-Process .\git-installer.exe -ArgumentList "/VERYSILENT /NORESTART /NOCANCEL /SP- /SUPPRESSMSGBOXES" -Wait

    # 清理安装文件
    Remove-Item git-installer.exe
}

# 验证安装
Write-Host "验证 Node.js 安装..."
try {
    node -v
} catch {
    Write-Host "Node.js 安装失败"
}

Write-Host "验证 npm 安装..."
try {
    npm -v
} catch {
    Write-Host "npm 安装失败"
}

Write-Host "验证 Git 安装..."
try {
    git --version
} catch {
    Write-Host "Git 安装失败"
}

Write-Host "环境安装完成！"
```

#### 2.1.3 Linux 一键安装脚本

创建一个名为 `install-env.sh` 的 bash 脚本：

```bash
#!/bin/bash

# 检测 Node.js 是否已安装
if command -v node &> /dev/null; then
    echo "Node.js 已安装: $(node -v)"
else
    echo "Node.js 未安装，开始安装..."
    # 更新包管理器
    echo "正在更新包管理器..."
    sudo apt update || sudo yum update || sudo dnf update
    
    # 安装 Node.js
    if command -v apt &> /dev/null; then
        sudo apt install -y nodejs npm
    elif command -v yum &> /dev/null; then
        sudo yum install -y nodejs npm
    elif command -v dnf &> /dev/null; then
        sudo dnf install -y nodejs npm
    fi
fi

# 检测 Git 是否已安装
if command -v git &> /dev/null; then
    echo "Git 已安装: $(git --version)"
else
    echo "Git 未安装，开始安装..."
    # 安装 Git
    if command -v apt &> /dev/null; then
        sudo apt install -y git
    elif command -v yum &> /dev/null; then
        sudo yum install -y git
    elif command -v dnf &> /dev/null; then
        sudo dnf install -y git
    fi
fi

# 验证安装
echo "验证 Node.js 安装..."
try {
    node -v
} catch {
    echo "Node.js 安装失败"
}

echo "验证 npm 安装..."
try {
    npm -v
} catch {
    echo "npm 安装失败"
}

echo "验证 Git 安装..."
try {
    git --version
} catch {
    echo "Git 安装失败"
}

echo "环境安装完成！"
```

                <router-link to="/help" class="dropdown-item" role="menuitem" @click="closeUserMenu">使用帮助</router-link>
                <router-link to="/about" class="dropdown-item" role="menuitem" @click="closeUserMenu">关于我们</router-link>
                <router-link to="/review-center" class="dropdown-item" role="menuitem" @click="closeUserMenu">项目审查</router-link>

  name: 'App',
  data() {
    return {
      userMenuOpen: false,
      isScrolled: false,
      menuCloseTimer: null
    }
  },
  watch: {
    $route() {
      this.closeUserMenu()
    }
  },
  mounted() {
    window.addEventListener('scroll', this.handleScroll, { passive: true })
    document.addEventListener('click', this.onDocClick)
  },
  beforeUnmount() {
    window.removeEventListener('scroll', this.handleScroll)
    document.removeEventListener('click', this.onDocClick)
    if (this.menuCloseTimer) clearTimeout(this.menuCloseTimer)
  },
  methods: {
    toggleUserMenu() {
      this.userMenuOpen = !this.userMenuOpen
    },
    closeUserMenu() {
      this.userMenuOpen = false
    },
    onUserMenuEnter() {
      if (this.menuCloseTimer) {
        clearTimeout(this.menuCloseTimer)
        this.menuCloseTimer = null
      }
    },
    onUserMenuLeave() {
      this.menuCloseTimer = setTimeout(() => {
        this.userMenuOpen = false
      }, 200)
    },
    onDocClick() {
      this.userMenuOpen = false
    },
    handleScroll() {
      this.isScrolled = window.scrollY > 8
    }
  }
}

**创建** **`build/vite.sample-app.config.js`**：

```javascript
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'
import { resolve } from 'path'
import fs from 'fs'

// 自定义插件：在构建完成后复制和重命名 HTML 文件
const copyHtmlPlugin = () => ({
  name: 'copy-html',
  closeBundle() {
    const distPath = resolve(__dirname, '../dist/sample-app')
    const buildPath = resolve(distPath, 'build')
    const buildSrcPath = resolve(__dirname)
    
    // 如果 build 目录存在，移动文件到根目录
    if (fs.existsSync(buildPath)) {
      // 读取生成的 sample-app.html 文件
      let htmlContent = fs.readFileSync(resolve(buildPath, 'sample-app.html'), 'utf8')
      
      // 修正资源引用路径，将 '../assets/' 改为 './assets/'
      htmlContent = htmlContent.replace(/\.\.\/assets\//g, './assets/')
      
      // 写入修正后的 sample-app.html
      fs.writeFileSync(resolve(distPath, 'sample-app.html'), htmlContent)
      
      // 复制为 index.html
      fs.writeFileSync(resolve(distPath, 'index.html'), htmlContent)
      
      // 删除 build 目录
      fs.rmSync(buildPath, { recursive: true })
    }
    
    // 复制 start.html (CDN版本) 到根目录
    if (fs.existsSync(resolve(buildSrcPath, 'start.html'))) {
      // 读取 start.html 文件
      let startContent = fs.readFileSync(resolve(buildSrcPath, 'start.html'), 'utf8')
      
      // 查找实际的 JS 文件名
      const assetsDir = resolve(distPath, 'assets')
      if (fs.existsSync(assetsDir)) {
        const jsFiles = fs.readdirSync(assetsDir).filter(file => file.endsWith('.js'))
        if (jsFiles.length > 0) {
          // 替换 JS 文件名
          startContent = startContent.replace(/\.\/assets\/main\.js/g, `./assets/${jsFiles[0]}`)
        }
      }
      
      // 写入修正后的 start.html
      fs.writeFileSync(resolve(distPath, 'start.html'), startContent)
    }
  }
})

// https://vite.dev/config/
export default defineConfig({
  // 插件配置
  plugins: [vue(), copyHtmlPlugin()],
  base: './',
  root: resolve(__dirname, '..'),
  build: {
    // 输出目录
    outDir: resolve(__dirname, '../dist/sample-app'),
    
    // 清空输出目录
    emptyOutDir: true,
    
    // Rollup 配置
    rollupOptions: {
      // 入口文件
      input: {
        main: resolve(__dirname, 'sample-app.html')
      },
      // 输出配置
      output: {
        // 使用 IIFE 格式，避免模块加载问题
        format: 'iife',
        // 资源文件命名
        entryFileNames: 'assets/[name]-[hash].js',
        chunkFileNames: 'assets/[name]-[hash].js',
        assetFileNames: 'assets/[name]-[hash].[ext]'
      }
    }
  },
  resolve: {
    alias: {
      '@': resolve(__dirname, '../src')
    }
  }
})
```

#### 4.4.2 版本回滚脚本

**创建** **`scripts/rollback.js`**：

```javascript
const fs = require('fs');
const path = require('path');
const { execSync } = require('child_process');

console.log('开始版本回滚...');

try {
  // 1. 检查当前目录是否为git仓库
  console.log('检查git仓库状态...');
  execSync('git status', { stdio: 'inherit' });

  // 2. 列出最近的提交
  console.log('\n最近的提交记录：');
  execSync('git log --oneline -5', { stdio: 'inherit' });

  // 3. 提示用户输入回滚目标
  const readline = require('readline').createInterface({
    input: process.stdin,
    output: process.stdout
  });

  readline.question('\n请输入要回滚到的提交哈希值: ', (commitHash) => {
    try {
      // 4. 执行回滚
      console.log(`\n回滚到提交: ${commitHash}`);
      execSync(`git reset --hard ${commitHash}`, { stdio: 'inherit' });

      // 5. 安装依赖
      console.log('\n安装依赖...');
      execSync('npm install', { stdio: 'inherit' });

      // 6. 构建项目
      console.log('\n构建项目...');
      execSync('npm run build', { stdio: 'inherit' });
      execSync('npm run build:sample-app', { stdio: 'inherit' });

      console.log('\n✅ 版本回滚完成！');
      console.log('\n📝 回滚内容：');
      console.log(`  - 回滚到提交: ${commitHash}`);
      console.log('  - 重新安装依赖');
      console.log('  - 重新构建项目');

    } catch (error) {
      console.error('❌ 回滚失败:', error.message);
    } finally {
      readline.close();
    }
  });

} catch (error) {
  console.error('❌ 回滚失败:', error.message);
  process.exit(1);
}
```

## 8. 扩展和维护

### 8.1 添加新子应用

1. **创建目录结构**：
   ```bash
   mkdir -p src/apps/{app名称}/views
   ```
   **命名规则**：
   - 子应用文件夹名应使用语义化的英文名
   - 采用kebab-case命名风格（小写字母和连字符）
   - 避免使用数字编号（如sample-app、app2等）
   - 示例：`sample-app`、`user-management`、`data-analysis`
2. **创建核心文件**：
   - `src/apps/{app名称}/App{App名称}.vue`
   - `src/apps/{app名称}/main.js`
   - `src/apps/{app名称}/views/Home.vue`
3. **创建打包配置**：
   - `build/{app名称}.html`
   - `build/vite.{app名称}.config.js`
4. **更新 package.json**：
   ```json
   {
     "scripts": {
       "build:{app名称}": "vite build --config build/vite.{app名称}.config.js",
       "dev:{app名称}": "vite --config build/vite.{app名称}.config.js"
     }
   }
   ```
5. **更新主路由**：在 `src/router/index.js` 中添加新子应用路由
6. **更新应用中心**：在 `src/views/AppCenter.vue` 中添加新子应用卡片

#### 8.3.3 文档更新说明

为确保系统的可维护性和可理解性，每次对应用进行更新时，必须同步更新相关文档：

1. **更新内容**：
   - 功能变更：新增、修改或删除功能时，更新产品设计文档和需求说明书
   - 接口变更：API 接口发生变化时，更新接口文档
   - 架构变更：系统架构发生变化时，更新概要设计文档和详细设计文档
   - 操作流程变更：操作流程发生变化时，更新应用操作说明书
   - 数据库变更：数据库结构发生变化时，更新数据库设计文档
2. **更新流程**：
   - 在应用更新完成后，立即开始文档更新
   - 确保文档内容与实际代码和功能保持一致
   - 在版本控制中，将代码变更和文档变更放在同一提交中
   - 定期审查文档，确保其准确性和完整性
3. **文档版本管理**：
   - 保持文档版本与应用版本一致
   - 在文档中记录版本变更历史
   - 确保文档的可追溯性
4. **更新检查清单**：
   - [ ] 功能变更是否已更新到产品设计文档
   - [ ] 接口变更是否已更新到接口文档
   - [ ] 架构变更是否已更新到概要设计文档和详细设计文档
   - [ ] 操作流程变更是否已更新到应用操作说明书
   - [ ] 数据库变更是否已更新到数据库设计文档
   - [ ] 文档版本是否与应用版本一致
   - [ ] 所有相关文档是否已同步更新

### 8.4 打包约束

在进行子应用打包时，必须严格按照以下约束执行：

1. **打包流程**：
   - 确保所有子应用都有独立的打包配置文件
   - 使用指定的打包命令（如 `npm run build:sample-app`）
2. **打包验证**：
   - 打包完成后，验证子应用是否能正常加载和运行
   - 测试所有路由和功能是否正常
   - 确保 CDN 版本也能正常运行
3. **打包标准**：
   - 打包输出必须符合项目规范
   - 静态资源路径必须正确配置
   - 代码必须经过优化和压缩
   - 确保打包后文件大小合理
4. **打包检查清单**：
   - [ ] 是否按照打包指南的步骤进行打包

## 9. 总结

本创建指南详细说明了如何创建一个与应用系统相同架构的多子应用系统框架。通过遵循这些步骤，您可以构建一个功能完整、架构清晰、易于扩展的 Vue 3 多子应用系统。

### 核心要点

- **多子应用架构**：支持多个独立子应用的集成
- **独立打包**：每个子应用可以独立打包和部署
- **本地访问**：支持直接打开 HTML 文件运行
- **统一配置**：提供统一的开发和打包配置
- **快速扩展**：支持快速添加新的子应用

通过本指南创建的系统将为企业级应用提供一个坚实的框架基础，可以根据具体业务需求快速构建和扩展功能模块。
