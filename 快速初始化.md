# 快速初始化（推荐给团队）

目标：用**最少步骤**得到与当前仓库**一致**的可运行项目，**不要**依赖通读《创建指南》从零手敲。

---

## 方式 A：Git 克隆（最轻、最可靠）

1. 在 Git 托管平台（GitHub / GitLab / Gitee 等）建立**只读仓库**，把本仓库推上去。
2. 成员执行：

```bash
git clone <仓库 URL> my-ltc-app
cd my-ltc-app
nvm use 20   # 或确保 Node >= 18，见 package.json / framework-baseline.json
npm ci       # 优先：与 lockfile 完全一致；无 lock 时用 npm install
npm run dev
```

3. 浏览器打开终端里提示的本地地址（一般为 `http://localhost:5173`）。

**说明**：`npm ci` 依赖 **`package-lock.json`**，务必纳入版本控制。

---

## 方式 B：发布「代码包」Release（无 Git 时）

1. 在干净目录打包（**不要**包含 `node_modules`、`dist`）：

```bash
# 示例：在项目根目录上级执行
tar -czvf ltc-framework-v1.1.0.tar.gz --exclude=node_modules --exclude=dist Project-sj
```

或在 Windows 资源管理器中压缩项目文件夹，注意删掉 `node_modules` 与 `dist` 以减小体积。

2. 成员解压后：

```bash
cd Project-sj
npm ci
npm run dev
```

3. 将 **`package-lock.json`** 与 **`framework-baseline.json`** 一并分发，便于核对版本。

---

## 方式 C：用 Cursor / Trae 等 AI 工具「对齐」而非「从零生成」

**不要**对 AI 说：「按《创建指南》全文生成项目」——容易与现网分叉。

**建议提示词模板**（复制后把路径改成本机克隆目录）：

```text
工作区已打开本仓库克隆（Vue3 + Vite + Arco）。请遵守：
1. 以 src/、vite.config.js、package-lock.json 为唯一事实来源。
2. 路由必须与 src/router/index.js 一致；应用中心路径为 /app-center。
3. 需要改依赖时对照 framework-baseline.json，并说明理由。
4. 不要删除或替换整份创建指南式重写；仅在现有结构上迭代功能。
```

把 `文档对齐清单.md`、`框架基线.md` 留在仓库里，供人工与 AI 抽查。

---

## 与《创建指南》的分工

| 场景 | 建议 |
|------|------|
| 新成员跑起来项目 | **本文 + README「快速开始」** 即可 |
| 从 0 手写另一个同名工程 | 才需要对照《创建指南》**并已修订的章节**（§1.2、§4.2.3 等） |
| 核对是否「同一套框架」 | **`framework-baseline.json`** + `npm ls --depth=0` |

---

## 可选：私有 npm 模板 / degit（进阶）

若团队熟悉 Node 生态，可将本仓库打成 **git 模板仓库**，使用：

```bash
npx degit <org>/ltc-framework#main my-app
cd my-app && npm ci && npm run dev
```

前提是模板仓库对成员可见，且仍保留 `package-lock.json`。

---

## 一句话结论

**最轻量路径**：**Git 克隆 + `npm ci` + `npm run dev`**，辅以 **`framework-baseline.json`** 锁版本；文档只读 **README** 与本文即可。
