## 领导 30 秒结论（口播用）

1. **契约收口**：接口与数据只认 **SDD**，页面与交互只认 **PRD**；开发、联调提测按**门禁举证**，不用群聊当合同。  
2. **对接前移**：**双周共创** + **三阶段确认**（需求目标、原型体验、上线预期），纪要带行动项与责任人。  
3. **AI 可管**：项目声明 **L1～L4**，越阶要 **PO 书面同意**；每季度登记 **1～2 个提效场景**（基线人时 + 目标）。

## 一、执行摘要（汇报展开）

### 1.1 背景与问题

- 设计工具与 AI 使用偏碎片化，产物与开发契约易脱节。  
- 客户对接偏晚、结构化不足，易引发返工与范围漂移。

### 1.2 目标（三条）

1. **制度统一**：单一事实源（SDD/PRD）、门禁举证、对接与台账有硬性标准。  
2. **对接前置**：双周共创 + 三阶段确认 + 1～2 周可见验证，承诺可追溯。  
3. **AI 可管**：L1～L4 分级声明、越阶审批、输出过 Gate；短长期清单量化。

### 1.3 适用范围

产品设计小组在编成员；项目中承担 PO/PM 职责人员，按角色执行对应条款。

### 1.4 与现有体系关系

| 层级 | 文档 | 作用 |
|------|------|------|
| 治理 | 《AI+产品全流程执行手册-治理版》 | 红线与 Gate 唯一依据 |
| 执行 | 《执行版》六步主线 | 与日常迭代对齐 |
| 制度镜像 | `../03-其他过程文档-参考/产品设计小组规范方案-制度篇（合并镜像）.md` | 完整制度 §0～§10 |
| 合并主文档 | `../03-其他过程文档-参考/项目全链路执行方案.md` | 战略 + 全链路 + 对接 |
| 一页纸 | `核心文档/领导汇报-产品小组规范（一页纸）.md` | **领导只看这一份即可** |
| **本文件** | 汇报与落地版 | 摘要展开 + 首季节奏 + 精简检查单 |

### 1.5 预期成效（与度量挂钩）

- 需求与契约：书面确认率、行动项关闭率；需求变更率先建基线。  
- 质量与效率：SDD 一次复核通过率、PRD 冻结后字段变更次数。  
- AI：L 阶次合规率、Gate 一次通过率、兜底切换次数（宜下降）。

## 二、核心机制总表（汇报可讲「六件事」）

| # | 机制 | 一句话 |
|---|------|--------|
| 1 | **单一事实源** | 契约与验收在 **SDD**；页面与交互在 **PRD**；对外承诺在**确认单/纪要**并带版本。 |
| 2 | **门禁举证** | 无需求确认双签、无 SDD 签署、无 PRD 冻结与字段映射，不得对应承诺开发/联调提测。 |
| 3 | **客户对接** | 双周共创 + **三阶段确认**；纪要含行动项与责任人日期。 |
| 4 | **对内对接** | 干系人台账 24h 内更新；**禁止**以群聊为唯一契约。 |
| 5 | **AI 分级** | 声明 **L1～L4**；越阶须 **PO 书面同意**；AI 不替代签署、冻结、发布放行。 |
| 6 | **闭环** | 规范类问题进**台账**；小组定期评审四维度。 |

## 四、执行标准速查（落地对照）

### 4.1 六步与门禁举证

**顺序**：需求生成 → 需求确认 → 界面生成 → 界面确认 → 开发测试 → 发布上线  

| 门槛 | 必须已具备的举证 |
|------|------------------|
| 需求确认完成 | 需求确认单 **PO+TL 双签** |
| 开发排期/开发 | **SDD 已签署** |
| 联调 / 提测 | **PRD 已冻结** + **界面元素 ↔ SDD 字段映射表** |
| 发布 | 测试、CI/CD、追溯链符合《治理版》 |

### 4.2 三阶段确认（证据名称）

| 阶段 | 证据 |
|------|------|
| 需求阶段 | 需求确认单（引用需求卡 + SDD 摘要） |
| 原型阶段 | PRD 冻结前评审纪要 + 字段映射表 |
| 上线前 | 上线确认记录（可与发布检查合并） |

### 4.3 AI 阶次 L1～L4（简表）

| 阶次 | 用途概要 |
|------|----------|
| L1 | 润色、纪要/清单草稿；短人工校对 |
| L2 | 模板内单块结构化（如单章 SDD、GWT 初稿） |
| L3 | 同需求链式多产物；全量 SDD 复核 |
| L4 | L3 + 机读校验留痕 + 追溯与模板库联动 |

**禁止**：AI 替代 SDD 签署、PRD 冻结决策、测试/发布放行。

### 4.4 条款等级

**必须** / **禁止** / **应当**（偏离须 PO 书面同意） / **建议** —— 释义见全文版 **§0.3**。

## 五、落地执行检查单（精简版，每迭代可打勾）

### 5.1 对接

- [ ] 干系人台账为当前版本  
- [ ] 本轮共创或确认会**纪要已归档**，行动项含责任人、日期  
- [ ] 无「仅群聊」的契约性承诺  

### 5.2 门禁

- [ ] 当前处于六步中哪一步、**唯一产物**与《执行版》一致  
- [ ] 若已需求确认：**双签**可查证  
- [ ] 若已开发：**SDD 签署**可查证  
- [ ] 若已联调/提测：**PRD 冻结 + 字段映射表**可查证  

### 5.3 AI

- [ ] 本项目/需求 **L 阶次已声明**；无未经 PO 同意的越阶  
- [ ] AI 产出已过 Gate；异常已留痕或入台账  

### 5.4 评审与台账

- [ ] 评审材料满足该次评审类型准入（见全文版 **§7.3.1**）  
- [ ] 台账开放项责任人与时限清晰

## 六、度量与汇报节奏（精简 KR）

| 维度 | 建议指标 |
|------|----------|
| 对接 | 书面确认率；行动项按期关闭率；需求变更率（先有基线） |
| 流程质量 | SDD 一次复核通过率；PRD 冻结后字段变更次数 |
| AI | L 阶次合规率；AI 产出 Gate 一次通过率；兜底次数 |
| 专项 | 核心方向可复用资产数；核心方向项目占比 |

**汇报节奏**：双月或季度用 **一页纸 + 本文 §二、§六**；落地证明用 **§三、§五**。
