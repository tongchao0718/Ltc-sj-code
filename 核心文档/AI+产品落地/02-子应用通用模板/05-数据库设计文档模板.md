- 状态字段统一使用标准编码。
- 关键链路保留 `trace_id` 便于问题追踪。
- 异步任务需持久化任务状态。
- 权限数据遵循最小权限原则。

## 6. 变更管理

- 新增/变更库表需同步更新 PRD 的“物理模型”和“接口契约”。
- 字段冻结后变更需走评审变更单。
- 模式冲突时，以评审结论为最终依据。

## 8. SQL清单（全量DDL，强制）

```sql
-- 方言：MySQL 8.x
CREATE TABLE protocol_template (
  protocol_template_id BIGINT PRIMARY KEY AUTO_INCREMENT,
  template_code VARCHAR(64) NOT NULL UNIQUE,
  template_name VARCHAR(128) NOT NULL,
  status VARCHAR(20),
  created_at DATETIME,
  updated_at DATETIME
);
```

> 要求：
> 1) 每个物理模型必须提供一段可执行建表 SQL（DDL），并注明数据库方言（MySQL/PostgreSQL 等）。  
> 2) `物理模型清单（全量）` 的字段明细必须与 `SQL清单` 一一对应（字段数量、类型、主键、必填一致）。  
> 3) 若字段清单与 DDL 不一致，视为门禁不通过（`blocked`）。
