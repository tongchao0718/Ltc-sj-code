# G2/G3自动门禁脚本契约模板

## 1. 目的

统一 G2（设计评审）与 G3（开发完成）自动门禁脚本输入输出，确保可接入 CI 并稳定阻断不合规提交。

## 2. 执行位置

- PR 检查流水线（必跑）
- 发布前流水线（必跑）

## 3. 输入约定

| 输入项 | 路径示例 | 必填 | 说明 |
|------|------|------|------|
| SDD JSON | `docs/子应用文档/<app-code>/01-需求与设计/SDD-v1.0.json` | 是 | 机读 SDD 主体 |
| 映射表 | `docs/子应用文档/<app-code>/01-需求与设计/页面-路由-接口-数据表映射表.md` | 是 | 关联完整性校验 |
| 门禁配置 | `docs/子应用文档/<app-code>/04-AI治理与审计/gate-config.json` | 否 | 可选参数 |

## 4. 输出约定

脚本输出必须为 JSON，字段如下：

```json
{
  "gate": "G2",
  "status": "passed",
  "failed_items": [],
  "warnings": [],
  "checked_at": "2026-04-28T16:50:00+08:00"
}
```

字段说明：
- `gate`：`G2` 或 `G3`
- `status`：`passed` / `failed` / `blocked`
- `failed_items`：失败项数组
- `warnings`：警告项数组
- `checked_at`：执行时间（ISO8601）

## 5. 阻断规则

1. `status=failed` 或 `status=blocked`，流水线必须阻断。
2. 输出中包含 `failed_items` 时，PR 不允许合并。
3. 发布前流水线失败时，发布流程必须终止并通知责任人。

## 6. 责任分工

- 规则维护：技术负责人
- 脚本维护：研发负责人
- 执行监督：项目经理
- 结果复核：产品/测试负责人
