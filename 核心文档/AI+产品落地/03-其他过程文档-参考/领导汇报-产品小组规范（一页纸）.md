## 结论（30 秒）

AIEP-DEV 采用 **AI+ 六步流程 + SDD/PRD 双真值 + Gate-1～4**，Agent 总入口为 Skill **`ai-full-process-design`**。界面须 **先做全量可交互原型**，再经客户 **G2-A（19 号文件）** 确认后 PRD 冻结，方可提测发布。

## 四件事（流程怎么走）

| # | 问题 | 答案 |
|---|------|------|
| 1 | 主线几步？ | 需求生成 → 需求确认 → **界面生成（100% UI）** → **界面确认（客户签字）** → 开发测试 → 发布 |
| 2 | 卡在哪？ | Gate-2（SDD 签署，步骤 2）· Gate-3（界面确认 + PRD 冻结，步骤 4）· Gate-4（发布） |
| 3 | 文档写哪？ | `AIEP-WEB/src/docs/子应用文档/{app}/` 四段目录 + `02-子应用通用模板/` |
| 4 | Agent 怎么跑？ | `npm run infer:process-step -- --app {app}` → 加载 Skill → 跑 G2/G3 脚本 |

## 三条红线

1. SDD 未签署 → 禁止全面开发与上线承诺  
2. PRD 未冻结 / 无 G2-A → 禁止联调提测  
3. 测试/CI/追溯链不完整 → 禁止发布
