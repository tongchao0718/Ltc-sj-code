**内容**：

尊敬的评委、各位观众，大家好！

今天我非常荣幸能够参加 QuickFix・AI 黑客松，向大家展示我们的参赛作品——**AI Coding-原型设计平台**。

我们的产品是一个基于 Vue 3 的多子应用架构系统框架，专为设计团队打造，让产品经理成员快速进行 AI 设计，快速交付。通过 AI Coding 编程工具（cursor、trae）快速进行界面设计。该框架支持多项目应用的独立设计、风格统一、独立打包、离线访问以及设计文档输出，能够在复杂场景以及公司现有工具（图颜、九功智能体）无法满足需求时，快速响应需求进行界面设计，并高效完成确认与交付。

## 2. 问题定义（2分钟）

### 痛点分析

**当前设计团队面临的核心挑战**：

1. **复杂场景需求**
   - 公司现有工具（图颜、九功智能体）在复杂场景下能力有限
   - 多个设计项目之间缺乏统一管理，资源共享困难
   - 设计成本高，人工干预多，容易出错

2. **设计周期冗长**
   - 复杂场景下原型设计周期长，无法快速响应业务需求
   - 缺乏标准化的设计规范，设计质量参差不齐
   - 设计团队协作困难，沟通成本高

3. **代码复用率低**
   - 前端代码复用率低，重复工作多
   - 缺乏标准化的组件库，设计一致性难以保证
   - 复杂场景下需要更高效的解决方案

4. **交付流程繁琐**
   - 需要手动编写设计文档，工作量大
   - 文档和设计难以保持同步
   - 缺乏智能化的设计辅助和决策支持

### 市场需求

根据 AI+产品落地方案的调研数据：

- 设计团队在复杂场景下需要更高效的解决方案

## 4. 解决方案（3分钟）

### 产品架构

**技术架构**：

- **前端框架**：Vue 3.5.31
- **构建工具**：Vite 6.4.1
- **路由管理**：Vue Router 4.6.4
- **状态管理**：Pinia
- **UI组件库**：Arco Design Vue 2.57.0
- **数据可视化**：ECharts 5.6.0
- **HTTP客户端**：Axios 1.14.0
- **国际化**：Vue I18n 9.14.5

**多子应用架构**：

- 主系统提供统一入口和管理界面
- 子应用独立设计、测试和部署
- 子应用支持本地直接打开 HTML 文件运行
- 主系统和子应用可以独立部署或集成部署

### 核心功能

1. **多子应用管理**：统一管理多个独立设计项目，支持添加、删除、更新和监控
2. **快速原型设计**：AI Coding 编程工具驱动的快速原型设计，辅助设计人员快速交付
3. **设计文档输出**：自动生成设计文档，简化交付流程
4. **风格统一**：统一的设计规范和组件库，确保界面风格一致性
5. **本地访问**：子应用支持本地直接打开 HTML 文件运行，无需服务器
6. **一键编译部署**：通过代码包进行一键编译（首次部署）
7. **一键更新**：通过更新包一个命令完成系统更新，支持版本回滚功能
8. **项目审查**：完整的项目审查工作流程
9. **主题切换**：支持浅色和深色主题
10. **国际化支持**：多语言切换
11. **数据可视化**：集成 ECharts 提供数据可视化能力

### 设计规范

- 统一的颜色系统（基于 Arco Design）
- 标准化的排版系统
- 组件样式规范
- 布局结构标准
- 动画效果规范

## 6. 关键交互示例（2分钟）

### 主要交互流程

**1. 应用管理**

- 用户登录平台，在应用中心查看所有设计项目
- 点击应用卡片进入项目详情页面
- 进行项目的管理和配置

**2. 快速原型设计**

- 设计人员使用 AI Coding 编程工具（cursor、trae）快速进行界面设计
- 平台提供标准化框架和组件库
- 加速设计过程，提高设计质量

**3. 设计文档输出**

- 设计完成后，平台自动生成设计文档
- 设计人员可以查看和编辑文档
- 一键交付，简化交付流程

**4. 项目审查**

- 审查人员进入项目审查中心
- 查看待审查项目列表
- 点击项目进入审查工作台，进行项目审查并提交审查结果

**5. 智能推荐**

- 用户在应用中心浏览设计项目
- 系统根据用户角色和历史行为
- 推荐相关设计模板和组件，提高设计效率

**6. 一键更新**

- 用户通过设置页面检查更新
- 查看更新内容列表
- 点击一键更新按钮
- 系统自动完成更新流程
- 显示更新进度和重启按钮
