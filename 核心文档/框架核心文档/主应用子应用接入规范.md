# 主应用子应用接入规范

> **适用范围**：在 AIEP-WEB 中新增或变更 Web 子应用时，除子应用独立工程外，**必须**同步更新主应用注册与嵌入配置。  
> **单一注册真值**：`AIEP-WEB/src/config/subApps.js`  
> **机读校验**：`npm run validate:sub-app-registry`

---

## 1. 原则

1. **双通道交付**：每个 Web 子应用须同时支持  
   - **独立打包访问**（`dev:{app}` / `build:{app}` → `dist/{folder}/`）  
   - **主应用嵌入访问**（`npm run dev` → 应用中心 / `/前缀/...`）
2. **注册表驱动**：应用中心卡片、首页应用数/页面数**禁止**在多处硬编码，统一读 `subApps.js`。
3. **门禁绑定**：SDD `gate-config.json` 中 `g2_rules.require_main_app_registry: true` 时，G2 校验主应用接入完整性。

---

## 2. 必改文件清单（Checklist）

| 序号 | 文件 | 动作 |
|------|------|------|
| 1 | `src/config/subApps.js` | 新增条目：`appCode`（与 SDD/gate 一致）、`folder`、`to`（默认入口）、`pageCount` |
| 2 | `src/router/index.js` | 增加顶层 `path: '/<前缀>'` 及子路由（与壳层 `*App.vue` 一致） |
| 3 | `src/App.vue` | `SUB_APP_EMBED_PREFIXES` 增加 `'/<前缀>'` |
| 4 | `src/apps/<folder>/<App>.vue` | 侧栏 `router-link` 使用 `base` 前缀（嵌入 vs 独立 Hash 双模式） |
| 5 | `package.json`（仓库根） | `dev:{folder}`、`build:{folder}` |
| 6 | `build/vite.<folder>.config.js` + `build/<folder>.html` | 独立打包入口 |
| 7 | SDD `acceptance` | 含 **AT-22**（或等价）：主应用嵌入 + `validate:sub-app-registry` |
| 8 | G3 任务表 | 含 **FE-INTEG-01** 主应用接入任务 |

**无需再改**（已接注册表）：

- `src/views/AppCenter.vue` — 读取 `subApps.js`
- `src/views/Dashboard.vue` — 读取 `getSubAppMetrics()`

## 3. `subApps.js` 字段约定

| 字段 | 必填 | 说明 |
|------|------|------|
| `id` | Y | 注册表内唯一键 |
| `appCode` | Y | 与 `gate-config.app_code`、SDD `app_code` 一致 |
| `name` | Y | 应用中心展示名 |
| `folder` | Y | `src/apps/` 下目录名 |
| `to` | Y | 主应用内默认入口（如 `/ai-smart-crm-admin/dashboard`） |
| `desc` / `icon` / `tag` | Y | 应用中心卡片文案 |
| `pageCount` | Y | 该子应用**叶子路由页**数量（用于首页「页面数量」） |

`MAIN_PAGE_COUNT` 固定为 **6**（dashboard、app-center、profile、review-center、help、about）。

## 4. 路由与嵌入

### 4.1 主应用路由

- 模式：`createWebHistory()`（History）
- 子应用顶层路径示例：`/ai-smart-crm-admin`
- 子路由使用**相对 path**（如 `dashboard`、`login`），避免写绝对 `/dashboard` 跳出前缀

### 4.2 子应用壳层 `base`

```javascript
const base = computed(() => {
  const isStandalone =
    window.location.hash.startsWith('#/') || window.location.protocol === 'file:'
  return isStandalone ? '' : '/<主应用前缀>'
})
```

菜单与编程式导航须使用 `base`（推荐 `composables/useCrmBase.js` 一类封装）。

### 4.3 嵌入布局

`App.vue` 对 `SUB_APP_EMBED_PREFIXES` 匹配路径时：

- `app-main--embed`：主区不滚动，高度交给子应用双栏布局
- 子应用根节点使用 `class="app-layout"` 以承接 flex 高度链

## 5. 验收与门禁

### 5.1 命令

```bash
# 全量：注册表 ↔ router ↔ App.vue ↔ Dashboard/AppCenter
npm run validate:sub-app-registry

# 单子应用（G2 按 app_code 检查）
npm run validate:sub-app-registry -- --app ai-smart-crm

# G2（gate-config 开启 require_main_app_registry 时含注册校验）
npm run validate:sdd -- --app ai-smart-crm --gate G2
```

### 5.2 SDD 验收建议

| ID | Given | When | Then |
|----|-------|------|------|
| AT-21 | 独立 build 成功 | 访问独立 HTML/ dev 端口 | 子应用功能页可用 |
| AT-22 | 主应用 dev + 注册表校验通过 | 应用中心或嵌入 URL | 顶栏+侧栏+目标页正常；首页应用数正确 |

## 6. 与 AI+产品流程的关系

| 流程步骤 | 主应用接入要求 |
|----------|----------------|
| 步骤3 界面生成 | Web 原型完成后完成注册表 + 路由；`validate:sub-app-registry` 通过 |
| 步骤5 开发测试 | G3 含 FE-INTEG-01；执行 AT-22 |
| G2 放行 | `require_main_app_registry: true` |

详见 `AI+产品全流程执行手册-执行版.md` 步骤3、步骤5。

---

## 7. 常见遗漏（反例）

| 反例 | 后果 |
|------|------|
| 只做 `build:xxx` 独立包 | 应用中心看不到；首页应用数不增加 |
| 只改 `AppCenter.vue` 硬编码 | 与 Dashboard 指标不一致 |
| `router.push('/dashboard')` 未加前缀 | 嵌入模式跳到主应用首页 |
| AT 只验独立构建 | G2 通过但主应用不可达 |
