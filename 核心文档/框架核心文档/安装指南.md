# 安装指南

## 1. 必要环境

在开始创建和运行项目之前，需要安装以下必要环境：

| 环境 | 版本要求 | 用途 |
|------|----------|------|
| Node.js | **18.0 或更高版本**（与 Vite 6 / 本仓库一致） | 运行 Vue 项目和构建工具 |
| npm | 8.0 或更高版本 | 包管理工具 |
| Git | 任意最新版本 | 版本控制 |

## 2. 环境安装方法

### 2.1 Node.js 安装

#### Windows 系统

1. **下载安装包**：
   - 访问 [Node.js 官网](https://nodejs.org/zh-cn/download/)
   - 下载 Windows 安装包（建议选择 LTS 版本）

2. **运行安装程序**：
   - 双击安装包，按照提示完成安装
   - 确保勾选 "Add to PATH" 选项

3. **验证安装**：
   ```bash
   node -v
   npm -v
   ```

#### macOS 系统

1. **使用 Homebrew**：
   ```bash
   brew install node
   ```

2. **或下载安装包**：
   - 访问 [Node.js 官网](https://nodejs.org/zh-cn/download/)
   - 下载 macOS 安装包并运行

3. **验证安装**：
   ```bash
   node -v
   npm -v
   ```

#### Linux 系统

1. **使用包管理器**：
   ```bash
   # Ubuntu/Debian
   sudo apt update
   sudo apt install nodejs npm

   # CentOS/RHEL
   sudo yum install nodejs npm

   # Fedora
   sudo dnf install nodejs npm
   ```

2. **或使用 nvm**：
   ```bash
   curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash
   nvm install --lts
   ```

3. **验证安装**：
   ```bash
   node -v
   npm -v
   ```

### 2.2 Git 安装

#### Windows 系统

1. **下载安装包**：
   - 访问 [Git 官网](https://git-scm.com/download/win)
   - 下载 Windows 安装包

2. **运行安装程序**：
   - 双击安装包，按照提示完成安装
   - 确保勾选 "Add Git to PATH" 选项

3. **验证安装**：
   ```bash
   git --version
   ```

#### macOS 系统

1. **使用 Homebrew**：
   ```bash
   brew install git
   ```

2. **或下载安装包**：
   - 访问 [Git 官网](https://git-scm.com/download/mac)
   - 下载 macOS 安装包并运行

3. **验证安装**：
   ```bash
   git --version
   ```

#### Linux 系统

1. **使用包管理器**：
   ```bash
   # Ubuntu/Debian
   sudo apt update
   sudo apt install git

   # CentOS/RHEL
   sudo yum install git

   # Fedora
   sudo dnf install git
   ```

2. **验证安装**：
   ```bash
   git --version
   ```

## 3. 一键安装脚本

### 3.1 Windows 一键安装脚本

创建一个名为 `install-env.ps1` 的 PowerShell 脚本：

```powershell
# 下载并安装 Node.js LTS
Write-Host "正在下载 Node.js LTS..."
Invoke-WebRequest -Uri "https://nodejs.org/dist/v20.11.1/node-v20.11.1-x64.msi" -OutFile "nodejs-installer.msi"

Write-Host "正在安装 Node.js..."
Start-Process msiexec.exe -ArgumentList "/i nodejs-installer.msi /quiet /qn /norestart" -Wait

# 下载并安装 Git
Write-Host "正在下载 Git..."
Invoke-WebRequest -Uri "https://github.com/git-for-windows/git/releases/download/v2.43.0.windows.1/Git-2.43.0-64-bit.exe" -OutFile "git-installer.exe"

Write-Host "正在安装 Git..."
Start-Process .\git-installer.exe -ArgumentList "/VERYSILENT /NORESTART /NOCANCEL /SP- /SUPPRESSMSGBOXES" -Wait

# 验证安装
Write-Host "验证 Node.js 安装..."
node -v
Write-Host "验证 npm 安装..."
npm -v
Write-Host "验证 Git 安装..."
git --version

# 清理安装文件
Remove-Item nodejs-installer.msi
Remove-Item git-installer.exe

Write-Host "环境安装完成！"
```

### 3.2 macOS 一键安装脚本

创建一个名为 `install-env.sh` 的 bash 脚本：

```bash
#!/bin/bash

# 安装 Homebrew（如果未安装）
if ! command -v brew &> /dev/null; then
    echo "正在安装 Homebrew..."
    /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
fi

# 安装 Node.js 和 Git
echo "正在安装 Node.js 和 Git..."
brew install node git

# 验证安装
echo "验证 Node.js 安装..."
node -v
echo "验证 npm 安装..."
npm -v
echo "验证 Git 安装..."
git --version

echo "环境安装完成！"
```

### 3.3 Linux 一键安装脚本

创建一个名为 `install-env.sh` 的 bash 脚本：

```bash
#!/bin/bash

# 检测 Node.js 是否已安装
if command -v node &> /dev/null; then
    echo "Node.js 已安装: $(node -v)"
else
    echo "Node.js 未安装，开始安装..."
    # 更新包管理器
    echo "正在更新包管理器..."
    sudo apt update || sudo yum update || sudo dnf update
    
    # 安装 Node.js
    if command -v apt &> /dev/null; then
        sudo apt install -y nodejs npm
    elif command -v yum &> /dev/null; then
        sudo yum install -y nodejs npm
    elif command -v dnf &> /dev/null; then
        sudo dnf install -y nodejs npm
    fi
fi

# 检测 Git 是否已安装
if command -v git &> /dev/null; then
    echo "Git 已安装: $(git --version)"
else
    echo "Git 未安装，开始安装..."
    # 安装 Git
    if command -v apt &> /dev/null; then
        sudo apt install -y git
    elif command -v yum &> /dev/null; then
        sudo yum install -y git
    elif command -v dnf &> /dev/null; then
        sudo dnf install -y git
    fi
fi

# 验证安装
echo "验证 Node.js 安装..."
try {
    node -v
} catch {
    echo "Node.js 安装失败"
}

echo "验证 npm 安装..."
try {
    npm -v
} catch {
    echo "npm 安装失败"
}

echo "验证 Git 安装..."
try {
    git --version
} catch {
    echo "Git 安装失败"
}

echo "环境安装完成！"
```

## 6. 安装步骤

### 6.1 自动安装（推荐）

项目提供了一个自动安装脚本，可以检测并安装所有缺失的依赖：

```bash
# 运行自动安装脚本
npm run install-deps
```

该脚本会自动检测并安装所有缺失的核心依赖和开发依赖，无需手动执行多个安装命令。

### 6.2 手动安装

如果需要手动安装依赖，可以执行以下步骤：

1. **打开命令行终端**：在项目根目录下打开命令行终端
2. **执行安装命令**：根据需要执行上述安装命令
3. **验证安装**：安装完成后，运行 `npm list` 检查依赖是否正确安装
4. **启动项目**：安装完成后，可以运行 `npm run dev` 启动开发服务器

## 7. 注意事项

- 确保 Node.js 版本为 18.0 或更高版本
- 确保 npm 版本为 8.0 或更高版本
- 如果安装过程中遇到网络问题，可以尝试使用 npm 镜像源
- 安装完成后，建议运行 `npm run build` 测试构建是否正常

## 8. 版本管理

### 8.1 更新版本号

当系统版本更新时，需要同步更新以下文件中的版本号：

1. **package.json**：更新 `version` 字段
2. **产品设计文档.md**：更新"关于我们"部分的版本号
3. **docs/子应用文档/sample-app/产品设计文档.md**：更新相关版本号
4. **框架基线.md**：更新基线版本标识

### 8.2 版本更新流程

1. 更新 `package.json` 中的版本号
2. 同步更新产品设计文档中的版本号
3. 运行 `npm install` 确保依赖正确安装
4. 运行 `npm run build` 测试构建是否正常
5. 提交代码并创建版本标签

### 8.3 版本号规范

遵循语义化版本规范（Semantic Versioning）：
- 主版本号（MAJOR）：不兼容的 API 修改
- 次版本号（MINOR）：向下兼容的功能性新增
- 修订号（PATCH）：向下兼容的问题修正

例如：`1.2.0` 表示主版本 1，次版本 2，修订号 0

## 9. 故障排除

### 9.1 依赖冲突

如果遇到依赖冲突，可以尝试：

```bash
# 清除 npm 缓存
npm cache clean --force

# 删除 node_modules 目录和 package-lock.json 文件
rm -rf node_modules package-lock.json

# 重新安装依赖
npm install
```

### 9.2 安装失败

如果安装失败，可以尝试：

```bash
# 使用 npm 镜像源
npm install --registry=https://registry.npmmirror.com

# 或使用 yarn
npm install -g yarn
yarn install
```

### 9.3 Node.js 安装失败

- 检查网络连接
- 尝试使用不同版本的 Node.js
- 手动下载安装包并运行

### 9.4 Git 安装失败

- 检查网络连接
- 尝试使用不同版本的 Git
- 手动下载安装包并运行

### 9.5 环境变量问题

- 确保安装过程中勾选了 "Add to PATH" 选项
- 重启终端或计算机
- 手动添加环境变量

## 10. 总结

通过本指南，你可以轻松安装项目所需的系统环境和依赖包，为项目的创建和运行做好准备。根据项目需求，你可以选择安装相应的可选依赖，以增强项目的功能和质量。

本指南还提供了版本管理的规范和流程，确保在系统版本更新时能够正确同步更新相关文档，保持文档与代码的一致性。
