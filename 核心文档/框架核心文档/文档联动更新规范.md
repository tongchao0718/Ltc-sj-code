# 文档联动更新规范

## 1. 目的

当任一核心文档发生变更时，确保关联文档同步更新，避免“单点更新、全局失真”。

## 2. 适用范围

- `核心文档/框架核心文档/` 下所有规范类文档
- `核心文档/AI+产品落地/` 下执行手册与模板文档
- `AIEP-WEB/src/skills/` 下流程与门禁技能文档

## 3. 联动更新总原则（强制）

1. 任一文档变更后，必须执行“影响分析 -> 联动更新 -> 门禁检查”三步。  
2. 若关联文档未同步完成，状态标记为 `blocked`，不得进入下一关卡。  
3. 文档更新优先级：**流程规则 > 模板规则 > 技能规则 > 示例说明**。  
4. 若规则冲突，以最新版本的执行手册与框架核心文档为准。  

## 4. 联动矩阵（变更 -> 必须同步）

| 变更文档类型 | 必须同步更新的文档 | 最小完成标准 |
|------|------|------|
| 执行流程/关卡规则（如 G2/G3） | `AI+产品全流程执行手册.md`、相关 `SKILL.md`、`SKILLS-DIRECTORY.md` | 流程术语一致、门禁条件一致 |
| 模板结构（02-子应用通用模板） | 执行手册模板条款、各阶段 `SKILL.md` 的“模板调用要求” | 模板名、章节、必填项一致 |
| 需求/PRD/SDD 规则 | 执行手册、对应模板、`requirements-clarification`/`prd-design-generation` | 验收与映射规则一致 |
| UI/交互规范 | `系统设计规范.md`、执行手册 Define 阶段、`ui-generation` | 状态流与界面确认规则一致 |
| 菜单信息架构规范 | 需求说明书、产品设计文档、PRD设计评审文档、页面路由映射表、前端菜单实现 | 菜单名称不带编号；菜单层级不超过五级；文档与实现一致 |
| API/数据库规范 | `详细设计文档模板.md`、`数据库设计文档模板.md`、`code-development` | 字段映射与契约要求一致 |
| 测试/门禁规则 | `test-validation`、门禁模板（08/09/12）与执行手册 Validate 章节 | 放行标准与阻断条件一致 |
| 打包/构建规则 | `创建指南.md`、`主应用打包指南.md`、`子应用打包指南.md`、README | 命令、产物路径、部署说明一致 |

补充要求：
1. 以上任一“主变更文档”更新后，必须在同次提交中完成对应联动文档更新或明确标注“暂缓原因 + 计划完成时间”。  
2. 若未完成联动更新且无暂缓说明，门禁状态标记 `blocked`。  
3. 涉及 `系统设计规范-ant.md` 的变更，需同步核对 `系统设计规范.md` 是否需要同源更新或标注差异说明。  

## 5. 执行步骤（每次文档变更必须执行）

1. **识别变更类型**：流程、模板、技能、技术规范。  
2. **定位关联清单**：按本规范第 4 节选取必更文档。  
3. **同步更新**：一次性提交主文档与关联文档。  
4. **一致性检查**：检索关键术语（关卡名、模板名、字段名）是否一致。  
5. **留痕归档**：在主文档版本记录写入“联动更新范围”。  

## 6. 联动检查清单（可复制）

```text
文档联动检查：
- [ ] 主文档已更新
- [ ] 关联文档已全部更新
- [ ] 术语与关卡名称一致（如 G2-A/G2-B）
- [ ] 菜单命名与层级一致（不带编号，最多五级）
- [ ] 模板文件名与路径一致
- [ ] Skills 与执行手册规则一致
- [ ] 版本记录已写联动范围
- [ ] 若有缺项，状态标记 blocked
```

## 7. 责任分工

- 主责：发起变更的角色（产品/研发/测试）  
- 复核：流程负责人或项目经理  
- 放行：关卡责任人（G2/G3/G4 对应负责人）  

## 8. 生效规则

本规范发布后，对后续所有流程规则、模板规则、skills 规则变更即时生效。  
历史存量文档按最近一次迭代逐步补齐，不要求一次性全量重写。
