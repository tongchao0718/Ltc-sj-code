# AIEP 新手入门

> **适用对象**：首次使用 AIEP-DEV 做子应用 AI+ 全流程交付的开发者。  
> **模型说明**：AIEP **不规定** LLM 型号；使用 **Cursor / TRAE 订阅自带模型** 即可，无需维护「推荐模型表」。  
> **详细环境**：`安装指南.md` · **执行 SOP**：`核心文档/执行说明.txt`

---

## 第 1 章：选择 Agent 工具

AIEP 通过 **Cursor Rules + Skills** 驱动 Agent，不绑定单一 IDE。团队常用：

| 工具 | 特点 | 与 AIEP 的配合 |
|------|------|----------------|
| **Cursor** | 模型齐全、Agent/Plan 模式成熟 | 打开仓库根目录；Rules 自动加载（`agent-skills-auto-exec.mdc`） |
| **TRAE** | 中文友好、国内可用 | `npm install` 后 Skills 同步至 `.trae/skills/`，无需手动 `/skill-name` |

**完成标准**

- [ ] 已安装 Cursor 或 TRAE 之一
- [ ] 知道 AIEP 流程由 **Skill 自动加载** 驱动，勿要求同事手工 `sync:skills` 或输入 `/ai-full-process-design`

---

## 第 2 章：打开正确项目

常见失败原因：只在 `AIEP-WEB/` 子目录打开 Agent，或打开了错误的 fork。

1. 用 Agent **打开仓库根目录**（含根 `package.json`、`AIEP-WEB/`、`核心文档/`）。
2. 在仓库根执行一次：

```bash
npm install
```

`postinstall` 会自动同步 Skills（Cursor 全局 + Trae `.trae/skills/`）。

3. 复制下面 Prompt 发给 Agent，确认路径：

```text
请帮我确认：你现在打开的项目，是不是 AIEP-DEV 仓库根目录（应包含根 package.json、AIEP-WEB/、核心文档/）。

不要修改任何文件，只检查目录结构。

若已在正确仓库，请只回复：已打开正确项目

若不是，请说明当前目录并提醒我在仓库根重新打开。
```

**完成标准**

- [ ] Agent 回复「已打开正确项目」
- [ ] 根目录 `npm install` 已成功（含 postinstall sync）

## 第 3 章：立项并推进子应用

新建子应用时，对 Agent 说：

```text
推进 {app-code}，步骤 0 立项
```

或明确参数：

```text
用 scaffold-sub-app 新建子应用，folder={folder}，title={展示名}，spec=arco
```

Agent 应自动 Read `scaffold-sub-app` Skill，运行 `npm run scaffold:sub-app`，并初始化四段文档目录。

已有子应用时：

```bash
npm run infer:process-step -- --app {app-code} --json
```

对 Agent 说：**「推进 {app-code}」** — 总编排 `ai-full-process-design` 会按推断步骤加载对应阶段 Skill。

**完成标准**

- [ ] `AIEP-WEB/src/apps/{folder}/` 与 `AIEP-WEB/src/docs/子应用文档/{app-code}/` 四段目录存在
- [ ] `npm run validate:sub-app-registry -- --app {app-code}` 通过
- [ ] `04-AI治理与审计/00-项目记忆.md` 已写入 `platform_type`

---

## 第 4 章：步骤 3 界面与资源库

步骤 3 目标：**全量界面可演示**（非单页 demo）。Agent 自动链：

`sub-app-resources` → `ui-generation` → `prd-design-generation`（step3）

协作技巧（与 IDE 无关）：

- **一任务一新对话**：改列表就只聊列表，避免上下文串台
- **多带截图**：参考图、验收反馈、Bug 截图比长文字更高效
- **Plan / Spec 再执行**：步骤 2 SDD、步骤 3 大批量界面前，先让 Agent 出计划再改文件

空子应用 `_resources/README.md` 顶部有 **起步提示**；规范见 `子应用资源库规范.md`。

**完成标准**

- [ ] `_resources/theme.json`、`components-manifest.json` 已按形态初始化
- [ ] 步骤 3 页面清单与 SDD/PRD 映射一致
- [ ] 可选：已跑 `design-review-pre-g2a` 输出 `G2-A预审报告.md`

---

## 第 5 章：Gate 与客户确认

步骤 4 为 **硬门禁**：无 PO/客户 G2-A「通过」→ Agent 状态 **blocked**，禁止冻结 PRD。

| 关卡 | 关键产物 |
|------|----------|
| Gate-1 | 需求确认、SDD 草案 |
| Gate-2 | `SDD-v*.json` + `validate:sdd --gate G2` |
| G2-A | 客户界面确认（步骤 4） |
| Gate-3 / G2-B | PRD 冻结、技术补齐 |

对 Agent 说：**「推进 {app-code}，进入步骤 4 客户确认」** 前，确认 G2-A 预审与全量界面已完成。

**完成标准**

- [ ] 理解步骤 4 须人工/客户签字，Agent 不可代签
- [ ] 知道 PRD 冻结与 SDD 契约分离（双真值）
- [ ] 后续步骤 5–6 见 `AI+产品全流程执行手册-执行版.md`

## 附录：Agent 协作习惯

| 习惯 | 说明 |
|------|------|
| 说「推进 {app-code}」 | 触发总编排与步骤 Skill 自动 Read |
| 说「导出子应用 {app-code}」 | 触发 `export-import-sub-app`，打源码交接 bundle（见《子应用打包指南》§9.0） |
| 说「导入子应用 {app-code}」 | 触发 `import:sub-app`；未给路径时 Agent 在 `dist/sub-app-bundles/` 推断最新 bundle |
| 说「打包子应用 {app-code}」 | 触发 `pack-sub-app`，产出静态 `dist/{folder}/`（**不是**源码导出） |
| 勿手工 sync Skills | `npm install` / 更新包已含 postinstall |
| 勿用 PRD Skill 代替 SDD | 步骤 2 必须用 `sdd-generation` |
| 子应用验证 Prompt | 见 `AIEP-WEB/src/skills/scaffold-sub-app/references/project-verify-prompt.md` |

### 交接说法速查（导出 vs 打包 vs 框架更新）

| 你要的结果 | 对 Agent 怎么说 |
|------------|-----------------|
| 同事继续改代码 | 「导出子应用 xxx（源码）」 |
| 客户只看页面 | 「打包子应用 xxx」或「导出静态页」 |
| 同事框架太旧 | 「**生成更新分发包**」（维护方）或「**更新框架** / 安装 update.js」（用户方） |
| 只交一个子应用 | 「导出子应用 xxx」→ 再「导入子应用 xxx」 |
