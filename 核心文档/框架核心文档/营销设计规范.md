## 一、设计理念

营销 2.0 Web 端 UE 规范面向电网营销类业务系统，强调：

- **统一**：统一画板尺寸、布局结构、组件形态，降低沟通与理解成本。
- **可读**：以 14px 为主字体，层级清晰，表格与表单信息密度适中。
- **可扩展**：支持多主题换肤（蓝/绿/紫/橙等），头部导航色可配置。
- **可落地**：与 Element UI / 图颜组件体系对齐，便于研发还原。

## 三、设计基准（分辨率）

为减少沟通成本，**统一以 1920×1080 为基准画板**设计；该分辨率在电网系统 PC 端最为普遍。


| 分辨率           | 说明        |
| ------------- | --------- |
| **1920×1080** | 基准设计稿（推荐） |
| 1440×900      | 常见适配目标    |
| 1366×768      | 最小桌面适配参考  |


> 设计说明：尽可能统一画板尺寸；开发阶段以弹性布局 + 最小宽度约束适配，避免固定像素硬编码导致小屏溢出。

## 四、布局规范

### 1. 典型后台结构

以 1920×1080、营销 2.0 默认布局为例：


| 区域       | 尺寸        | 说明                    |
| -------- | --------- | --------------------- |
| 左侧导航（展开） | **235px** | 点击导航按钮可收放             |
| 左侧导航（收缩） | **54px**  | 仅图标，需保留 Tooltip       |
| 顶部栏      | **64px**  | 产品 LOGO + 产品名称 + 全局操作 |
| 页签栏      | **36px**  | 多页签切换区域               |
| 内容区      | 自适应       | 右侧主工作区                |


### 2. 尺寸计算公式

- 顶部栏高度：`48 + 8n`（营销 2.0 取 **64px**）
- 侧边栏宽度：`200 + 8n`（营销 2.0 展开取 **235px**）

### 3. 顶栏与侧栏交互

- 左上角放置产品 LOGO 与产品名称。
- 侧栏收放通过导航按钮控制；收缩后宽度 54px，展开后 235px。
- 侧栏文字：14px regular；inactive `#A3AFBC`，active `#3A96FE`。
- 侧栏分割线：`#3A96FE` 40% 透明度 1px；选中项背景 `#ecf5ff`。

### 4. 内容区留白

- 内容区域外间距及相邻布局容器间距：**24px**
- 布局容器内边距及标题内边距：左右 **24px**，上下 **20px**

### 5. 侧栏垂直起始位置

- 左侧导航**白色面板整体**（含背景与边框）与页签栏下缘保持 **24px** 间距，露出页面背景 `#F5F7FA`。
- 禁止仅在菜单项内部增加 `padding-top` 而侧栏面板仍顶到页签栏；应通过侧栏容器 `margin-top: 24px`（并相应 `height: calc(100% - 24px)`）实现**区域整体下移**。

### 6. 工作区栅格与留白（侧栏 ↔ 内容）

`md-body` 工作区采用 **Flex + gap** 统一控制留白，Token：`--yx-spacing-page: 24px`。

| 位置 | 规格 | 实现 |
|------|------|------|
| 页面左/右/下外缘 | **24px** | `md-body` 的 `padding-left/right/bottom` |
| **侧栏与内容区之间** | **24px**（红框位） | `md-body` 的 `gap: 24px`；**禁止**用 `md-main` 的 `padding-left` 叠加替代 |
| 侧栏、内容区顶对齐 | 距页签栏 **24px** | 侧栏与 `md-main` 均 `margin-top: 24px` |
| 内容卡片内边距 | 上 **20px**、左右下 **24px** | `md-panel` |
| **列表页：查询卡片 ↔ 列表卡片** | **16px**（YXUI 间距 `#s3` 标注） | `md-list-page` 的 `gap: 16px` |
| 同页其他独立模块块间距 | **24px** | 页面级模块之间（非查询/列表双卡片） |

> **易错点 1**：侧栏与内容区用 `gap: 24px`，与列表内双卡片 **16px** 是两级间距，不可混用。  
> **易错点 2**：若同时设置 `gap: 24px` 与 `md-main { padding-left: 24px }`，侧栏与内容区间距会变成 **48px**。

## 五、间距与栅格

### 1. 基本单位

主流桌面分辨率在竖直与水平方向均可被 **8** 整除，栅格基本单位取 **8**；日常间距优先选用 **4 的倍数**：`4 / 8 / 12 / 16 / 20 / 24 / 32`。

### 2. 常用间距


| 场景 | 间距 |
|------|------|
| 页面/模块外间距 | 24px |
| **列表页查询卡片与列表卡片之间** | **16px**（YXUI `#s3` 标注；同页双容器） |
| 卡片/容器内边距（水平） | 24px |
| 卡片/容器内边距（垂直） | 20px |
| 详情列表行间距 | 8px |
| 组件间最小安全间距 | 8px |

## 六、字体与排版

### 1. 字体家族（按优先级）

**中文**

1. PingFang SC（macOS）
2. Microsoft YaHei（Windows）
3. Source Han Sans CN / 黑体（备选）

**英文 / 数字**

1. Helvetica Neue
2. Helvetica
3. Arial
4. Roboto（备选）

> 无特殊需求时优先使用系统默认字体，减少加载成本。

### 2. 字号层级


| 层级                 | 字号       | 用途                |
| ------------------ | -------- | ----------------- |
| 系统导航主标题 / 大数字      | 18px     | 导航主标题、重点金额        |
| 列表标题 / 二级标题 / 弹窗标题 | 16px     | 区块标题              |
| **正文（主字体）**        | **14px** | 默认文本、字段、菜单、页签、面包屑 |
| 次要 / 辅助 / 提示       | 12px     | 说明、分页辅助信息         |
| 分页等特殊              | 12px     | 分页符等少量场景          |


主字体设为 **14px**：兼顾可读性与表格信息密度；字号大于 14px 时，表格多列场景易显拥挤。

### 3. 行高


| 类型      | 行高                             |
| ------- | ------------------------------ |
| 单行标题    | 与字号差 4～6px（如 16px 字 → 22px 行高） |
| 多行标题    | **1.3** 倍                      |
| 多行正文    | **1.5** 倍                      |
| 表格/表单单行 | 22px（14px 正文）                  |


### 4. 字重

- Regular（400）：正文、菜单、表单
- Medium（500）：强调标签、侧栏选中
- Semibold（600）：区块小标题

## 八、换肤与主题

系统支持多套主题；**主色（按钮、链接、选中态等）随主题变化**，**头部导航背景色**可独立配置。


| 主题         | 系统主色      | 头部导航参考色               |
| ---------- | --------- | --------------------- |
| 蓝色（**默认**） | `#3A96FE` | `#124F9A`             |
| 绿色         | `#09958C` | `#095E59` / `#015A54` |
| 紫色         | `#5E5EED` | `#18186D` / `#3232BE` |
| 橙色         | `#FF9324` | `#5F3101` / `#DE7101` |


### 主题状态

每种主色均定义：

- Normal（默认）
- Hover（悬停）
- Click（点击）
- 渐变（头部装饰，按需）

### 背景灰度（用户可选）

辅助背景层级：`#FFFFFF` → `#F5F6FA` → `#EFF0F5` → `#EBECF1` → `#E6E7ED` → `#DBDCE1` → `#F0F0F0`

**落地要求**：子应用通过 CSS 变量或 ConfigProvider 切换主题，禁止在组件内硬编码主题色。

## 九、组件规范

### 1. 按钮


| 类型        | 规格                                                                                    |
| --------- | ------------------------------------------------------------------------------------- |
| 主按钮       | 背景 `#3A96FE`，文字 14px `#FFFFFF`；Hover `#1990FF`；Active `#0252A8`；Disabled 背景 `#A3CEFF` |
| 次按钮       | 边框 `#C6E2FF`，文字 14px `#3A96FE`；Hover 背景 `#EDF5FF`                                     |
| 文字 / 链接按钮 | 14px `#3A96FE`，无背景；标题容器内上下 padding 为 0                                                |
| 危险按钮      | 文字 / 背景 `#F5222D`，需二次确认场景                                                             |
| 尺寸        | 查询区 **28px**（`#s14`）；通用表单/弹窗 **32px**（`#s13`）；Loading / Disabled 状态必须可见              |


状态全集：`normal` / `hover` / `click` / `loading` / `disable`

### 2. 表格（YXUI `#s8`）

- 表头背景 `#FAFAFA`（列表页可用 `#EDF5FF`），正文 14px `#606266`
- **行高 36px**；单元格内边距 **10px**（四向）
- 表格模块内边距：上 **20px**、左右下 **24px**；模块外间距 **24px**
- 分割线 `#C6E2FF`；行 hover 背景 `#EDF5FF` / `#F3F8FF`
- 支持排序、筛选、分页、固定列
- 必须提供 loading / empty / error 态

### 3. 分页

- 翻页器单元尺寸：**22×22**
- 每页条数档位：**10 / 20 / 50**
- 总条数、跳转、页码需与查询条件联动（变更条件重置到第 1 页）

### 4. 标题

- 页面级标题 16px，容器内上下 padding 20px、左右 24px
- 与面包屑、操作按钮同一标题栏对齐

### 5. 面包屑

- 14px；链接 `#606266`；分隔符 `/` 使用 `#CFCFCF`
- 当前页 `#000000`；分割线 `#CFCFCF`

### 6. 标签页（YXUI `#s12`）


| 级别   | 高度       | 说明       |
| ---- | -------- | -------- |
| 壳层页签 | **36px** | 顶栏下多页签栏（`md-tabs`） |
| 内容一级 Tab | **40px** | 内容区主 Tab（卡片上方） |
| 二级标签 | 自适应      | 卡片内选项切换  |


样式变体（图颜组件对应）：

- 默认样式
- 按钮化（空心）
- 卡片化样式

标签较多时支持左右切换；宽度随字数自适应。

### 7. 输入框

- 高度 32px；边框 `#C6E2FF`；圆角 4px
- 聚焦边框 `#3A96FE`；错误 `#F56C6C` / `#FF4D4D`
- 支持前缀/后缀、带操作按钮、字数限制与过滤

### 8. 表单

- 标签 14px；必填项红色 `*`
- 校验错误：字段边框 + 文案同步
- 大表单分组 / 分步；提交失败滚动至首个错误项

#### 8.1 列表页查询区（强制）

列表页采用 **「查询卡片 + 列表卡片」** 双容器结构；两卡片垂直间距 **16px**（YXUI 间距 `#s3` 标注）。查询区须符合以下栅格与换行规则（参考 YXUI 稽查指标管理等列表页）：


| 项     | 规格                                                                |
| ----- | ----------------------------------------------------------------- |
| **查询卡片 ↔ 列表卡片** | 垂直间距 **16px**（`--yx-spacing-section`） |
| 区块标题  | 16px + 左侧 4×16px 主色竖条（`MdSectionTitle`）                           |
| 栅格列数  | **每行 3 列**，等宽 `repeat(3, minmax(0, 1fr))`                         |
| 表单项布局 | **标签左、控件右** 横向排列；标签 14px `#212121`，控件高度 **28px**（YXUI `#s14`）     |
| 列间距   | 水平 **20px**，行间距 **16px**（YXUI `#s14`）                           |
| 按钮间距  | 重置与查询按钮间距 **10px**                                              |
| 占位符   | 输入「请输入{标签}」；下拉「请选择」                                               |
| 操作按钮  | **重置**（次按钮）+ **查询**（主按钮）计入栅格单元，与条件项**同一 Grid 流**内排列，不得脱离栅格单独浮动于右侧 |
| 换行规则  | 条件项与按钮合计超过 3 个时按序换行；见下表                                           |


**按钮组换行与列位**（`N` = 查询条件个数）：


| 条件数 N   | 布局示意                                          |
| ------- | --------------------------------------------- |
| 0       | 第 1 行：`[ · · [重置][查询] ]` 按钮组独占一行、右对齐          |
| 1       | 第 1 行：`[ 条件1 · [重置][查询] ]` 按钮组占第 3 列          |
| 2       | 第 1 行：`[ 条件1 ] [ 条件2 ] [ [重置][查询] ]`          |
| 3       | 第 1 行三条件；第 2 行：`[ · · [重置][查询] ]` 按钮组独占一行、右对齐 |
| 4       | 第 1 行三条件；第 2 行：`[ 条件4 · [重置][查询] ]` 按钮组占第 3 列 |
| 5       | 第 1 行三条件；第 2 行：`[ 条件4 ] [ 条件5 ] [ [重置][查询] ]` |
| 3k（k≥1） | 每满 3 个条件换行；按钮组在下一行独占整行并右对齐                    |


**实现约定**（marketing-demo / 营销子应用）：

- 统一使用 `MdFilterForm` 组件，禁止各页自行编写查询区布局。
- 次按钮：边框 `#C6E2FF`、文字 `#3A96FE`；主按钮 `#3A96FE` 填充。
- 下拉框右侧展示 Chevron 箭头；聚焦边框 `#3A96FE`。

- 失败可重试；成功/失败色分别使用功能色

### 10. 选择器 / 树选择 / 日期时间

- 下拉面板背景 `#FFFFFF`，选项 12～14px `#606266`
- 选中/hover：`#EDF5FF` + 主色文字
- 树选择支持搜索、多选、懒加载
- 日期格式默认 `YYYY-MM-DD`；范围选择需明确起止

### 11. 下拉菜单

- 常规下拉 + **图标式固定悬浮**（始终在一屏固定位置，便于快捷操作）
- 项高度与 14px 文字行高一致；分割线 `#E5E7EE`

### 12. 单选 / 多选 / 开关 / 滑块

- 选中色 `#3A96FE`；边框未选中 `#DCDFE6` / `#C0C4CC`
- 开关 ON `#3A96FE`，OFF 轨道 `#DCDFE6`
- 滑块轨道浅色 `#C3DFFE`，填充 `#3A96FE`

### 13. 弹窗（Dialog / MessageBox）

- **弹出位置**：右侧内容区中间（不含左侧导航与顶栏）
- **蒙版**：`#000000` 50%
- **宽度**：按内容百分比，常见 **30% / 60% / 70% / 80% / 90%**
  - 单列（树选、短表单）：建议 30%
  - 多列（三列表单）：建议 70%
  - 内容过多：高度 80%，允许纵向滚动，尽量避免横向滚动
- **简单警告**：固定高度约 176px，无操作按钮
- **标题栏**：高度 **48px**，左右内边距 **24px**；标题 16px / 行高 22px `#000000`；关闭按钮 24×24px
- 底部按钮右对齐

### 14. 消息提示（Toast / Notification）


| 类型  | 背景参考                  | 用途         |
| --- | --------------------- | ---------- |
| 消息  | `#EDF5FF`             | 一般信息，可自动消失 |
| 成功  | `#F0FAEB` / `#E2F3D8` | 操作成功       |
| 警告  | `#FDF6EC` / `#FAECD8` | 权限、到期提醒    |
| 错误  | `#FEF1F0` / `#FDE2E2` | 失败、校验不通过   |


### 15. 流程图

- 节点填充浅色（如 `#F5FAFF`），边框 `#DDF3F7`
- 进行中 `#FFA853`，完成 `#59A843`，服务节点 `#2981C1`
- 连线清晰，文字 12～14px

### 16. 详情列表（Descriptions）

- 标签**右对齐**，非等宽字行
- 默认行高 **36px**，多行内容自适应
- 纵向行间距 **8px**

### 17. 滚动条

侧栏、主内容区、表格横向滚动、弹窗内容区等可滚动区域统一采用细窄圆角滚动条：


| 项         | 规格                                    |
| --------- | ------------------------------------- |
| 宽度 / 高度   | **6px**                               |
| 滑块默认      | `#C6E2FF`（或 `rgba(198,226,255,0.95)`） |
| 滑块 Hover  | 半透明主色 `rgba(58,150,254,0.72)`         |
| 滑块 Active | `#3A96FE`                             |
| 轨道        | 透明                                    |
| 圆角        | 胶囊形（`border-radius: 999px`）           |


Firefox 使用 `scrollbar-width: thin`；WebKit 使用 `::-webkit-scrollbar` 系列伪元素。禁止沿用浏览器默认粗滚动条。

## 十、交互规范

继承 AIEP 真实交互约束（与 `系统设计规范-ant.md` 第四节一致，营销子应用**强制**）：

1. 查询条件变更必须触发真实请求并重置分页。
2. 增删改、审批、导入导出须完整链路：确认 → loading → 结果回写 → 失败可恢复。
3. 页面状态：`loading / success / empty / error` 四类必须可见。
4. 防重复提交：按钮 loading / 禁用 + 幂等。
5. 权限：无权限不可见或禁用并说明原因。

### 营销域补充

- 列表页：筛选 + 表格 + 分页为标准三段式；筛选区遵循 §8.1 三列栅格与按钮换行规则。
- 详情页：Descriptions + 操作区；长内容分区折叠。
- 流程类：流程图仅作状态展示时，节点需可点击跳转对应业务单。
- Demo 顶栏产品名称示例：`YX-XT-DEMO`（可替换为正式系统名）。
- 表头背景：列表场景推荐 `#EDF5FF`（与 YXUI 列表页一致）；通用表格默认 `#FAFAFA` 见 §9.2，以产品/页面类型择一，同页不混用。

## 十三、验收清单（最小集）

- 基准分辨率按 1920×1080 设计，1366 宽度无横向滚动
- 布局：侧栏 235/54px、顶栏 64px、页签 36px；侧栏/内容距页签 **24px**；侧栏与内容区间 **gap 24px**（§4.6）
- 列表页：查询卡片与列表卡片间距 **16px**（§5.2、§8.1、YXUI `#s3`）；查询区三列栅格与按钮换行正确
- 主色 `#3A96FE` 及 hover/active/light 态完整
- 正文 14px、标题 16px、辅助 12px 层级清晰
- 间距使用 4 的倍数，页面外间距 24px
- 按钮、表格、分页、表单、弹窗、消息 7 类组件齐全
- 弹窗标题栏 48px、蒙版 50%；滚动条 6px 细窄样式（§17）
- 换肤至少支持蓝 / 绿主题切换
- 关键业务交互符合真实请求约束（非假数据切换）
- 与在线 UE 标注视觉抽检通过（内网打开 YXUI 链接对比）
