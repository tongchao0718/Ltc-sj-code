# 应用系统产品设计文档

## 1. 产品概述

### 1.1 产品名称

应用系统

### 1.2 产品定位

本产品是一个基于 Vue 3 的多子应用架构系统框架，旨在为企业级应用提供统一的管理平台，支持多个独立子应用的集成、管理和部署。

### 1.3 产品价值

- **统一管理**：为多个子应用提供统一的入口和管理界面
- **独立部署**：每个子应用可以独立打包和部署，减少耦合
- **灵活扩展**：支持快速添加新的子应用，满足业务需求的快速变化
- **本地访问**：子应用支持本地直接打开 HTML 文件运行，无需服务器
- **标准化**：提供统一的开发规范和打包配置，确保开发效率和质量
- **完整功能**：集成了仪表盘、应用中心、项目审查等核心功能
- **丰富的子应用**：示例应用包含仪表盘、数据可视化、列表页、表单页等完整功能
- **现代化设计**：采用 Arco Design 组件库，支持深色/浅色主题切换
- **便捷更新**：提供一键更新和版本回滚功能，简化系统维护
- **状态管理**：集成 Pinia 状态管理库，支持跨组件数据共享
- **国际化支持**：集成 Vue I18n，支持多语言切换
- **数据可视化**：集成 ECharts，提供丰富的图表组件

## 2. 系统架构

### 2.1 技术栈

- **前端框架**：Vue 3
- **构建工具**：Vite
- **路由**：Vue Router 4
- **状态管理**：Pinia
- **UI 组件库**：Arco Design
- **数据可视化**：ECharts
- **HTTP 客户端**：Axios
- **国际化**：Vue I18n
- **样式**：原生 CSS + CSS 变量 + Less
- **测试框架**：Vitest

#### 2.2.2 核心流程

1. **主系统启动**：加载主系统框架，显示应用中心
2. **子应用访问**：从应用中心或导航栏进入子应用
3. **子应用独立运行**：直接打开子应用的 HTML 文件运行
4. **子应用开发**：单独启动子应用的开发服务器
5. **子应用打包**：独立打包子应用为静态文件
6. **系统更新**：运行一键更新脚本，自动更新系统到最新版本
7. **版本回滚**：运行版本回滚脚本，恢复到之前的版本

## 3. 功能模块

### 3.1 主系统框架

#### 3.1.1 应用中心

- **功能**：展示所有子应用的九宫格界面
- **布局**：3x3 网格布局，每个子应用显示为卡片
- **交互**：点击卡片进入对应子应用
- **设计**：响应式设计，支持移动端适配

#### 3.1.2 仪表盘

- **功能**：展示系统概览和快速访问
- **内容**：系统状态、最近访问的应用、快速操作按钮
- **布局**：卡片式布局，清晰展示关键信息
- **交互**：点击快速操作按钮进入对应功能
- **设计**：响应式设计，支持移动端适配

#### 3.1.3 导航栏

- **功能**：提供系统级导航
- **内容**：系统名称、应用中心链接、用户信息（个人中心下拉菜单）
- **设计**：存总系统设计规范、简洁现代，高度55px，响应式设计
- **布局**：应用名称在左侧、导航按钮在右侧，多个导航按钮横向展示，左右留白50px
- **交互**：滚动时导航栏有阴影和背景变化效果，个人中心有下拉菜单，鼠标移出下拉菜单，下拉菜单自动关闭

#### 3.1.4 个人中心

- **功能**：提供使用帮助、关于我们、项目审查等功能
- **内容**：
  - **使用帮助**：查看系统使用指南
  - **关于我们**：查看系统信息和团队信息
  - **项目审查**：进入项目审查中心
- **布局**：下拉菜单
- **交互**：
  - 点击个人中心按钮展开/收起菜单
  - 鼠标移入菜单区域保持菜单打开
  - 鼠标移出菜单区域自动关闭菜单
  - 点击菜单项后自动关闭菜单
- **设计**：白色背景，阴影效果，简洁现代

#### 3.1.5 帮助与关于

- **功能**：提供系统帮助和关于信息
- **内容**：
  - **使用帮助**：设计人员和评审专家指南，包含基本操作、子应用管理、系统更新等帮助信息
  - **关于我们**：系统信息（版本 1.2.1）、描述、核心特性、技术栈和版权信息
- **设计**：弹窗形式，宽度 80vw，最大宽度 600px，清晰展示信息
- **交互**：点击个人中心菜单进入，关闭按钮退出，点击遮罩层关闭

#### 3.1.6 主题切换

- **功能**：支持浅色/深色主题切换
- **设计**：导航栏右侧主题按钮，使用太阳/月亮图标
- **交互**：点击切换主题，自动保存用户偏好到 localStorage

- **追踪矩阵**：需求文档维护“需求 -> PRD -> 测试 -> 证据”映射。

## 4. 技术实现

### 4.1 前端实现

- **Vue 3**：使用 Composition API，代码结构清晰
- **Vue Router**：主应用使用 HTML5 History 模式，子应用使用 Hash 模式，支持本地文件访问
- **Pinia**：使用 Pinia 进行状态管理，支持跨组件数据共享和持久化存储
- **Arco Design**：使用 Arco Design 组件库，提供统一的 UI 组件和设计规范
- **ECharts**：使用 ECharts 实现数据可视化图表，支持丰富的图表类型和交互
- **Axios**：使用 Axios 进行 HTTP 请求，支持请求拦截和响应拦截
- **Vue I18n**：使用 Vue I18n 实现国际化支持，支持多语言切换
- **CSS 变量**：统一的设计系统，便于主题管理
- **Less**：使用 Less 预处理器，提高样式开发效率
- **响应式设计**：适配不同屏幕尺寸
- **组件库**：可复用组件，提高开发效率
- **Vitest**：使用 Vitest 进行单元测试，提高代码质量
- **本地存储**：使用 localStorage 存储用户配置和数据
- **版本管理**：使用 versions 目录存储版本备份
- **一键更新**：使用 Node.js 脚本实现系统自动更新
- **设计规范**：遵循《系统设计规范》文档中的设计标准，确保界面风格一致

### 4.2 设计规范

本系统采用统一的设计规范，详细内容请参考《系统设计规范》文档。设计规范包括：

- **颜色系统**：基于 Arco Design 的颜色方案，包括主色调、功能色和中性色
- **排版系统**：统一的字体、字号和行高
- **组件样式**：按钮、卡片、表单等组件的标准样式
- **布局结构**：统一的布局方式和响应式设计
- **动画效果**：流畅的过渡动画和微交互
- **主题系统**：支持浅色和深色模式

设计规范是系统的标准风格，所有子应用和组件都应遵循此规范，确保整个系统的视觉一致性和用户体验。

### 4.3 构建配置

- **Vite**：快速的构建工具，支持热更新
- **多配置**：为每个子应用提供独立的构建配置
- **相对路径**：使用 `base: './'`，支持本地文件访问
- **IIFE 格式**：避免模块加载问题，提高兼容性
- **代码压缩**：使用 terser 进行代码压缩，减少文件体积

### 4.3 部署方案

- **静态部署**：部署到任何静态文件服务器
- **本地访问**：直接打开 HTML 文件运行
- **CDN 版本**：提供包含所有依赖的 CDN 版本

## 5. 部署与维护

### 5.1 部署方案

### 5.1.1 主系统部署

1. **打包**：`npm run build`
2. **部署**：将 `dist/` 目录部署到任何静态文件服务器
3. **访问**：通过服务器地址访问

### 5.1.2 子应用部署

1. **打包**：`npm run build:sample-app`
2. **部署选项**：
   - **独立部署**：将 `dist/sample-app/` 目录部署到服务器
   - **本地运行**：直接使用 `index.html` 或 `start.html`
   - **集成部署**：作为主系统的一部分部署

### 5.1.3 环境要求

- **服务器**：任何支持静态文件的服务器
- **浏览器**：Chrome、Firefox、Safari、Edge 等主流浏览器
- **网络**：本地运行无需网络，部署到服务器需要网络连接

### 5.2 维护指南

#### 5.2.1 代码维护

- **代码规范**：遵循 Vue 3 最佳实践，使用 ESLint 和 Prettier 确保代码质量
- **目录结构**：保持统一的目录结构，将复杂组件拆分为更小、更可维护的组件
- **命名规范**：
  - 子应用文件夹名：使用语义化的英文名，采用 kebab-case 命名风格（如 `sample-app`）
  - 组件文件名：使用 PascalCase 命名风格（如 `SampleApp.vue`）
  - 路由名称：使用 PascalCase 命名风格，前缀与子应用名称一致（如 `SampleAppHome`）
  - 变量和函数：使用 camelCase 命名风格
- **文档更新**：及时更新相关文档，确保文档与代码保持同步
- **代码组织**：遵循单一职责原则，实现可复用组件库
- **设计规范**：遵循《系统设计规范》文档中的设计标准，确保界面风格一致

#### 5.2.2 问题排查

- **浏览器控制台**：查看错误信息和性能指标
- **网络请求**：检查资源加载情况和网络性能
- **路由配置**：确认路由配置正确，检查路由跳转逻辑
- **打包配置**：检查打包配置参数，确保打包结果符合要求
- **性能监控**：使用浏览器开发工具监控应用性能

#### 5.2.3 版本管理

- **Git 管理**：使用 Git 进行版本控制，遵循 Git 工作流
- **分支策略**：主分支、开发分支、功能分支，确保代码质量
- **发布流程**：测试、构建、部署、验证，确保发布质量
- **版本控制**：保持文档版本与应用版本一致，记录版本变更历史
